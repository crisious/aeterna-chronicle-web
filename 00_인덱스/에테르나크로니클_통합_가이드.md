# ⚔️ 에테르나 크로니클 (Aeterna Chronicle) — 통합 가이드

> **프로젝트 상태**: 🟢 메인 콘텐츠 95% 완성 | 시나리오 콘텐츠 90% | 기술 명세 85%
> **최종 업데이트**: 2026-03-06
> **타겟 플랫폼**: 웹(Phase 1), 멀티엔진(Phase 2-3)
> **개발팀**: .NET 40대 게임 개발자 및 협력팀

---

## 📚 이 가이드에 대해

이 문서는 에테르나 크로니클 프로젝트의 **58개 마크다운 문서를 6개의 종합 가이드로 통합**한 것입니다.

### 구성
- **6개 마스터 문서**: 각 영역별 완전한 설계 명세
- **1개 이슈 트래커**: 미완료 항목 및 개선 계획
- **1개 정합성 체크리스트**: 릴리즈 검증 항목

### 사용 흐름

```
신규 팀원
    ↓
1️⃣ 이 문서 (통합 가이드) 읽기 — 프로젝트 전체상 파악
    ↓
2️⃣ GDD 최종본 읽기 — 핵심 설계 이해
    ↓
3️⃣ 맡은 영역별 마스터 문서 읽기 — 상세 명세 확인
    ↓
4️⃣ 이슈 트래커 확인 — 현재 진행 상황 및 장애물 파악
    ↓
5️⃣ 개발 착수
```

## 🎮 프로젝트 핵심 개요

### 게임 기본 정보
| 항목 | 내용 |
|------|------|
| **제목** | 에테르나 크로니클 (Aeterna Chronicle) |
| **장르** | 판타지 RPG (싱글플레이, 5장 분기형 스토리) |
| **플랫폼** | 웹(Phase 1) → PC(Unity) → 모바일(Unreal) 계획 |
| **타겟 플레이타임** | 메인 스토리 30-40시간 + 수집/경험 콘텐츠 |
| **기술 스택** | React(프론트) / .NET 8(백엔드) / PostgreSQL / WebSocket |
| **출시 목표** | 2026년 상반기 얼리액세스 |

### 세계관 핵심 설정
| 항목 | 내용 |
|------|------|
| **세계명** | 에테르나(Aeterna) |
| **배경 시대** | 세계력 3,412년(대망각 발생 후 212년) |
| **기술 수준** | 에테르-기계 문명(증기기관 + 신비 에너지 결정체) |
| **주요 갈등** | 기억 소멸 vs 기억의 수호 |
| **플레이어 역할** | 기억을 되찾으러 온 모험가 |

### 게임 구조
- **총 5개 챕터** + 4개 정규 엔딩 + 1개 패배 루트
- **7개 주요 지역** (에레보스, 실반헤임, 솔라리스 사막, 아르겐티움, 망각의 고원, 북방 영원빙원, 브리탈리아 자유항)
- **31명 NPC** (주연 7명, 조연 13명, 적대자 6명, 세계관 5명)
- **31개 몬스터 + 20개 보스**
- **100개 아이템** (장비 60 + 소모 20 + 재료 20)
- **50곡 이상 BGM**

## 📞 자주 묻는 질문 (FAQ)

### Q1: 지금 바로 개발을 시작할 수 있나요?
**A**: 기본적으로 yes, 부분적으로 no입니다.
- ✅ **가능**: Phase 1 웹 프론트엔드/백엔드 구조 설계, 기본 UI 구현, 챕터 1-2 전투 로직
- ❌ **보류**: NPC 대화 없이는 감정 선택 불가, 세계관 외전 없이는 코덱스 시스템 미구현

권장: **핵심 이슈 2개(NPC 대화, 세계관 외전) 50% 진행 후 개발 본격화**

### Q2: .NET 개발자로서 우선 확인할 것은?
**A**: 이 순서로 읽으세요:
1. `tech_architecture.md` — 전체 아키텍처
2. `game_systems.md` — 데이터 모델 및 계산 로직
3. 이슈 트래커 #7, #8, #9 — WebSocket, DB, 전투 시뮬레이션

### Q3: 엔딩 조건이 복잡한데, 어떻게 구현하나요?
**A**: `시나리오_마스터.md`의 "4개 정규 엔딩 + 패배 루트" 섹션과 이슈 트래커 #6 "대화 분기 → 플래그 매핑"을 함께 읽으세요. PostgreSQL 플래그 테이블 구조는 이슈 #8을 참고.

### Q4: 문서가 충돌하면?
**A**: `정합성_검증_체크리스트.md`의 "기준 문서(Single Source of Truth)"를 확인하세요. 예를 들어 엔딩 명칭 충돌은 `멀티엔딩_플래그_설계.md`를 기준으로 합니다.

### Q5: 신규 팀원이 합류하면?
**A**: 이 문서부터 읽고, GDD → 해당 영역 마스터 문서 순으로 진행. 약 4-6시간 소요.
