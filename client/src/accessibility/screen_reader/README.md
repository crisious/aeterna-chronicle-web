## 1. 개요

Canvas 기반 게임에서 HTML UI 오버레이 계층의 WCAG 2.1 AA 수준 스크린리더 지원.

**3대 축:**
1. **ARIA 라벨링** — 모든 인터랙티브 요소에 의미 부여
2. **포커스 관리** — 논리적 탭 순서 + 모달 포커스 트랩
3. **라이브 리전** — 동적 콘텐츠(전투 로그/채팅/알림)의 실시간 읽기

## 3. 포커스 관리

### 3.1 포커스 순서도

```
전체 포커스 흐름 (Tab 순서):

[게임 캔버스]
  ↕ (Tab 전환)
[HUD 요소]
  HP/MP → 스킬 1→2→3→4 → 포션 → 퀘스트
  ↕
[모달 열림 시 — 포커스 트랩]
  인벤토리: 탭 → 아이템 그리드 → 상세 → 닫기
  설정: 탭 → 설정 항목들 → 적용 → 닫기
  대화: 대사 → 선택지/다음 → (반복)
  ↕
[채팅]
  채널 탭 → 메시지 로그 → 입력 → 전송
```

### 3.2 포커스 트랩 규칙

| 상황 | 동작 |
|------|------|
| 모달 열림 | 모달 첫 포커스 요소로 이동, 외부 Tab 차단 |
| 모달 닫힘 | 모달 열기 전 포커스 위치로 복원 |
| Tab → 마지막 요소 | 첫 요소로 순환 |
| Shift+Tab → 첫 요소 | 마지막 요소로 순환 |
| Escape | 모달 닫기 (중첩 시 가장 안쪽부터) |

### 3.3 포커스 표시기

```css
/* 기본 포커스 인디케이터 */
:focus-visible {
  outline: 3px solid #4488FF;
  outline-offset: 2px;
}

/* 고대비 모드 포커스 인디케이터 */
.aeterna-high-contrast :focus-visible {
  outline: 3px solid #00FF00;
  outline-offset: 2px;
  box-shadow: 0 0 0 5px rgba(0, 255, 0, 0.3);
}

/* 게임 캔버스 내 커스텀 포커스 (캔버스 렌더링) */
.canvas-focus-indicator {
  border: 3px solid #FFD700;
  animation: pulse 1.5s ease-in-out infinite;
}
```

## 4. 라이브 리전 (aria-live)

### 4.1 리전 분류

| 리전 | aria-live | 사유 |
|------|----------|------|
| 전투 로그 | `assertive` | 즉시 읽어야 하는 중요 정보 (피격, 스킬 사용) |
| 타겟 정보 | `polite` | 타겟 변경 시 알림 (현재 읽기 중단 안 함) |
| 채팅 메시지 | `polite` | 새 메시지 도착 시 (대기 후 읽기) |
| 시스템 알림 | `assertive` | 레벨업, 퀘스트 완료, 아이템 획득 |
| 대사 텍스트 | `polite` | NPC 대사 진행 |
| 클래스 설명 | `polite` | 선택 변경 시 |
| 던전 상세 | `polite` | 선택 변경 시 |
| 아이템 상세 | `polite` | 선택 변경 시 |
| 퀘스트 트래커 | `polite` | 퀘스트 업데이트 시 |

### 4.2 전투 로그 읽기 제어

```typescript
/**
 * 전투 중 초당 수십 개의 로그가 생성될 수 있으므로
 * 스크린리더 과부하 방지를 위한 제어:
 *
 * 1. 중요도 필터: critical(HP 위험/사망) > high(스킬) > low(일반 타격)
 * 2. 배치 처리: 100ms 내 같은 유형은 합산 ("3회 타격, 총 450 데미지")
 * 3. 요약 모드: 5초마다 요약 읽기 ("HP 75%, 적 2마리 남음")
 */
interface CombatLogConfig {
  /** 최소 읽기 중요도 */
  minPriority: 'low' | 'high' | 'critical';
  /** 배치 간격 (ms) */
  batchInterval: number;
  /** 요약 모드 활성 */
  summaryMode: boolean;
  /** 요약 주기 (ms) */
  summaryInterval: number;
}
```
