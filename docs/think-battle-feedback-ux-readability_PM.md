## 3. 사용자 니즈 (Needs)

1. **N1 — 결과의 의미를 즉시 안다**: 내가 가한/받은 타격이 약점인지·저항인지·면역인지·크리티컬인지·빗나감인지를 0.5초 내 색·형태로 판독.
2. **N2 — 강한 타격은 강하게 느낀다**: 크리티컬·약점은 크기·색·모션으로 명확히 강조되어 타격감을 준다.
3. **N3 — 내 상태를 빠짐없이 파악한다**: 어떤 상태이상이 몇 개·얼마나 남았는지 한눈에, 누락 없이.
4. **N4 — 누구나 읽을 수 있다**: 색약·소형 화면에서도 형태/아이콘/충분한 폰트로 구분 가능.
5. **N5 — 혼란 없이 정돈된다**: 동시 다발 피드백이 겹치지 않고 정리되어 보인다.
6. **N6 — 하나의 시각 언어**: 전투 전체가 디자인 토큰(SSOT) 기반의 일관된 색·크기 체계를 따른다.

## 5. 수용 기준 (Acceptance Criteria)

> 형식: Given / When / Then + **측정 가능 지표**. 검증은 `verify-core` battle 시나리오 + UAT.

### AC-1 (US-1, US-2) — 데미지 타입별 색·크기 일치
- **Given** 전투에서 한 대상에 피해가 발생하고,
- **When** 데미지 팝업이 표시될 때,
- **Then** 색상은 `BATTLE_COLORS.damage`의 해당 타입(normal/critical/heal/miss/weak/resist/immune) 값과 **정확히 일치**하며, 폰트 크기는 `DAMAGE_POPUP_SIZE`의 해당 값과 일치한다.
- **측정**: 7개 타입 × (색 hex, 폰트 px) = **14개 단언 PASS**. 크리티컬 색 `0xffd700` 적용 (≠ 기존 `#FF4444`). 하드코딩 색/크기 잔존 **0건** (grep 감사).

### AC-2 (US-3) — 크리티컬/약점 강조
- **Given** 크리티컬 또는 약점 피해가 발생하고,
- **When** 팝업이 표시될 때,
- **Then** 크리티컬은 일반 대비 폰트 **≥1.8배**(32 vs 16)이며 카메라 쉐이크(`critShakeMs 150ms`)가 동반된다. 약점은 전용 색(`weak 0xff6b35`)과 강조 크기(22px)로 표시된다.
- **측정**: 크리/약점 강조 분기 단언 PASS, 일반과 시각적으로 구별됨(스크린샷 UAT).

### AC-3 (US-4) — 상태이상 가독성
- **Given** 캐릭터에 상태이상이 1개 이상 활성화되고,
- **When** 아이콘·라벨·중첩·지속시간 바가 렌더될 때,
- **Then** 모든 텍스트는 **≥14px**이며, 중첩(`×N`)·남은 시간 바가 함께 표시된다.
- **측정**: `auditLegibilityViolations()` 위반 **0건**. 라벨/중첩 폰트 **≥14px** 단언 PASS (현재 8px → 위반).

### AC-4 (US-5) — 상태이상 overflow
- **Given** 한 캐릭터에 상태이상이 **5개 이상** 활성화되고,
- **When** 표시가 갱신될 때,
- **Then** 상위 4개 아이콘 + **`+N` 오버플로 카운터**가 표시되어 총 개수를 인지할 수 있다.
- **측정**: 6개 부여 시 "4 아이콘 + `+2`" 표시 단언 PASS.

### AC-5 (US-6) — 14px 봉인
- **Given** 데미지/상태이상 텍스트가 생성되고,
- **When** 모바일 뷰포트 스케일이 적용될 때,
- **Then** `TextLegibilityGuard`(또는 동등 클램프)를 경유하여 최종 폰트 **< 14px가 0건**이다.
- **측정**: 4개 뷰포트(`Mobile Viewport 4/4`) × 데미지/상태이상 = 봉인 위반 **0건**.

### AC-6 (US-7) — 색약 대응
- **Given** 색약 모드가 켜지고,
- **When** 데미지 타입/상태이상이 표시될 때,
- **Then** 색 외에 **형태/아이콘/패턴**(예: 약점=`!`, 저항=`▼`, 면역=`✕`, 또는 `PatternOverlay`)이 병행되어 색 없이도 구분 가능하다.
- **측정**: 색약 시뮬레이션(deuteranopia/protanopia) 하에서 7개 데미지 타입 구분 가능(UAT 체크리스트), DESIGN.md §1 원칙 충족.

### AC-7 (US-8) — 겹침 방지
- **Given** 한 프레임에 **3개 이상** 팝업이 동시 발생하고,
- **When** 배치될 때,
- **Then** 팝업 간 시각적 충돌이 최소화(수직 스태거 또는 충돌 회피 오프셋)되어 각 숫자가 판독 가능하다.
- **측정**: 동시 3타 시나리오에서 팝업 바운딩박스 중첩률 **임계 이하**(UAT 스크린샷 판정).

### AC-8 (US-9) — miss/heal 일관성
- **Given** 공격이 빗나가거나 회복이 발생하고,
- **When** 결과가 표시될 때,
- **Then** 동일한 팝업 파이프라인으로 `miss`(회색·"MISS")·`heal`(초록·`+N`)이 표시된다.
- **측정**: miss/heal 팝업 렌더 단언 PASS, 시각 일관성 UAT 확인.

## 6. 범위 (Scope)

**In scope**: 데미지 팝업(색/크기/타입/크리·약점 강조/miss·heal), 상태이상 표시(폰트·overflow·중첩·지속), 14px 봉인 연결, 색약 형태 병행, 겹침 완화, SSOT 토큰 정합.

**Out of scope** (이번 스프린트 제외, 차기 백로그):
- 전투 밸런스·데미지 공식 변경 (수치 로직 불변, **표시 계층만** 개선)
- 신규 상태이상 종류 추가
- 사운드/보이스 피드백 개편 (별도 토픽)
- ATB 게이지·턴 큐 UI 리디자인 (별도 토픽)
