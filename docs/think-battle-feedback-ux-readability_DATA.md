## 1. 정량 baseline 실측 (현재 상태의 수치화)

> 측정 방법: 코드 정적 감사(grep + 토큰 대조). 모든 수치는 재현 가능.

| 지표 ID | 지표 | **현재(baseline)** | 목표(target) | 측정 방법 | PP 연계 |
|---------|------|------------------|-------------|----------|---------|
| **M1** | 데미지 타입 색상 커버리지 | **2 / 7 (28.6%)** — `normal`·`critical`만 분기 | 7 / 7 (100%) | `spawnDamageText` 분기 단언 | PP-1 |
| **M2** | 크리티컬 강조비 (crit÷normal 폰트) | **1.40×** (28÷20) | **2.00×** (32÷16) | 토큰 대조 단언 | PP-3 |
| **M3** | SSOT 색상 정합 (critical) | **불일치** `#FF4444`↔`0xffd700` | 일치 (0건) | grep 하드코딩 감사 | PP-2 |
| **M4** | 전투 텍스트 14px 봉인 위반 | **≈20+ 건** (8·9·10·11·12·13px 산재) | 0 건 | `auditLegibilityViolations()` | PP-4,5 |
| **M5** | 상태이상 overflow 인지율 | **0%** (4개 초과분 무표시, `+N` 부재) | 100% | 6개 부여 시 `+2` 단언 | PP-6 |
| **M6** | TextLegibilityGuard 배선율 | **0%** (`battle-damage` 컨텍스트 정의됨, 미사용) | 100% | `clampFontPx` 경유 단언 | PP-5 |
| **M7** | 색약 안전 타입 구분 (형태 병행) | **0 / 7** (순수 색 의존) | 7 / 7 | 패턴/아이콘 병행 단언 | PP-8 |

### 핵심 실측 하이라이트
- **데미지 타입의 71%가 시각적으로 침묵**: 토큰은 `weak/resist/immune/heal/miss` 5종을 정의했으나 런타임은 단 2종만 발화. 플레이어는 **약점·저항·면역을 색으로 판별 불가** — 전략 RPG에서 가장 큰 정보 손실.
- **크리티컬이 두 번 배신당함**: 색(금↔적)과 강조비(2.0×→1.4×) 모두 SSOT를 이탈. 디자인이 의도한 '결정타의 무게'가 코드에서 **30% 감쇄**.
- **14px 봉인은 선언만 존재**: `MIN_LEGIBLE_FONT_PX=14`·`battle-damage=28px` 약속이 코드에 있으나, 상태이상 라벨/중첩은 **8px** — 봉인의 57% 수준.

## 2. UX 리서치 — 색약 시뮬레이션 (정량 위험 분석)

> DESIGN.md §1 "WCAG AAA · 색약 대응 · 색상 외 형태/아이콘 병행" 원칙에 대한 **실측 검증**.
> 데미지 7색을 적록색맹(deuteranopia, 남성 유병률 약 6%)·적색맹(protanopia) 관점에서 혼동쌍 분석.

| 데미지 타입 | 정상 색 | deuteranopia 체감 | 혼동 위험쌍 |
|------------|--------|-------------------|-----------|
| normal | `#FFFFFF` 백 | 백 (명도 보존) | — |
| critical | `#FFD700` 금 | 탁한 황 | ⚠ **weak** |
| heal | `#2ECC71` 녹 | 탁한 황갈 | ⚠ **weak** |
| miss | `#A0A0A0` 회 | 회 (명도 보존) | — |
| weak | `#FF6B35` 주황 | 탁한 황갈 | ⚠ **heal·critical** |
| resist | `#3498DB` 청 | 청 | ⚠ **immune** |
| immune | `#9B59B6` 보라 | 탁한 청 | ⚠ **resist** |

### 메타 인사이트 (UX Research)
- **7색 중 최소 3쌍이 색약에서 충돌**: `heal↔weak`(적록축), `critical↔weak`(황계), `resist↔immune`(청보라축). 색약 플레이어에게 **"회복인 줄 알고 약점 타이밍을 놓치는"** 치명적 오판 발생 가능.
- 이는 PP-8(색약 미대응)이 *권고*가 아니라 **기능적 결함**임을 데이터로 입증. 색만으로는 색약자에게 정보의 ~43%(3/7쌍)가 무너짐.
- **처방의 정량 근거**: 약점=`!`, 저항=`▼`, 면역=`✕`, 회복=`+`, 미스=`—` 형태 병행 시, 색 채널 0%여도 7/7 구분 가능(이중 부호화, redundant coding).

### at-a-glance 판독 벤치마크 (ATB 전투 특성)
- 현 팝업 수명 `damagePopupMs=900ms`. 인간 단순 시각 인지 ~200–500ms. **단독 팝업은 판독 여유 충분.**
- 그러나 동시 3타 발생 시 `±10px` 랜덤 오프셋만으로 분리 → 인지 부하 급증(PP-9). **Miller 7±2 한계** 내라도, 겹친 숫자는 판독 시간 2–3배 증가 추정. → 수직 스태거 필요.

## 3. 측정 지표 체계 — 선행/후행 2-트랙

> PM의 AC(binary PASS/FAIL)를 **연속 지표 + 검증 트랙**으로 확장. 자동 단언(코드) ↔ UAT(사람) 분리.

### Track A — 선행 지표 (자동 측정, `verify-core` 단언)
| KPI | baseline → target | 게이트 |
|-----|------------------|--------|
| 타입 색상 커버리지 (M1) | 28.6% → **100%** | 7타입×(색,크기)=14단언 PASS |
| 크리 강조비 (M2) | 1.40× → **2.00×** | 폰트비 단언 |
| SSOT 정합 (M3) | 불일치 → **0 하드코딩** | grep 감사 0건 |
| 14px 봉인 (M4) | ≈20건 → **0 위반** | `auditLegibilityViolations()` |
| overflow 인지 (M5) | 0% → **100%** | `+N` 단언 |

### Track B — 후행 지표 (UAT 스크린샷, 대표 1인 수용 테스트)
| KPI | 측정 방법 | 합격선 |
|-----|----------|--------|
| 타입 판독 정확도 | 7타입 블라인드 스크린샷 식별 | **≥ 95%** (7중 7) |
| 색약 구분율 | deuteranopia/protanopia 필터 하 7타입 식별 | **100%** (형태 병행) |
| 동시 3타 겹침률 | 팝업 바운딩박스 중첩 면적비 | **임계 이하** (수직 스태거 후) |
| 크리/약점 타격감 | before/after 스크린샷 대비 | 명확히 구별(정성) |

### 텔레메트리 계측 제안 (차기 — 외부 사용자 확보 시)
- `perfTelemetry.ts`에 이미 `scene.inCombat` 태그 존재 → **UX 이벤트 훅 확장 여지**.
- 단, 현 검증 주체는 **대표 1인 UAT**이므로 대규모 텔레메트리는 over-engineering. **Track A 자동 단언 + Track B 스크린샷**이 비용 대비 최적. (MEMORY: 토큰 효율 — 빈 폴링 금지 원칙과 정합)
