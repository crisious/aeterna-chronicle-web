## 2) 등급별 디자인 규칙

### 2.1 일반 몬스터 (Normal)

| 요소 | 규칙 |
|------|------|
| 크기 | S(32×32) ~ M(64×64) |
| 외곽선 | 2px 검정, 오라 없음 |
| 디테일 | 기본 — 종 식별 가능한 최소 디테일 |
| 색상 | 지역 팔레트 1차/2차 색상 |
| 애니메이션 | idle 4프레임, attack 4프레임, hit 2프레임, death 4프레임 |
| 방향 | 2방향 (정면/측면, 미러링) |
| 실루엣 | 16×16 축소 시 종 구분 가능 |

### 2.2 엘리트 몬스터 (Elite)

| 요소 | 규칙 |
|------|------|
| 크기 | M(64×64) ~ L(96×96) |
| 외곽선 | 2px 검정 + 1px 색상 오라 (등급 색상) |
| 디테일 | 일반 대비 추가 장식/문양/변이 부위 |
| 색상 | 지역 팔레트 + 발광색 강조 |
| 애니메이션 | idle 4, attack 6, hit 2, death 4, special 4 |
| 방향 | 2방향 |
| 차별화 | 동일 지역 일반 몬스터의 "강화 변이"로 디자인 |

### 2.3 보스 몬스터 (Boss)

| 요소 | 규칙 |
|------|------|
| 크기 | L(96×96) ~ XL(128×128) |
| 외곽선 | 3px 검정 + 2px 강한 오라 + 배경 이펙트 |
| 디테일 | 최고 수준. 고유 장식, 문양, 특수 부위 |
| 색상 | 지역 팔레트 + 금/은 강조 + 전용 발광 |
| 애니메이션 | idle 4, attack 6, hit 2, death 6, special 6, phase 전환 |
| 방향 | 정면 중심 (보스전 고정 구도) |
| 페이즈 변화 | HP 50% 이하 시 외형 변화 (2페이즈) |

### 2.4 레이드 보스 (Raid)

| 요소 | 규칙 |
|------|------|
| 크기 | XL(128×128) 이상 (최대 256×256) |
| 외곽선 | 3px 검정 + 3px 풍부한 오라 + 파티클 + 배경 전체 변화 |
| 디테일 | 극한 디테일. 부위별 독립 애니메이션 |
| 색상 | 전용 팔레트 (지역 기반 + 전용 강조) |
| 애니메이션 | idle 6, attack 8, hit 2, special 8, phase 3개 |
| 방향 | 정면 고정 |
| 페이즈 변화 | 3페이즈 (75%/50%/25% HP) 외형 변화 |

## 3) 크기 규격 상세

### 3.1 크기-등급 매핑

| 크기 | 프레임 | 타일 점유 | 대상 예시 |
|------|--------|----------|----------|
| S (Small) | 32×32 px | 0.5×0.5 | 박쥐, 쥐, 벌레, 포자 |
| M (Medium) | 64×64 px | 1×1 | 늑대, 골렘, 좀비, 해골 |
| L (Large) | 96×96 px | 1.5×1.5 | 미니보스, 대형 수호자 |
| XL (Extra Large) | 128×128 px | 2×2 | 챕터 보스, 레이드 보스 |
| XXL (Colossal) | 256×256 px | 3×3 | 레이드 최종 보스 전용 |

### 3.2 크기 비율 가이드

```
                            ┌──────────────┐
                            │              │
               ┌─────────┐  │   XL 128px   │
    ┌──────┐   │         │  │              │
 ┌──┐      │   │  L 96px │  │              │
 │S │ M 64 │   │         │  │              │
 │32│      │   │         │  │              │
 └──┘      │   └─────────┘  │              │
    └──────┘                 └──────────────┘
```

## 4) 실루엣 구분 가이드

### 4.1 카테고리별 실루엣 원칙

| 카테고리 | 실루엣 키워드 | 금지 실루엣 |
|----------|-------------|------------|
| 비행형 | 날개 확장, 상체 넓음 | 접힌 날개 (지상형과 혼동) |
| 인간형 | 직립, 무기 돌출 | 완전 대칭 (캐릭터와 혼동) |
| 수생형 | 유선형, 지느러미/촉수 돌출 | 직립형 |
| 곤충형 | 다리 많음, 안테나, 분절 | 4족형과 혼동 |
| 4족형 | 수평, 다리 4개, 꼬리 | 인간형/2족과 혼동 |
| 무정형 | 불규칙, 흐르는 형태 | 명확한 대칭 |
| 식물형 | 뿌리/줄기/꽃 구조 | 동물형 실루엣 |
| 기계형 | 기하학적, 기어/파이프 돌출 | 유기체형 |
| 유령형 | 하체 소멸/흐름, 상체 인간형 | 완전한 인간형 |

### 4.2 동일 지역 내 차별화 규칙

1. **같은 카테고리 2종 이하**: 지역당 동일 카테고리 몬스터 최대 2종
2. **크기 차별화**: 같은 카테고리 시 반드시 크기 등급 다르게
3. **돌출부 차별화**: 뿔/날개/꼬리/무기 중 최소 1개 다르게
4. **16px 테스트**: 16×16 축소 시 서로 다른 형태인지 확인

## 6) AI 생성 프롬프트 템플릿

### 6.1 일반 몬스터 프롬프트

```
Template: MONSTER_NORMAL

2D pixel art RPG monster sprite, {monster_name},
{category_type} creature type,
{appearance_description},
{region_name} region, {region_color_palette},
{size_class} size ({pixel_size}),
idle pose, front-facing,
dark fantasy style, menacing but not gory,
top-left 45-degree lighting, 3-step shading,
2px black outline, no anti-aliasing,
transparent background, game sprite ready

Variables:
  monster_name: 몬스터 이름
  category_type: "undead humanoid" / "spectral" / "plant" / "beast" 등
  appearance_description: 카탈로그 외형 개요 확장
  size_class: "small" / "medium" / "large"
  pixel_size: "32x32" / "64x64" / "96x96"
```

### 6.2 엘리트 몬스터 프롬프트

```
Template: MONSTER_ELITE

2D pixel art RPG elite monster sprite, {monster_name},
enhanced variant of {base_monster_reference},
{enhanced_features},
{aura_color} faint aura glow around edges,
{region_name} region, {region_color_palette},
{size_class} size ({pixel_size}),
menacing idle pose, front-facing,
extra ornamental details, glowing accents,
2px black outline + 1px {aura_color} outer glow,
transparent background, game sprite ready

Variables:
  base_monster_reference: 기반 일반 몬스터 이름
  enhanced_features: "larger horns, armored plates, glowing eyes" 등
  aura_color: 등급 오라 색상
```

### 6.3 네거티브 프롬프트 (몬스터 전용)

```
Negative: 3D render, realistic, cute, chibi, friendly-looking,
watermark, text, low quality, blurry,
too many details for pixel size, anti-aliased edges,
human anatomy (unless humanoid), extra limbs beyond design,
modern elements, sci-fi (unless machine region),
same silhouette as another monster in region
```
