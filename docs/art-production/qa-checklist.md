## 1) QA 프로세스 개요

```
[AI 생성 완료] → [자동 QA (qa_checker.py)] → [수동 QA (이 체크리스트)]
                                              → [승인] → 후처리 파이프라인
                                              → [반려] → 재생성/수정
```

### 1.1 QA 단계

| 단계 | 도구 | 판정 기준 |
|------|------|----------|
| 1차 자동 | `qa_checker.py` | 해상도, 알파, 색상 범위, 네이밍 |
| 2차 수동 | 이 체크리스트 | 스타일 일관성, 디자인 적합성, 게임 내 시인성 |
| 3차 인게임 | Unity/웹 테스트 | 실제 화면 렌더링 확인 |

## 2) 자동 QA 항목 (qa_checker.py)

### 2.1 해상도 규격

| 에셋 타입 | 규격 (px) | 허용 오차 |
|-----------|----------|----------|
| 캐릭터 스프라이트 | 64×64 | ±0 |
| 몬스터 (S) | 32×32 | ±0 |
| 몬스터 (M) | 48×48 | ±0 |
| 몬스터 (L) | 64×64 | ±0 |
| 몬스터 (XL) | 96×96 | ±0 |
| 아이콘 | 32×32 | ±0 |
| 이펙트 (S/M/L) | 64/128/256 | ±0 |
| 타일 | 32×32 | ±0 |
| 배경 레이어 | 가변 (16:9) | 2의 거듭제곱 권장 |
| UI 패널 | 가변 | 최대 2048 |

### 2.2 투명도 / 알파 채널

- [ ] RGBA 모드 (32bit) 확인
- [ ] 배경 영역 완전 투명 (alpha = 0)
- [ ] 스프라이트 외곽 안티앨리어싱 없음 (반투명 비율 < 5%)
- [ ] 스프라이트 내부 불투명 (alpha = 255)
- [ ] 투명 비율 10~80% 범위 (전체 불투명 또는 전체 투명은 이상)

### 2.3 색상 일관성

- [ ] 고유 색상 256개 이하 (픽셀아트 기준)
- [ ] 평균 채도 0.95 이하
- [ ] 지역 팔레트 범위 내 (color_correct.py의 지역별 팔레트 참조)
- [ ] 등급별 테두리/발광 색상 규격 준수

### 2.4 네이밍 규칙

```
{타입}_{지역}_{이름}_{변형}_{크기}.png
```

- [ ] 형식: `char_erebos_ether-knight_idle-front_64x64.png`
- [ ] 소문자 + 하이픈만 사용
- [ ] 공백 없음
- [ ] 타입 접두사: `char`, `mon`, `npc`, `icon`, `tile`, `bg`, `vfx`, `ui`

### 2.5 프레임 정합

- [ ] 동일 캐릭터/몬스터의 모든 프레임 크기 일치
- [ ] 모션당 프레임 수 규격 준수 (idle:4, walk:6, attack:6, cast:5, hit:3, death:5)
- [ ] 5방향 세트 완결 (D, DL, L, UL, U)

## 3) 수동 QA 항목

### 3.1 스타일 일관성 (style-guide.md 대조)

- [ ] **비주얼 톤**: 다크 판타지 픽셀아트 톤 유지
- [ ] **라인 스타일**: 2px 검정 외곽선, 안티앨리어싱 없음
- [ ] **셰이딩**: 2~3단계 명암 (하이라이트/중간/그림자)
- [ ] **지역 색감**: 해당 지역 팔레트와 조화
- [ ] **장비 화려함**: 등급에 맞는 디테일 수준
- [ ] **참조 작품 톤 부합**: Octopath/Chrono Trigger 감성

### 3.2 디자인 적합성

- [ ] **실루엣 구분**: 축소 시 (16x16) 다른 에셋과 구별 가능
- [ ] **캐릭터 비율**: 머리:몸 = 1:2~1:3 (SD 비율)
- [ ] **무기/방어구 가시성**: 클래스 식별 가능
- [ ] **표정/포즈 적합**: 역할/성격 반영
- [ ] **몬스터 등급 구분**: 일반 < 엘리트 < 보스 < 레이드 시각적 격차
- [ ] **아이콘 가독성**: 32×32에서 아이템 종류 즉시 인식

### 3.3 애니메이션 적합성 (프레임 시퀀스)

- [ ] **대기 모션**: 미세 움직임, 자연스러운 루프
- [ ] **이동 모션**: 발 접지 타이밍 자연스러움
- [ ] **공격 모션**: 예비 동작 → 타격 → 후딜 명확
- [ ] **스킬 시전**: 집중 → 발동 → 후폭풍 연출
- [ ] **피격 모션**: 방향성 피격 반응
- [ ] **사망 모션**: 자연스러운 쓰러짐

### 3.4 기술 적합성

- [ ] **GPU 최적화**: 시트 크기 2의 거듭제곱
- [ ] **파일 크기**: 단일 시트 < 10MB
- [ ] **메타데이터 JSON**: 프레임 좌표/크기 정합
- [ ] **미러링 호환**: 좌우 반전 시 부자연스러운 요소 없음

## 4) 등급별 QA 심사 기준

| 항목 | 최소 기준 (Pass) | 우수 기준 (Excellent) |
|------|-----------------|---------------------|
| 해상도 정합 | 규격 정확 일치 | - |
| 알파 채널 | RGBA, 배경 투명 | 반투명 0% |
| 색상 수 | ≤ 256 | ≤ 64 |
| 팔레트 일치 | 범위 내 | 마스터 팔레트 색상만 사용 |
| 실루엣 구분 | 16x16에서 구별 | 8x8에서도 구별 |
| 애니메이션 | 모션 자연스러움 | 타격감/무게감 전달 |
| 전체 인상 | 게임 톤 부합 | 참조 작품 수준 |
