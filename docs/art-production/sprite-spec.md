## 1) 캐릭터 스프라이트 시트

### 1.1 기본 규격

| 항목 | 규격 |
|------|------|
| 프레임 크기 | 64×64 px |
| 외곽선 | 2px 검정, 안티앨리어싱 없음 |
| 색상 | RGBA 32bit, 투명 배경 |
| 피벗 | 하단 중앙 (x: 0.5, y: 0.9) |
| 시트 최대 크기 | 2048×2048 px |
| 프레임 간 패딩 | 2px |

### 1.2 방향 체계 — 8방향

| 인덱스 | 방향 | 약어 | 미러링 |
|--------|------|------|--------|
| 0 | 아래 (정면) | D | - |
| 1 | 아래왼쪽 | DL | - |
| 2 | 왼쪽 | L | - |
| 3 | 위왼쪽 | UL | - |
| 4 | 위 (뒷면) | U | - |
| 5 | 위오른쪽 | UR | UL 수평 미러 |
| 6 | 오른쪽 | R | L 수평 미러 |
| 7 | 아래오른쪽 | DR | DL 수평 미러 |

> **실제 제작**: 5방향(0~4)만 제작. 5/6/7은 수평 미러링으로 자동 생성.

### 1.3 모션 정의 — 7종

| 모션 | 코드 | 프레임 수 | FPS | 루프 | 용도 |
|------|------|----------|-----|------|------|
| 대기 | `idle` | 4 | 5 | ✅ | 대기 상태 |
| 이동 | `walk` | 6 | 10 | ✅ | 필드/전투 이동 |
| 공격 (근접) | `attack_melee` | 6 | 12 | ❌ | 에테르 기사, 기억 파괴자 |
| 공격 (원거리) | `attack_ranged` | 6 | 12 | ❌ | 그림자 직조사, 기억술사 |
| 스킬 시전 | `cast` | 5 | 10 | ❌ | 마법/스킬 사용 |
| 피격 | `hit` | 3 | 12 | ❌ | 피해 반응 |
| 사망 | `death` | 5 | 8 | ❌ | HP 0 |

### 1.4 프레임 수 합계 (캐릭터 1체당)

| 구분 | 계산 | 프레임 수 |
|------|------|----------|
| 모션별 프레임 합계 | 4+6+6+6+5+3+5 | 35 |
| × 실제 제작 방향 | ×5 | 175 |
| + 미러링 방향 | ×3 (미러, 실제 제작 불필요) | +105 (자동) |
| **총 실사용 프레임** | | **280** |
| **실제 제작 프레임** | | **175** |

### 1.5 시트 레이아웃

```
스프라이트 시트 레이아웃 (1체분):

         idle(4)  walk(6)  atk_m(6) atk_r(6) cast(5)  hit(3)  death(5)
Dir 0(D) [....] [......] [......] [......] [.....] [...] [.....] 
Dir 1(DL)[....] [......] [......] [......] [.....] [...] [.....] 
Dir 2(L) [....] [......] [......] [......] [.....] [...] [.....] 
Dir 3(UL)[....] [......] [......] [......] [.....] [...] [.....] 
Dir 4(U) [....] [......] [......] [......] [.....] [...] [.....] 

열: 35 프레임 × 64px = 2240px → 2048 제한 시 2행 분할
행: 5 방향 × 64px = 320px

→ 실제: 2행 분할 (모션 전반 / 모션 후반)
  행1: idle(4) + walk(6) + atk_m(6) + atk_r(6) = 22프레임 × 64 = 1408px
  행2: cast(5) + hit(3) + death(5) = 13프레임 × 64 = 832px
  × 5방향 = 10행 × 64px = 640px

시트 크기: 1408 × 640 px (2048 제한 내)
```

### 1.6 네이밍 규칙

```
패턴: {class}_{advancement}_{sheet_type}.png

예시:
  ether_knight_base_sprites.png      — 에테르 기사 기본직
  ether_knight_base_sprites.json     — 메타데이터 (atlas)
  memory_weaver_2nd_sprites.png          — 기억술사 2차 전직
  shadow_weaver_3rd_sprites.png      — 그림자 직조사 3차 전직

Atlas JSON 구조:
{
  "frames": {
    "idle_D_0": { "x": 0, "y": 0, "w": 64, "h": 64 },
    "idle_D_1": { "x": 66, "y": 0, "w": 64, "h": 64 },
    ...
  },
  "animations": {
    "idle": { "frames": ["idle_D_0","idle_D_1","idle_D_2","idle_D_3"], "fps": 5, "loop": true },
    ...
  },
  "meta": {
    "size": { "w": 1408, "h": 640 },
    "class": "ether_knight",
    "advancement": "base"
  }
}
```

## 2) 몬스터 스프라이트 시트

### 2.1 기본 규격

| 항목 | 일반 (S/M) | 엘리트 (M/L) | 보스 (L/XL) | 레이드 (XL/XXL) |
|------|-----------|-------------|------------|----------------|
| 프레임 크기 | 32/64 px | 64/96 px | 96/128 px | 128/256 px |
| 방향 | 2 (정면/측면) | 2 | 1 (정면) | 1 (정면) |
| idle | 4 | 4 | 4 | 6 |
| attack | 4 | 6 | 6 | 8 |
| hit | 2 | 2 | 2 | 2 |
| death | 4 | 4 | 6 | 10 |
| special | - | 4 | 6 | 8 |
| phase | - | - | 6 | 10 |

### 2.2 프레임 수 합계

| 등급 | 모션 프레임 | × 방향 | 총 프레임 | 실제 제작 |
|------|-----------|--------|----------|----------|
| 일반 | 14 | ×2 | 28 | 28 (미러링 없음) |
| 엘리트 | 20 | ×2 | 40 | 40 |
| 보스 | 30 | ×1 | 30 | 30 |
| 레이드 | 44 | ×1 | 44 | 44 |

### 2.3 몬스터 시트 레이아웃

```
일반 몬스터 (M 64px):
  행1(정면): idle(4) + attack(4) + hit(2) + death(4) = 14 × 64 = 896px
  행2(측면): idle(4) + attack(4) + hit(2) + death(4) = 14 × 64 = 896px
  시트: 896 × 128 px

보스 (XL 128px):
  행1: idle(4) + attack(6) = 10 × 128 = 1280px  
  행2: hit(2) + death(6) + special(6) = 14 × 128 = 1792px
  행3: phase(6) = 6 × 128 = 768px
  시트: 1792 × 384 px
```

### 2.4 네이밍 규칙

```
패턴: mon_{region}_{name}_{grade}.png

예시:
  mon_erebos_memory_dust_normal.png       — 에레보스 기억 먼지 (일반)
  mon_erebos_fog_guardian_elite.png        — 에레보스 안개 수호자 (엘리트)
  mon_erebos_ruin_watcher_boss.png         — 에레보스 폐허의 감시자 (보스)
  mon_erebos_nebulos_raid.png              — 에레보스 네뷸로스 (레이드)
```

## 3) NPC 스프라이트 시트

### 3.1 규격

| 항목 | 규격 |
|------|------|
| 프레임 크기 | 64×64 px |
| 방향 | 1 (정면) — 필드 배치용 |
| 모션 | idle (4프레임) + 대화 반응 (2프레임) = 6프레임 |
| 시트 | 384 × 64 px |

### 3.2 네이밍

```
패턴: npc_{region}_{name}.png

예시:
  npc_erebos_cryo.png
  npc_silvanhime_seraphine.png
```

## 4) 이펙트 스프라이트 시트

### 4.1 규격

| 항목 | 규격 |
|------|------|
| 프레임 크기 | 64×64 / 128×128 px (이펙트 크기별) |
| 방향 | 무방향 (전방위 이펙트) |
| 모션 | 시작(2) + 루프(4~6) + 종료(2) = 8~10프레임 |
| 블렌딩 | Additive (발광) / Normal (물리) |

### 4.2 네이밍

```
패턴: vfx_{category}_{name}.png

예시:
  vfx_skill_ether_slash.png      — 에테르 참격
  vfx_hit_physical_01.png        — 물리 타격
  vfx_buff_heal_aura.png         — 치유 오라
```

## 6) 품질 검증 규칙

| 항목 | 기준 | 검증 방법 |
|------|------|----------|
| 프레임 크기 일관 | 동일 시트 내 모든 프레임 동일 크기 | 자동 스크립트 |
| 투명 배경 | 알파 채널 존재, 배경 완전 투명 | 알파 검증 |
| 외곽선 | 2px 검정 (#000000) | 엣지 검출 |
| 패딩 | 프레임 간 2px 이상 | 경계 검사 |
| 피벗 일관 | 캐릭터 발 위치 ±2px 내 | 수동/반자동 |
| 시트 크기 | 2048×2048 이하 | 자동 |
| 네이밍 | 규칙 준수 | regex 검사 |
| Atlas 정합 | JSON과 PNG 프레임 수 일치 | 자동 |
