| 차수 | 위치 | 본 스프린트 변경 |
|---|---|---|
| **1차 SSOT** | 본 파일 + `/DESIGN.md` §5 | 신규 — 데미지/상태이상 가독성 정본 |

**중요 — 기존 분산 출처 통폐합**: 현재 상태이상 색상이 세 곳에 흩어져 있어 가독성·일관성 적자의 근원이옵니다.

## 1. 데미지 팝업 가독성 토큰 (DAMAGE_READABILITY)

### 1.1 진단 — 기존 약점

| # | 약점 | 근거 (현재 코드) |
|---|---|---|
| D1 | 한 방향 그림자만 → 밝은 VFX 위 묻힘 | `.ac-dmg { text-shadow: 2px 2px 0 #000 }` |
| D2 | 색약 사용자 크리티컬/약점/저항 구분 불가 | `damage.critical #FFD700` vs `damage.weak #FF6B35` — **색만** |
| D3 | resist·immune·miss 14px → 난전 중 안 보임 | `DAMAGE_POPUP_SIZE.resist=14` 등 |
| D4 | 다단히트·전체공격 시 팝업 겹침 | 스태거/오프셋 명세 부재 |

### 1.2 처방 — 8방향 픽셀 아웃라인

색상 외 **테두리**로 어떤 배경 위에서도 분리. 픽셀 정합성 위해 1px 단색 검정 8방향.

```
text-shadow:
  -1px -1px 0 #000, 0 -1px 0 #000, 1px -1px 0 #000,
  -1px  0   0 #000,                1px  0   0 #000,
  -1px  1px 0 #000, 0  1px 0 #000, 1px  1px 0 #000;
```

> `prefers-contrast: more` 에서는 2px 로 확대 (기존 §8 접근성 블록 확장).

### 1.3 처방 — 색약 대응 형태 마커 (prefix 글리프)

색에 의존하지 않고 **형태 prefix** 로 데미지 종류를 1차 식별. 픽셀 친화 유니코드.

| 종류 | 마커 | 의미론 | 색(유지) | 크기(보정) |
|---|:---:|---|---|---|
| normal | — | 기본 | `#FFFFFF` | 16 → **16** |
| critical | `★` | 치명타 강조 | `#FFD700` | 32 → **36** |
| weak (약점) | `▲` | "더 박힌다" 상향 | `#FF6B35` | 22 → **24** |
| resist (저항) | `▼` | "덜 박힌다" 하향 | `#3498DB` | 14 → **18** |
| immune (무효) | `⊘` | 차단 | `#9B59B6` | 14 → **18** |
| heal | `✚` | 치유 십자 | `#2ECC71` | 18 → **20** |
| miss | `✗` | 빗나감 | `#A0A0A0` | 14 → **16** |

> 마커는 숫자 **앞**에 반각 간격으로 배치: `★ 9999`, `▲ 1240`, `▼ 86`, `⊘ MISS`.
> 색맹 시뮬레이션(적록/청황/전색맹) 3종 전부에서 ★/▲/▼/⊘ 형태가 유지됨 → 색 없이도 판별.

### 1.4 처방 — 다단히트 스태거 (겹침 해소)

동일 타깃에 연속 데미지(다단히트·전체공격) 발생 시:

| 파라미터 | 값 | 설명 |
|---|---|---|
| `staggerDelayMs` | 90 | n번째 팝업 등장 지연 = `n × 90ms` |
| `staggerOffsetX` | ±14 | 짝/홀 인덱스 좌우 분산 (`-14, +14, -14…`) |
| `staggerOffsetY` | -6 | n번째마다 시작 높이 누적 상향 |
| `maxConcurrent` | 6 | 7개째부터는 합산 표기 `… ×N` |

## 2. 상태이상 통합 토큰 (STATUS_FEEDBACK — 15종 단일 출처)

각 효과 = **색(가독성 보정) + 글리프(형태) + 카테고리 테두리 + 대비등급**. 색 하나라도 빠지면 색약·저대비 환경에서 식별 실패하므로 **3중 잉여(색·형태·테두리)** 로 설계.

| id | 라벨 | 글리프 | 색 v0 (기존) | **색 v1 (보정)** | 카테고리 | 보정 사유 |
|---|---|:---:|---|---|---|---|
| poison | 독 | `☠` | `#00FF00` | `#3DD13D` | dot | 순녹색 명도 과다 → 흰 숫자와 충돌, 톤다운 |
| burn | 화상 | `▲` | `#FF4400` | `#FF5A2C` | dot | 채도 유지·명도 +6% |
| bleed | 출혈 | `◆` | `#CC0000` | `#E03A3A` | dot | 어두워 배경 묻힘 → 명도 상향 |
| stun | 기절 | `✦` | `#FFFF00` | `#FFE14D` | control | 순노랑 → 크리티컬 금색과 분리 |
| silence | 침묵 | `⊘` | `#8844AA` | `#A35FD1` | control | 명도 +12% (대비 확보) |
| freeze | 빙결 | `❄` | `#44BBFF` | `#5FC8FF` | control | 미세 상향 |
| charm | 매혹 | `♥` | `#FF66CC` | `#FF7FD1` | control | 유지급 |
| slow | 감속 | `↓` | `#6688CC` | `#6E92D6` | debuff | 대비 보강 |
| blind | 실명 | `◐` | `#444444` | `#8A8A9E` | debuff | **FAIL→AA** 거의 검정 → 회보라 |
| curse | 저주 | `✸` | `#660066` | `#9B4DC4` | debuff | **FAIL→AA** 명도 대폭 상향 |
| attack_up | 공격↑ | `⚔` | `#FF8800` | `#FF9933` | buff | 유지급 |
| defense_up | 방어↑ | `⛊` | `#4488FF` | `#5599FF` | buff | 유지급 |
| haste | 가속↑ | `⏩` | `#44FF88` | `#4DE88A` | buff | 명도 톤다운(눈부심) |
| regen | 재생↑ | `✚` | `#88FF44` | `#9BE85A` | buff | 톤다운 |
| shield | 보호 | `⛨` | `#FFDD44` | `#FFD94D` | buff | 유지급 |

> 글리프는 폴백·스펙 표현. 실제 런타임은 `assets/generated/ui/icons/status/status_{id}.png`(18×18) 우선, 로드 실패 시 글리프 텍스트 폴백 — 이중 안전.

### 2.1 카테고리 테두리 색 (형태 잉여 #2)

아이콘 칩 테두리로 효과 **부류**를 한눈에. 색약에도 테두리 **두께·패턴**으로 구분 (`PatternOverlay.ts` 연동).

| 카테고리 | 테두리 색 | 패턴 | 의미 |
|---|---|---|---|
| dot (지속피해) | `#FF7A3C` | 점선 | 매 턴 깎임 |
| control (행동제어) | `#FF4D6A` | 실선 굵게 | 행동 봉쇄 위험 |
| debuff (약화) | `#9B59B6` | 실선 | 능력 저하 |
| buff (강화) | `#FFD700` | 이중선 | 아군 이득 |

## 3. 상태이상 칩 컴포넌트 (.ac-status-chip)

기존 `.ac-status-icon` 12px → **18px 칩**으로 승급. 글리프·테두리·스택·잔여턴 4정보 통합.

```
┌────────┐  ← 카테고리 테두리(색+패턴)
│  ☠  ③│  ← 글리프 + 스택뱃지(우상단, 2 이상일 때만)
│▓▓▓▓░░│  ← 잔여 턴 게이지(하단 2px, 카테고리 색)
└────────┘
```

| 요소 | 사양 |
|---|---|
| 칩 크기 | 18×18 (모바일 16) |
| 글리프 | 중앙, 픽셀 폰트, 11px |
| 스택 뱃지 | 우상단 8px 원, `stacks ≥ 2` 일 때만 노출 |
| 잔여턴 게이지 | 하단 2px, `remainingDuration / maxDuration` |
| 정렬 우선순위 | control > dot > debuff > buff (위험 먼저) |
| 최대 표시 | 4개 + `+N` 오버플로 칩 |

## 4. 접근성 매트릭스 (배경 `#0D0D1A` 기준)

WCAG 명도 대비. **AA = 4.5:1**(텍스트)·**3:1**(아이콘/그래픽). 색만으로 AA 미달인 효과는 글리프+테두리로 보장.

| 효과 | 색 v1 | 대비비(근사) | 색 단독 등급 | 글리프 보장 |
|---|---|---|---|---|
| poison `#3DD13D` | 🟢 | ~10:1 | AAA | ✓ |
| burn `#FF5A2C` | 🟠 | ~6:1 | AA | ✓ |
| bleed `#E03A3A` | 🔴 | ~5:1 | AA | ✓ |
| stun `#FFE14D` | 🟡 | ~14:1 | AAA | ✓ |
| freeze `#5FC8FF` | 🔵 | ~9:1 | AAA | ✓ |
| blind `#8A8A9E` (구 `#444`) | ⚪ | ~7:1 (구 ~2.5 ❌) | AA | ✓ |
| curse `#9B4DC4` (구 `#606`) | 🟣 | ~4:1 (구 ~1.8 ❌) | AA(graphic) | **✓ 필수** |
| slow `#6E92D6` | 🔵 | ~7:1 | AA | ✓ |

> **핵심**: `blind`·`curse` 는 구 색상이 대비 FAIL(글자 식별 불가)이었음. v1 색 보정 + 글리프 + 테두리 3중으로 해소. 색약 3종(적록 8%·청황·전색맹)은 글리프 형태로 100% 판별 보장.

## 5. ASCII 모킹업 — 개선 전 / 후

### 5.1 데미지 팝업 (난전, 3타깃 동시)

```
[BEFORE — 색만, 겹침, 묻힘]              [AFTER — 형태+아웃라인+스태거]
   1240   86                              ▲1240    ▼86
      9999(겹침)8888                        ★9999
   MISS                                       ★8888  ← +90ms 스태거
                                           ✗MISS
  └ 흰 1240 이 폭발 VFX 위 증발            └ 8방향 검정 외곽선 → 항상 분리
  └ 크리/약점 색맹 구분 불가              └ ★/▲/▼ 형태로 색 없이 판별
```

### 5.2 상태이상 트레이 (파티 슬롯)

```
[BEFORE 12px 색점]          [AFTER 18px 칩 + 글리프]
 ● ● ● ●                    ┌──┐┌──┐┌──┐┌──┐
 (무슨 효과인지 색만)        │✦②││☠③││↓ ││⛨ │  +2
                            └━━┘└┄┄┘└──┘└══┘
                            기절 독  감속 보호
                            ↑control(굵은빨강) ↑dot(주황점선) ↑buff(금이중)
```
