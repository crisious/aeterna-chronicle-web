## 0. 한 문단 요약

에테르나 크로니클은 FF6식 ATB 전투의 핵심 피드백(데미지 숫자·상태이상 아이콘)을 이미 렌더하고 있으나, **표시가 작고(8~12px), 색에만 의존하며, SSOT 토큰과 어긋나** 정보가 한눈에 들어오지 않사옵니다. 전투 토큰(`battle-tokens.ts`)에는 데미지 7종 색·7종 폰트 크기와 약점/저항/면역 구분이 이미 정의돼 있으나 **DoT·상태이상 렌더 경로가 이를 쓰지 않고 하드코딩**하여, 1차 SSOT(`DESIGN.md`)가 약속한 *14px 봉인·색약 병행 표기·WCAG AAA*를 전투 화면이 위반하고 있사옵니다. 본 PRD는 신규 색·신규 폰트를 만들지 아니하옵고, **이미 있는 SSOT 토큰을 단일 헬퍼로 묶어 렌더 경로에 배선**하고 **색 외 형태/아이콘/텍스트 단서를 병행**시키는 데 집중하옵니다. 사용자는 대표(crisi) 본인 1인이며, 색약·소화면 가정을 1급 제약으로 둡니다. Build 단계 **3~4일 내** 완료를 지향하옵나이다.

## 1. 배경 & 문제 정의

### 1.1 현재 상태 (As-Is)

| 영역 | 현황 | 미해결 |
|------|------|--------|
| 데미지 팝업 | 일반/크리/힐 색 구분 + 상승 애니메이션 동작 | 약점/저항/면역(weak/resist/immune) **시각 미반영**, 색-단독, SSOT 폰트 미사용 |
| 상태이상 아이콘 | 최대 4종·라벨·지속바·중첩 표시 동작 | 라벨 `8px` 판독 난해, 색약 팔레트 미연동, 정보 과밀 |
| DoT 숫자 | 효과별 색 팝업·상승·페이드 동작 | `12px` 하드코딩, 동시 발생 시 `Math.random()` 위치로 **숫자 겹침** |
| 토큰 정합 | `BATTLE_COLORS.damage`·`DAMAGE_POPUP_SIZE` 7종 SSOT 존재 | 렌더 경로 **미배선** — 1차 SSOT와 코드 괴리 |
| 접근성 | 색약 모듈(`accessibility/colorblind/`)·모션 감소 자산 존재 | 전투 데미지/상태이상에 **미적용** |

### 1.2 문제 (다섯 매듭 — 두련사 진단과 정합)

1. **SSOT 이중화 / 14px 봉인 위반** — `DAMAGE_POPUP_SIZE`(normal 16·crit 32…)와 `BATTLE_COLORS.damage`가 정본인데, 렌더는 `12px`·`8px`을 하드코딩. `DESIGN.md` 14px 봉인을 전투 텍스트가 깨뜨림.
2. **색-단독 표기 (WCAG AAA 위반)** — 데미지/힐, 독/출혈/화상이 *색으로만* 구분. 적록 색약(deuteranopia)에서 독(녹)·출혈(적)·화상(주황) 혼동, 힐(녹)·데미지 혼동.
3. **약점/저항/면역 피드백 부재** — 전투 의사결정의 핵심 신호(weak/resist/immune)가 SSOT엔 있으나 화면에 안 나타나 플레이어가 속성 상성을 학습할 단서가 없음.
4. **정보 과밀 / 소화면 판독 실패** — 캐릭터 위 `-60px` 좁은 띠에 아이콘 20px·라벨 8px·지속바 3px·중첩 ×N이 빽빽. 효과 다수 시 무엇이 무엇인지 분간 불가.
5. **동시 피드백 충돌 / 모션 비가드** — 동시 DoT가 `Math.random()` 분산만으로 겹침. 화상 깜박임(200ms)이 `prefers-reduced-motion`·모션 감소 설정과 무관하게 항상 점멸.

### 1.3 가치 가설

표시를 SSOT에 정렬하고 색 외 단서를 병행하면:
- 전투 정보 1회 식별 시간 단축 → "방금 약점이었나/저항이었나" 되묻는 인지부하 감소
- 색약·소화면에서도 동일 정보 도달 → `DESIGN.md` AAA 원칙 실현, 회귀 게이트로 영구 보존
- 신규 색/폰트 0개 추가 → 토큰 시스템 단일성 유지, 디자인 부채 미발생

## 2. 목표 & 비목표

### 2.1 목표 (Goals)
- **G1** 전투 데미지·상태이상 텍스트를 `battle-tokens.ts` SSOT(폰트·색)에 100% 정렬, 하드코딩 폰트 0건.
- **G2** 데미지/힐 및 상태이상 카테고리를 **색 + (형태 | 아이콘 | 부호/텍스트)** 2종 이상 단서로 표기 (WCAG AAA 색-비의존).
- **G3** 약점/저항/면역(weak/resist/immune)을 데미지 팝업에 시각·텍스트로 노출.
- **G4** 동시 다발 피드백(DoT·복수 상태이상)의 겹침 제거 — 결정적(deterministic) 배치.
- **G5** 모션·점멸이 모션 감소 설정을 존중하도록 가드.

### 2.2 비목표 (Non-Goals)
- 전투 로직/밸런스/데미지 공식 변경 (표시 계층 전용).
- 신규 색 팔레트·신규 폰트·신규 상태이상 종류 추가.
- 서버 전송 데이터 스키마 변경 (`StatusEffectData`/`TickDamageEvent` 유지, 표시 메타만 클라 산정).
- 모바일 신규 레이아웃 설계 (소화면 가독은 폰트·간격 토큰으로 흡수, 별도 반응형 작업 아님).
- 사운드/햅틱 피드백 (별도 토픽).

## 5. 사용자 스토리 & 수용 기준

### US-BF-01 — 약점/저항 한눈 식별 (P-주)
> 전투 중 플레이어로서, 내 공격이 **약점을 찔렀는지·저항됐는지·면역인지**를 숫자 색에만 의존하지 않고 즉시 알고 싶다. 그래야 속성 상성을 학습하고 다음 행동을 정한다.

**수용 기준** (Given–When–Then):
- Given 적이 화염 약점일 때, When 화염 데미지가 들어가면, Then 팝업은 `weak` 색(0xff6b35) + **"WEAK!" 라벨** + **▲ 아이콘** + 크기 22px(`DAMAGE_POPUP_SIZE.weak`)로 표시된다.
- Given 적이 저항할 때, Then `resist` 색(0x3498db) + **"RESIST" 라벨** + **▽** + 14px로 표시된다.
- Given 면역일 때, Then `immune` 색 + **"IMMUNE" 라벨**로, 숫자 대신 텍스트가 우선된다.
- 위 단서는 색약 시뮬 4종에서 모두 식별 가능(라벨·형태가 색과 독립).

### US-BF-02 — 데미지/힐 색약 구분 (P-색약)
> 적록 색약 플레이어로서, **힐과 데미지를 색 없이도** 구분하고 싶다.

**수용 기준**:
- 힐은 부호 **`+`** 접두 + 상승 방향, 데미지는 부호 없음(또는 `−`) — 부호로 1차 구분.
- 크리티컬은 색(0xffd700) + **크기 32px** + 텍스트 강조(굵게/외곽선) 2종 이상.
- `showDotDamage`의 힐 판정이 SSOT `BATTLE_COLORS.damage.heal`을 참조(하드코딩 `#44ff44` 제거).

### US-BF-03 — 상태이상 카테고리 식별 (P-색약, P-소화면)
> 플레이어로서, 캐릭터에 걸린 효과가 **이로운지(버프)·해로운지(디버프)·행동불가(제어)·지속피해(DoT)**인지 작은 화면에서도 알고 싶다.

**수용 기준**:
- 5카테고리(buff/debuff/control/dot/special)가 **테두리 색 + 형태(모서리/아이콘 글리프) + 한글 라벨** 2종 이상으로 구분.
- 라벨 폰트 ≥ 14px(또는 토큰이 정한 `statusLabel` 최소값) — `8px` 폐기.
- 효과 4개 초과 시 "+N" 요약 배지로 과밀 방지(아이콘 무한 증식 금지).

### US-BF-04 — 동시 피드백 비충돌 (P-주)
> 플레이어로서, 광역기로 **여러 적에게 동시에 DoT**가 터질 때 숫자가 겹쳐 안 보이는 일이 없길 바란다.

**수용 기준**:
- Given 한 대상에 3개 DoT가 같은 프레임에 발생, Then 팝업은 `Math.random()`이 아닌 **결정적 스택 오프셋**으로 세로 정렬, 상호 간격 ≥ 토큰 `minGap`.
- 팝업 풀 상한 초과 시 가장 오래된 것부터 회수(상한 명시).

### US-BF-05 — 모션 감소 존중 (P-색약/광과민 가정)
> 광과민 가능 사용자로서, **화상 점멸·카메라 흔들림**이 모션 감소 설정을 따르길 바란다.

**수용 기준**:
- `prefers-reduced-motion` 또는 게임 내 모션 감소 ON 시, 화상 깜박임은 **정적 오버레이**로, 크리 카메라 쉐이크는 **억제**.
- 정보(어떤 효과인지)는 모션 없이도 100% 보존.
