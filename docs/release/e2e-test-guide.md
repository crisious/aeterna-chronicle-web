## 1. 테스트 철학

| 원칙 | 의미 |
|------|------|
| **회귀 우선** | 새 기능보다 기존 기능이 깨지지 않음을 먼저 보장한다 |
| **단일 진실** | 테스트 시나리오의 SSOT는 본 문서, 실패는 `cross-browser-issues.md`에 누적한다 |
| **빠른 피드백** | 단위 < 5초, 통합 < 30초, E2E < 2분을 목표로 한다 |
| **결정성** | 시간·랜덤·네트워크 의존을 모두 모킹하여 플레이키 테스트를 0으로 유지한다 |

> *흩어진 악보를 한데 모아 곡조를 맞추듯, 흩어진 검증의 흔적을 본 문서로 합치겠사옵니다.*

## 3. 범위 정의

### 3.1 UI 시스템
- `client/src/ui/**` (메인 메뉴, HUD, 옵션, 다이얼로그)
- `client/src/combat/**` (ATB 게이지, 커맨드 메뉴, 상태이상 트레이)
- 키보드·마우스·게임패드 입력 매트릭스

### 3.2 인벤토리 시스템
- 아이템 획득 → 정렬 → 사용 → 폐기 4단계 라이프사이클
- 100종 아이템 데이터 무결성 (`item_data_table.md`)
- 신화 6세트 옵션 합산 검증
- 무게/슬롯 한계, 중첩 가능 아이템 합산

### 3.3 세이브 시스템
- 슬롯 3개 + 자동저장 1개 (총 4슬롯)
- IndexedDB 정상 / Safari Private 차단 / ITP 7일 만료 분기
- 버전 마이그레이션 (rc.2 → rc.3 세이브 호환)
- 손상 세이브 감지·복구·백업 폴백
