## 1. 디자인 원칙 (몬스터 전용 5계명)

| # | 원칙 | 한 줄 정의 | 실천 |
|---|------|-----------|------|
| 1 | **위계는 광원으로** | 사이즈 ±16px만으론 약하다. 림라이트가 주연이다. | 일반 = 없음 / 엘리트 = 1px 골드 / 보스 = 2px 에테르블루 |
| 2 | **실루엣 우선** | 16x16 축소해도 카테고리 식별. 색은 두 번째. | 5종 카테고리 강제 (인간형·짐승형·곤충형·기계형·원소형) |
| 3 | **지역은 팔레트로** | 챕터 = 색 / 등급 = 광원. 두 축이 직교. | 지역 팔레트 7세트 + Tier 광원이 곱해짐 |
| 4 | **AI는 컨셉, 사람이 픽셀** | 양자화 + 외곽선 재작업이 라이선스 방패 | 픽셀 차이율 ≥ 60% (게이트 ⑤ pixel_diff) |
| 5 | **호흡으로 말한다** | idle 모션은 캐릭터의 인격이다 | ±1/±2/±3px breathing — Tier별 진폭 차등 |

> 가춘운의 한마디: **"크기로 위협하면 단조롭고, 광원으로 위협하면 시(詩)가 된다."** ✨

## 2. Tier 토큰 5축 — DESIGN.md §10 신설

### 2.1 토큰 정의 (TypeScript SSOT)

```typescript
// src/design-tokens/monster_tier.ts
export const MONSTER_TIER = {
  normal: {
    size: { w: 32, h: 32 },                  // 1.0x base
    outline: { width: 2, color: '#000000' }, // pure black, no gradient
    rimLight: null,                          // 없음
    palette: { max: 16, file: 'normal.aseprite-pal' },
    idle: { frames: 4, amplitudePx: 1, fps: 6 },
    particles: { count: 0 },
    introMs: 0,                              // 진입 컷씬 없음
  },
  elite: {
    size: { w: 48, h: 48 },                  // 1.5x
    outline: { width: 2, color: '#000000', highlight: { width: 1, color: '#FFD700' } },
    rimLight: { width: 1, color: '#FFD700', position: 'top-left' },
    palette: { max: 24, file: 'elite.aseprite-pal' },
    idle: { frames: 6, amplitudePx: 2, fps: 8 },
    particles: { count: [2, 3], color: '#FFD700', spawnRate: 'subtle' },
    introMs: 800,                            // HP바 페이드인만
  },
  boss: {
    size: { w: 64, h: 64, asymmetricAllowed: true },  // 2x+, 비대칭 허용
    outline: { width: 2, color: '#000000', highlight: { width: 2, gradient: ['#89CFF0', '#FFD700'] } },
    rimLight: { width: 2, color: '#89CFF0', position: 'full-silhouette' },
    palette: { max: 32, file: 'boss.aseprite-pal' },
    idle: { frames: 8, amplitudePx: 3, fps: 10 },
    particles: { count: [5, 8], color: '#89CFF0', spawnRate: 'ambient', screenAmbient: true },
    introMs: 3000,                           // 진입 시 화면 진동 + 컬러 시프트 + 이름 타이포
  },
} as const;

export type MonsterTier = keyof typeof MONSTER_TIER;
```

### 2.2 시각 위계 미리보기 (ASCII 모식도)

```
NORMAL (32px)         ELITE (48px)              BOSS (64px+)
┌─────────┐           ┌──────────────┐          ┌──────────────────────┐
│  ▓▓▓    │           │   ✨▓▓▓▓     │          │  ▓▓▓▓▓▓▓▓▓✨✨✨▓▓▓ │
│  ▓██▓   │           │   ▓████▓     │          │  ▓████████████▓     │
│  ▓██▓   │           │ ✨▓████▓     │          │ ▓██████████████▓✨ │
│  ▓▓▓    │           │   ▓▓▓▓       │          │  ▓▓▓▓▓▓▓▓▓▓▓▓▓     │
│         │           │      ✨      │          │   ✨ idle particles ✨│
└─────────┘           └──────────────┘          └──────────────────────┘
outline 2px black     + 1px gold highlight       + 2px ether-blue rim
no rim                rim top-left only          full silhouette glow
no particles          2-3 idle particles         5-8 + screen ambient
breath ±1px / 4f      breath ±2px / 6f           breath ±3px / 8f
```

## 3. 컬러 시스템 — 발광 3색 + 지역 7팔레트

### 3.1 Tier 발광 3색 (SSOT)

```css
/* DESIGN.md §10.1 — Tier Glow Tokens */
--monster-glow-normal: transparent;        /* 발광 없음 */
--monster-glow-elite:  #FFD700;            /* 골드 (보석/소중함) */
--monster-glow-boss:   #89CFF0;            /* 에테르 블루 (위협/신성) */

/* 보스 페이즈 전환 시 컬러 시프트 (페이즈 2 진입) */
--monster-glow-boss-phase2: #FF4444;       /* 진홍 (분노) */
```

> **왜 골드 → 에테르블루인가?** 골드는 "귀하다 (loot drop 기대)", 에테르블루는 "이 세계의 본질 (위협)". 보스는 단순 강함이 아니라 **세계관의 무게**를 표현. 💙✨

### 3.2 지역 팔레트 7세트 (챕터 직교)

| 지역 | 챕터 | 베이스 | 보조 | 한정 색 (16색 중 4색) | 톤 |
|------|------|--------|------|---------------------|-----|
| 🌑 에레보스 | Ch.1 | `#2A2A3A` | `#4A4A5A` | `#6B5B95` `#8B7AA8` `#5C4B7E` `#3D3050` | 잿빛/보랏빛 — 망각 |
| 🌳 실반헤임 | Ch.2 | `#2D4A2D` | `#4A6B3A` | `#7BA05B` `#8BC34A` `#558B2F` `#33691E` | 청록/연두 — 기억의 숲 |
| 🏜️ 솔라리스 | Ch.3 | `#8B4513` | `#CD853F` | `#FF8C00` `#FFA500` `#D2691E` `#A0522D` | 호박/주황 — 불꽃 |
| 🏰 아르겐티움 | Ch.4 | `#B0B0B0` | `#7B68EE` | `#C0C0C0` `#9370DB` `#4B0082` `#191970` | 은/심청 — 제국 |
| 🏔️ 영원빙원 | Ch.4 | `#B0E0E6` | `#5F9EA0` | `#E0FFFF` `#AFEEEE` `#48D1CC` `#008B8B` | 빙청 — 얼어붙은 기억 |
| ⚓ 브리탈리아 | Ch.3~4 | `#4682B4` | `#5F9EA0` | `#87CEEB` `#B0C4DE` `#2F4F4F` `#708090` | 해청 — 자유항 |
| 💀 망각의 고원 | Ch.5 | `#1A1A2E` | `#16213E` | `#FF4444` `#89CFF0` `#FFD700` `#0D0D1A` | 다크 + 3색 액센트 — 최종 결전 |

> **MVP 60종 = 에레보스(Ch.1) 30종 + 실반헤임(Ch.2) 30종**. 두 팔레트만 먼저 양자화 파일로 구워두면 됩니다! 💪

### 3.3 팔레트 곱 — Tier × 지역

```
일반 몬스터  = 지역 팔레트 16색 그대로
엘리트     = 지역 16색 + 골드 발광 4색 + 그림자 4색 = 24색
보스       = 지역 16색 + 에테르블루 8색 + 페이즈2 진홍 8색 = 32색
```

이 구조 덕에 **챕터 정체성 + 등급 위계**가 한눈에 직교!

## 5. 타이포그래피 — 몬스터명·HP바 위계

### 5.1 몬스터명 타이포 4티어

| Tier | 폰트 | 크기 | 색상 | 효과 | 사용처 |
|------|------|------|------|------|-------|

| 엘리트 | `Galmuri11` | 13px | `#FFD700` | 1px shadow `#000` | HP바 위 + 도감 카드 |
| 보스 진입 | `BookkMyungjo` | 32px | `#FFD700` → `#89CFF0` 그라디언트 | 페이드인 1.2s + 글로우 | 보스 인트로 컷씬 (3s) |
| 보스 전투 | `Galmuri11` | 16px | `#89CFF0` | 2px shadow + idle pulse | HP바 위 (전투 중) |

### 5.2 HP바 위계

```
일반:    [▰▰▰▰▰▰▱▱▱▱]  단일 진홍 #FF4444 · 높이 4px · 외곽 1px
엘리트:  [▰▰▰▰▰▰▱▱▱▱]  진홍 + 골드 외곽 2px · 높이 6px
보스:    [▰▰▰▰▰▰▰▱▱▱]  화면 상단 풀폭 · 높이 12px · 페이즈 분할 표시
         페이즈1 진홍 / 페이즈2 진홍→보라 / 페이즈3 보라→흑
```

- 카테고리 라벨 (5종): `Galmuri11` 9px `#606060` 좌상단

## 6. 레이아웃 — 전투·도감·보스 인트로

### 6.1 전투 화면 몬스터 배치 (1280x720 기준)

```
┌──────────────────────────────────────────────────┐ y=0
│  보스 HP바 풀폭 (보스 전용, h=48 + padding)        │ y=0~80 (보스만)
├──────────────────────────────────────────────────┤
│                                                  │
│         [몬스터 슬롯 1]    [몬스터 슬롯 2]         │ y=200~440
│           (일반/엘리트)      (일반/엘리트)          │
│                                                  │
│              [몬스터 슬롯 3]                       │ y=300 (옵션)
│                                                  │
│         [BOSS 슬롯] (단독, 비대칭 허용)            │ y=180~480
│                                                  │
├──────────────────────────────────────────────────┤
│   ATB 게이지 + 명령 메뉴                           │ y=560~720
└──────────────────────────────────────────────────┘
```

- 일반/엘리트 슬롯: 192x192 박스 안에 sprite 중앙 정렬
- 보스 슬롯: 384x384 박스, 비대칭 허용 (박스 밖 ±32px)
- HP바: sprite 상단 -16px (일반/엘리트), 화면 상단 풀폭 (보스)

### 6.2 도감 UI 그리드 (160종 → MVP 60종)

```
┌─────────────────────────────────────────────────────┐
│ [전체] [인간형] [짐승형] [곤충형] [기계형] [원소형]      │ 5종 카테고리 필터
│ [Ch.1] [Ch.2] [Ch.3] ...                            │ 챕터 필터
├─────────────────────────────────────────────────────┤
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐                │
│  │ N │ │ N │ │ E │ │ N │ │ N │ │ E │                │ 일반 6/줄 (셀 96x96)
│  └───┘ └───┘ └───┘ └───┘ └───┘ └───┘                │ 엘리트 = 골드 테두리
│  ┌───┐ ┌───┐ ┌───┐ ...                              │
│  └───┘ └───┘ └───┘                                  │
├─────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────┐   │
│  │  BOSS  [풀폭 카드 768x192 + 시네마틱 배경]    │   │ 보스 = 풀폭 1/줄
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

> **PRD 결정**: 도감 UI는 **본 스프린트 분리 (별도 P3 2주)**. 본 디자인 시스템에 그리드 SSOT만 남기고, 카드 컴포넌트 구현은 후속 스프린트로 인계. ✅

### 6.3 보스 인트로 시퀀스 3-5초 ✨

```
T=0.0s  화면 진동 시작 (amplitude 4px, freq 12Hz)
T=0.2s  배경 페이드 to #0D0D1A (alpha 0 → 0.7)
T=0.5s  보스 sprite 페이드인 + 등장 (scale 0.8 → 1.0)
T=1.0s  rim light glow 가동 (#89CFF0 pulse)
T=1.5s  보스 이름 타이포 페이드인 (32px 그라디언트)
T=2.5s  HP바 좌→우 슬라이드인 (full width, 0.8s)
T=3.3s  화면 진동 종료, 배경 alpha 0.7 → 0.0 (전투 가능)
T=3.5s  플레이어 입력 활성화
```

→ Build 단계에서 계섬월에게 **`<BossIntroSequence boss-id="..." />`** 컴포넌트 신설 요청. Phaser Tween 체이닝으로 구현 가능.

## 7. AI→터치업 비주얼 체크리스트

### 7.1 5단계 게이트 (각 단계 통과해야 다음으로)

```
┌──────────────────────────────────────────────────────────┐
│ ① 컨셉 탐색 (AI 생성)                                       │
│   ├─ Adobe Firefly (1순위, 라이선스 안전)                     │
│   ├─ SDXL + Pixel Art LoRA (2순위, 출처 추적 강제)            │
│   └─ 4종 후보 / 1024x1024 / 5분                              │
├──────────────────────────────────────────────────────────┤
│ ② 후보 선정 + 실루엣 검증                                    │
│   ✅ 16x16 축소 시 5카테고리 식별 가능?                        │
│   ✅ Tier 토큰 일치? (림라이트 위치, 비대칭 정도)               │
│   ❌ 작가 스타일 흔적? (ex: "Amano 풍") → 즉시 폐기            │
├──────────────────────────────────────────────────────────┤
│ ③ 다운샘플 + 팔레트 양자화 (필수)                              │
│   nearest-neighbor → Tier별 사이즈 (32/48/64)                │
│   팔레트 강제: 지역 .aseprite-pal 적용 (16/24/32색)           │
│   잔여 antialiasing 픽셀 = 0                                 │
├──────────────────────────────────────────────────────────┤
│ ④ 수동 픽셀 터치업 (라이선스 안전 핵심)                        │
│   외곽선 2px 재작업 (Tier 토큰 outline.width 준수)            │
│   하이라이트/림라이트 수동 재배치                              │
│   픽셀 차이율 ≥ 60% (게이트 ⑤ pixel_diff)                    │
│   소요: 일반 30분 / 엘리트 60분 / 보스 2-3시간                 │
├──────────────────────────────────────────────────────────┤
│ ⑤ 애니메이션 프레임 분기                                      │
│   idle / attack / hit / death 슬라이싱 (Tier별 프레임 수)      │
│   보스 = 페이즈 전환 모션 추가 (3 페이즈 권장)                  │
└──────────────────────────────────────────────────────────┘
```

### 7.2 비주얼 합격/반려 기준 표

| 체크 | 합격 | 반려 |
|------|------|------|
| 외곽선 | 2px 일정 검정 | 1.5px 흐림 / 부분 끊김 |
| 팔레트 | 지정 16/24/32색만 | 1픽셀이라도 외부 색 |
| 안티앨리어싱 | 0픽셀 | 1픽셀이라도 잔존 |
| 림라이트 | Tier 위치 정확 | 위치 어긋남 / 사이즈 차이 |
| 실루엣 | 16x16 카테고리 식별 | 5초 내 식별 불가 |
| 픽셀 차이율 | AI 원본 대비 ≥ 60% | < 60% (사람 손길 부족) |

## 8. 접근성 — 색맹 4모드 호환

> 진채봉/이소화 §2.17 SSOT와 정합. 몬스터 비주얼도 4모드 전부 PASS여야 도감 등재. 🌈

### 8.1 색상 외 형태 단서 병행

| 정보 | 색상 단서 | 형태 단서 (필수) |
|------|----------|----------------|
| 등급 (Tier) | 발광 색 (gold/blue) | **외곽선 두께·하이라이트** |
| 페이즈 (Boss) | 진홍 → 보라 | **HP바 분할선** |
| 카테고리 | 지역 팔레트 | **실루엣 형상** (5종) |
| 상태 (poisoned/burning) | 컬러 오버레이 | **상태 아이콘 12종 SSOT 재사용** |

### 8.2 색맹 4모드 검증

```yaml
colorblind_audit:
  modes: [protanopia, deuteranopia, tritanopia, achromatopsia]
  per_monster:
    - rim light 식별 가능? (form check)
    - HP바 색 구분 가능? (or 분할선 의존)
    - 카테고리 식별 가능? (silhouette만으로 충분?)
  pass: 4모드 전부 식별 가능
  fail: 형태 단서 보강 후 재테스트
```

> 어머~ 이거 다행히 **형태 단서가 메인**이라 색맹 4모드 자동 PASS 가능성 높아요! 광원 위계가 두께 차이도 같이 가져가니까~ ✨
