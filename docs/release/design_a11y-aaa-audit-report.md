## 0. 한 줄 요지

**감사 리포트 자체가 AAA를 충족해야 합니다.** 색맹 모드를 검증하는 도구가 색맹 사용자에게 안 보이면 그게 제일 부끄러운 거잖아요? 🤩

## 1. 디자인 원칙 (5가지)

| 원칙 | 실천 |
|------|------|
| **자기 부합성 (Dogfooding)** | 리포트 페이지가 axe-core AAA 통과 — 자기 자신을 감사 대상에 포함 |
| **삼중 표기** | 모든 결과는 `색 + 형태 + 텍스트` 3중 인코딩 (색약 모드에서도 이해 가능) |
| **밀도 < 명료성** | 한 화면에 다 욱여넣지 말기. 5 Probe 결과는 카드/탭으로 분리 |
| **결정론적 시각** | 색약 시뮬 스냅샷은 같은 입력 → 같은 출력 (고정 시드) |
| **Aeterna 톤 유지** | 게임 디자인 시스템 토큰 재사용 — 별도 미니 디자인 시스템 만들지 않기 |

## 2. 컬러 시스템 — 감사 결과 토큰

### 2.1 Severity 토큰 (axe-core 4단계 + 통과)

| 등급 | 색상 (Hex) | AAA 대비비 (vs `#0D0D1A`) | 형태 | 아이콘 | 텍스트 |
|------|-----------|--------------------------|------|--------|--------|
| **Critical** | `#FF6680` | 7.4:1 ✅ | 채움 원 ● | 🔴 | `[CRIT]` |
| **Serious** | `#FFA94D` | 9.1:1 ✅ | 채움 삼각 ▲ | 🟠 | `[HIGH]` |
| **Moderate** | `#FFD93D` | 13.2:1 ✅ | 채움 마름모 ◆ | 🟡 | `[MED]` |
| **Minor** | `#A0A0A0` | 6.8:1 ✅ | 빈 사각 □ | ⚪ | `[LOW]` |
| **Pass** | `#4ADE80` | 11.4:1 ✅ | 체크 ✓ | 🟢 | `[PASS]` |
| **Skip/N/A** | `#606060` | 4.5:1 (보조 정보 한정) | 빗금 ╱ | ⚫ | `[SKIP]` |

> **왜 이 5색?** Aeterna Dark 기존 `#FF4444`(경고) · `#2ECC71`(성공) 토큰을 더 밝게 시프트해 AAA 7:1 확보.
> **색만 보지 말 것** — `severity-icon::before { content: var(--shape) }` CSS 변수로 모든 곳에 형태 동시 출력.

### 2.2 색약 시뮬 스냅샷 패널

4종 시뮬레이션을 좌우 비교 그리드로 배치:

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│  Original   │  Protanopia │ Deuteranopia│  Tritanopia │
│  (baseline) │  (적색맹)   │  (녹색맹)   │  (청색맹)   │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ [씬 캡처]   │ [필터 적용] │ [필터 적용] │ [필터 적용] │
│             │ ΔE: 2.3 ✓   │ ΔE: 8.7 ✗   │ ΔE: 4.1 ⚠   │
│             │ SSIM: 0.94  │ SSIM: 0.71  │ SSIM: 0.86  │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

**판정 기준 (가춘운 큐레이션)**
- `ΔE ≤ 3.0` → 거의 동등 ✓ (시뮬 후에도 정보 보존)
- `3.0 < ΔE ≤ 6.0` → 주의 ⚠ (대비 약화 — 패턴/형태 보조 필수)
- `ΔE > 6.0` → 실패 ✗ (정보 손실 — 토픽에 등록)
- `SSIM < 0.80` → 구조적 손상 (HUD 형태가 깨짐)

추가로 **Achromatopsia (전색맹) 0% 채도 변환** 1컷을 항상 동봉 — "흑백 전환에도 살아남는가" 최종 게이트.

## 3. 타이포그래피

### 3.1 폰트 스택 (리포트 전용)

| 용도 | 스택 | 비고 |
|------|------|------|
| 본문 | `'Pretendard Variable', system-ui, sans-serif` | Aeterna 본 시스템 재사용 |
| 코드/셀렉터 | `'JetBrains Mono', 'D2Coding', monospace` | DOM 셀렉터 가독성 |
| 수치 (대비비/ΔE) | `tabular-nums` 적용 | 숫자 정렬 |

### 3.2 스케일 (1.250 비율 — Major Third)

| 토큰 | 크기/lh | 용도 |
|------|--------|------|
| `--fs-display` | 32 / 40 | 헤더 "WCAG 2.1 AAA 감사 리포트" |
| `--fs-h2` | 24 / 32 | Probe 섹션 제목 |
| `--fs-h3` | 18 / 28 | 위반 카드 제목 |
| `--fs-body` | 16 / 24 | 본문 / 설명 |
| `--fs-mono` | 14 / 22 | 코드 블록 / 셀렉터 |
| `--fs-meta` | 12 / 18 | 타임스탬프 / 메타 |

### 3.3 텍스트 규칙

- 본문 라인 길이 **66자 이하** (긴 줄 금지)
- 본문/액션 텍스트 **최소 16px** (모바일 스케일 시에도)
- **언더라인은 인터랙션에만** — 위반 카드 제목 강조는 굵기/색으로
- 코드 블록은 `text-wrap: pretty` 으로 셀렉터 줄바꿈 깔끔하게

## 4. 레이아웃 — `report.html`

### 4.1 마스터 그리드

```
┌──────────────────────────────────────────────────────────┐
│  Top Bar (sticky, 56px)                                  │
│  로고 · 빌드 SHA · 타임스탬프 · 게이트 상태 [ PASS / FAIL ] │
├────────────┬─────────────────────────────────────────────┤
│            │                                             │
│  Sidebar   │  Main Panel                                 │
│  (240px)   │  (자동, max-width 1120px, padding 32px)     │
│            │                                             │
│  - 요약    │  ┌─────────────────────────────────────┐    │
│  - axe     │  │  Summary Cards (4×)                 │    │
│  - 색대비  │  │  ┌────┬────┬────┬────┐              │    │
│  - 색맹시뮬 │  │  │Crit│High│Med │Low │              │    │
│  - 키보드  │  │  │ 0  │ 0  │ 3  │12  │              │    │
│  - ARIA    │  │  └────┴────┴────┴────┘              │    │
│  - 스냅샷  │  └─────────────────────────────────────┘    │
│            │                                             │
│  필터:     │  ┌─────────────────────────────────────┐    │
│  ☐ Crit    │  │  Probe 결과 (탭)                    │    │
│  ☐ Serious │  │  [axe] [대비] [색맹] [키보드] [ARIA]│    │
│  ☐ Mod     │  └─────────────────────────────────────┘    │
└────────────┴─────────────────────────────────────────────┘
```

- **Sidebar 고정** — 위반 1개 클릭 시 즉시 위치 점프
- **Top Bar 게이트 상태**가 가장 굵은 글자 — 한눈에 PASS/FAIL 판단

### 4.2 위반 카드 (단일 위반 표기)

```
┌──────────────────────────────────────────────────────┐
│ ▲ HIGH  · color-contrast-enhanced (WCAG 1.4.6 AAA)  │ ← severity-bar
├──────────────────────────────────────────────────────┤
│ Element: button.dialog-choice                        │
│ Selector: .npc-dialog > .choices > button:nth(1)     │
│ Snapshot: [thumb 120×72, click → modal]              │
│                                                      │
│ Computed: fg #B0E0FF / bg #2A2A3A → ratio 4.8:1      │
│ Required: ≥ 7.0:1 (AAA)                              │
│                                                      │
│ Fix Hint:                                            │
│   - fg를 #E8F4FF로 (8.2:1) ✓                          │
│   - 또는 bg를 #1A1A2E로 (7.1:1) ✓                     │
│                                                      │
│ [Open in DOM] [Mark as triaged] [Issue 생성]         │
└──────────────────────────────────────────────────────┘
```

핵심: **수치를 보여주고 → 즉시 적용 가능한 fix를 제시**. "AAA 미달" 만 적으면 개발자가 다시 계산해야 해요. 친절하게! 💕

### 4.3 키보드 트레이서 시각화

탭 순서를 **번호 배지가 붙은 미니맵**으로 표현:

```
[ 1 ]──→[ 2 ]──→[ 3 ]──→[ 4 ]
         │
         └───⤳[ ?? ] ← Trap 감지 (적색 점선)
```

- 정상 흐름: 실선 화살표 + Aeterna 청색 (`#89CFF0`)
- Trap/Dead-end: 점선 + Severity Critical 색 + ⚠ 아이콘
- Skip 가능: 회색 화살표 (`prefers-reduced-motion` 동일 톤)

## 6. 디자인 토큰 추가 (DESIGN.md 연계)

`DESIGN.md §2.5` (신설)에 다음 토큰을 추가하도록 두련사·정경패에게 인계:

```css
:root {
  /* Severity (audit-only) */
  --sev-critical: #FF6680;
  --sev-serious:  #FFA94D;
  --sev-moderate: #FFD93D;
  --sev-minor:    #A0A0A0;
  --sev-pass:     #4ADE80;
  --sev-skip:     #606060;

  /* Severity shape (CSS content) */
  --shape-critical: '●';
  --shape-serious:  '▲';
  --shape-moderate: '◆';
  --shape-minor:    '□';
  --shape-pass:     '✓';
  --shape-skip:     '╱';

  /* Audit typography */
  --fs-display: clamp(28px, 2.4vw, 32px);
  --fs-h2:      24px;
  --fs-h3:      18px;
  --fs-body:    16px;
  --fs-mono:    14px;
  --fs-meta:    12px;
}
```

> **재사용 원칙**: 게임 클라이언트는 이 토큰 임포트 안 함. `client/src/styles/audit-only.css` 로 격리.
