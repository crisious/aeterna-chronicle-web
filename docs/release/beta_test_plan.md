## 1. 베타 테스트 개요

| 항목 | CBT (비공개 베타) | OBT (공개 베타) |
|------|------------------|----------------|
| **목적** | 핵심 게임플레이 검증 + 크리티컬 버그 제거 | 서버 부하 테스트 + 최종 밸런스 + 사용자 경험 |
| **플랫폼** | 웹 브라우저 (전용 CBT URL) | Steam Playtest |
| **인원** | 500명 (초대제) | 무제한 |
| **기간** | 2주 (D-35 ~ D-21) | 1주 (D-14 ~ D-7) |
| **NDA** | ✅ 필수 (스크린샷/스트리밍 금지) | ❌ 없음 (공개 가능) |
| **콘텐츠 범위** | 챕터 1~3 (에레보스→실반헤임→솔라리스) | 챕터 1~4 + PvP 시즌 |
| **레벨 캡** | 30 | 50 |

## 2. CBT (비공개 베타) 상세

### 2.1 참가자 선정 기준

| 카테고리 | 인원 | 선정 기준 |
|----------|------|----------|
| RPG 하드코어 | 200명 | RPG 플레이 100h+ 경험, 피드백 의지 |
| 접근성 테스터 | 50명 | 시각/청각/운동 장애 경험자, 접근성 전문가 |
| 스트리머/미디어 | 50명 | 사전 리뷰용 (NDA 해제 후 스트리밍 가능) |
| 지역별 | 100명 | 한국 40 / 북미 30 / 일본 20 / 기타 10 |
| 랜덤 선발 | 100명 | Discord/Steam 위시리스트 기반 추첨 |

### 2.2 신청 양식

```
1. Discord ID:
2. 국가/지역:
3. 언어 (한/영/일):
4. RPG 플레이 경험 (타이틀 3개+):
5. 하드웨어 (OS/브라우저/RAM):
6. 접근성 요구사항 (해당 시):
7. 피드백 제공 가능 시간 (주당 h):
8. 왜 에테르나 크로니클에 관심을 가졌는가 (50자+):
```

### 2.3 CBT 일정

| 일차 | 활동 | 초점 |
|------|------|------|
| Day 1~3 | 온보딩 + 캐릭터 생성 + 챕터 1 | FTUE 메트릭, 튜토리얼 |
| Day 4~7 | 챕터 2 + 일반 던전 | 전투 밸런스, 클래스 체감 |
| Day 8~10 | 챕터 3 + 엘리트 던전 | 난이도 곡선, 보상 체감 |
| Day 11~12 | 자유 플레이 | 콘텐츠 소진 속도, 반복 플레이 |
| Day 13~14 | 최종 설문 + 피드백 정리 | 종합 만족도, NPS |

### 2.4 NDA 조항 (요약)

- 게임 화면, 텍스트, 시스템 정보의 공개 금지
- SNS/블로그/유튜브/트위치 등 모든 채널 포함
- NDA 위반 시: CBT 접근 즉시 차단, 향후 이벤트 영구 제외
- NDA 해제: CBT 종료 후 7일 (D-14) 또는 OBT 시작 시

## 3. OBT (공개 베타) 상세

### 3.1 Steam Playtest 활용

- Steam의 Playtest 기능으로 별도 앱 없이 참여 가능
- "Request Access" 버튼 → 즉시 승인 (무제한)
- 본편과 동일 스토어 페이지에서 접근 → 위시리스트 전환 극대화

### 3.2 서버 부하 테스트 시나리오

| 시나리오 | 동시 접속 목표 | 테스트 항목 |
|----------|--------------|-----------|
| 정상 부하 | 500 CCU | 응답 시간 <200ms, 프레임 드롭 0 |
| 피크 부하 | 2,000 CCU | 오토스케일링 트리거, DB 커넥션 풀 |
| 스파이크 | 5,000 CCU (5분간) | 서버 생존, 큐 시스템 동작 |
| 레이드 집중 | 50 레이드 동시 | 전투 동기화, 보상 원자성 |
| PvP 집중 | 200 매치 동시 | 매치메이킹 지연, 결과 정합성 |

### 3.3 OBT 일정

| 일차 | 활동 | 측정 항목 |
|------|------|----------|
| Day 1 | 출시 → 접속 폭주 예상 | 피크 CCU, 큐 대기 시간 |
| Day 2~3 | 메인 스토리 진행 | 이탈률, 챕터별 전환률 |
| Day 4~5 | 던전/PvP 오픈 | 매치메이킹, 밸런스 데이터 |
| Day 6 | 핫픽스 배포 (필요 시) | 패치 후 안정성 |
| Day 7 | OBT 종료 + 최종 메트릭 수집 | 전체 리포트 |

## 4. 피드백 수집 도구

### 4.1 수집 채널

| 채널 | 도구 | 용도 |
|------|------|------|
| 인게임 리포팅 | 커스텀 UI (단축키: F12) | 스크린샷 자동 첨부 + 카테고리 선택 |
| Discord | #beta-피드백 + #beta-버그 | 자유형 피드백 + 토론 |
| 설문조사 | Google Forms / Typeform | 구조화된 설문 (중간/최종) |
| 자동 수집 | Sentry (에러 트래킹) | 클라이언트 에러 자동 리포트 |
| 텔레메트리 | 커스텀 이벤트 로깅 | 플레이 패턴, 이탈 지점, 클래스 분포 |

### 4.2 버그 리포트 템플릿

```markdown
## 버그 리포트

**환경:**
- OS:
- 브라우저 + 버전:
- 해상도:
- 접속 시간:

**재현 단계:**
1.
2.
3.

**기대 결과:**
(정상 동작)

**실제 결과:**
(발생한 문제)

**심각도:** 🔴 크리티컬 / 🟡 메이저 / 🔵 마이너 / ⚪ 코스메틱

**스크린샷/영상:**
(첨부)
```

### 4.3 피드백 분류 체계

| 심각도 | 정의 | 대응 시간 |
|--------|------|----------|
| 🔴 크리티컬 | 게임 크래시, 데이터 손실, 진행 불가 | 4시간 내 |
| 🟡 메이저 | 기능 미작동, 심한 밸런스 이슈, UI 파괴 | 24시간 내 |
| 🔵 마이너 | 오타, 경미한 UI 깨짐, 비핵심 기능 이상 | OBT 후 패치 |
| ⚪ 코스메틱 | 시각적 사소함, "있으면 좋겠다" | 백로그 |

## 5. 성공 기준

### 5.1 CBT 성공 기준

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 크리티컬 버그 | 0건 (CBT 종료 시) | Sentry + 수동 리포트 |
| 메이저 버그 | <5건 | 분류 후 카운트 |
| 평균 플레이시간 | >2h/세션 | 텔레메트리 |
| DAU / 전체 참가자 | >60% | 로그인 로그 |
| NPS (Net Promoter Score) | >40 | 최종 설문 |
| 튜토리얼 완료율 | >90% | 텔레메트리 |
| 챕터 3 도달률 | >50% | 진행도 로그 |

### 5.2 OBT 성공 기준

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 피크 CCU 처리 | 2,000+ 안정 | 서버 메트릭 |
| 서버 업타임 | >99.5% (7일간) | 모니터링 |
| 크래시 비율 | <0.5% (세션 대비) | Sentry |
| 위시리스트 전환 | +2,000건 (OBT 기간) | Steamworks |
| 유저 리뷰 긍정률 | >80% | Discord/Steam |
| 평균 응답 시간 | <200ms | APM |
| 매치메이킹 대기 | <30초 (노멀 시간대) | PvP 로그 |

### 5.3 출시 게이트 (Go/No-Go)

| 기준 | Go 조건 | No-Go 조건 |
|------|---------|-----------|
| 크리티컬 버그 | 0건 | 1건이라도 잔존 |
| 서버 안정성 | 업타임 99.5%+ | 99% 미만 |
| NPS | 40+ | 20 미만 |
| 유저 긍정률 | 70%+ | 50% 미만 |
| 데이터 무결성 | 세이브/결제 정합 100% | 1건이라도 손실 |

**No-Go 시**: 출시 연기 2~4주, 크리티컬 이슈 수정 후 미니 OBT 재실시.
