## 1. 디자인 관점 핵심 문제 3가지

### 1.1 🚨 등급별 비주얼 위계가 "크기"에만 의존

현재 `monster-design/overview.md`는 일반(32px) / 엘리트(48px) / 보스(64px)로 **사이즈만** 차등을 둠.

**문제**: 사용자는 멀리서 작은 픽셀 보고 "어, 큰 놈이네 = 강한 놈이네"만 인지. 16px 차이는 720p 화면에서 너무 미묘함.

**디자인 원칙 부재**:
- 광원·하이라이트 차등 (보스만 림라이트?)
- 실루엣 복잡도 (일반=단순, 보스=비대칭+장식)
- 색상 채도 (보스만 ±10% 채도 부스트?)
- 오라/파티클 (엘리트 이상만 idle 파티클)
- 모션 위계 (보스 idle 8프레임 vs 일반 4프레임)

**FF6 레퍼런스**: 카이저드래곤 vs 일반 도마뱀 — 사이즈 2배인데 **색상 위계, 비대칭 디자인, idle 호흡 모션**으로 위압감 90%를 만듦.

### 1.2 🚨 AI 생성 → 터치업 워크플로우 = "감"으로 굴러감

현재 `ai-prompt-master.md`에 프롬프트는 있지만:
- 생성 후 **터치업 체크리스트 없음** (외곽선 2px? 팔레트 색상 16개 이내? 안티앨리어싱 픽셀 잔존?)
- **반려/재생성 기준 없음** — "느낌이 안 나"를 정량화 불가
- AI 출력물 → Aseprite/Photoshop 핸드오프 **표준 워크플로우 부재**
- **버전 관리 부재** — `monster_v1.png`, `monster_v1_final.png`, `monster_v1_final_REAL.png` 지옥행

### 1.3 🚨 라이선스 안전성 = 시한폭탄

AI 생성 이미지의 함정:
- 사용한 모델이 어떤 데이터로 학습됐는지 출처 추적 불가
- 특정 아티스트 스타일이 그대로 나오는 경우 (예: "Yoshitaka Amano style")
- **상업 출시(2026 상반기 얼리액세스) 시 DMCA/저작권 분쟁 리스크**

현재 메모: 프로젝트에 **라이선스 검증 SOP가 0건**. 이대로 출시하면 진짜 큰일 나요... 😱

## 2. UI/UX 트렌드 벤치마크 (2026)

### 2.1 등급별 비주얼 위계 — 동시대 RPG 사례

| 게임 | 일반 | 엘리트 | 보스 |
|------|------|--------|------|
| **Octopath Traveler 2** (2023) | 기본 픽셀 | **흰색 림라이트** + 0.5x 호흡 | 풀스크린 + 동적 카메라 줌 |
| **Sea of Stars** (2023) | 단순 실루엣 | **에테리얼 파티클** idle | 다단 페이즈 + 컬러 시프트 |
| **Chained Echoes** (2022) | 16색 팔레트 | 24색 + 1px 글로우 | 32색 + 화면 전체 비네팅 |

**가춘운의 추천**: Octopath식 **림라이트 차등**이 픽셀아트에 가장 잘 먹힘. 1-2px만 추가해도 "이거 다른 놈"이 즉각 인지됨.

### 2.2 AI→픽셀 터치업 — 2026 워크플로우 표준

업계 동향:
- **Pixellab.ai** (Indie 표준): AI 생성 → 자동 다운샘플 → 팔레트 양자화 → 수동 픽셀 정정
- **Astortion 개발팀** (Steam 인디): SD + ControlNet으로 실루엣 고정 → Aseprite 30분 터치업
- **Bo Salem 워크플로우**: AI 출력 → **Hue/Saturation 16색 양자화 (필수)** → 픽셀 단위 외곽선 재작업

**공통 패턴**: AI는 **컨셉/실루엣 빠른 탐색**에 쓰고, **최종 픽셀은 사람이 양자화**. 이게 라이선스 회피에도 도움 (출력물의 75%는 사람이 다시 그림).

## 3. 디자인 시스템 관점 권장 방향 (Plan 단계 인풋)

### 3.1 등급별 비주얼 위계 토큰 (5개 축)

```yaml
monster.tier:
  normal:
    size: 32x32
    outline: 2px #000
    rim_light: none
    palette_max: 16
    idle_particles: 0
    idle_frames: 4
    motion_amplitude: ±1px
  elite:
    size: 48x48
    outline: 2px #000 + 1px highlight
    rim_light: 1px #FFD700 (top-left)
    palette_max: 24
    idle_particles: 2~3 (subtle)
    idle_frames: 6
    motion_amplitude: ±2px (breathing)
  boss:
    size: 64x64+ (비대칭 허용)
    outline: 2px #000 + 2px gradient highlight
    rim_light: 2px ether-blue (#89CFF0) full silhouette
    palette_max: 32
    idle_particles: 5~8 + screen ambient
    idle_frames: 8
    motion_amplitude: ±3px (heavy breathing)
    extra: 진입 시 화면 진동 + 컬러 시프트
```

이걸 **토큰화**해서 `DESIGN.md §10 Monster Tier Tokens` 신설 권장!

### 3.2 AI→터치업 워크플로우 5단계 표준

```
[1. 컨셉 탐색]
  AI 모델: Stable Diffusion XL + Pixel Art LoRA
  프롬프트: 등급별 템플릿 (ai-prompt-master.md 확장)
  출력: 4종 후보 1024x1024
  소요: 5분

[2. 후보 선정 + 실루엣 검증]
  체크: 16x16 축소 시 종 구분 가능?
  체크: 등급 토큰 일치? (림라이트 위치, 비대칭 정도)
  소요: 5분

[3. 다운샘플 + 팔레트 양자화 (필수)]
  도구: Pixelorama 또는 Aseprite
  처리: nearest-neighbor → 32/48/64 px → 팔레트 16/24/32색 강제
  소요: 5분

[4. 수동 픽셀 터치업 (라이선스 안전 핵심)]
  외곽선 2px 재작업 (AI 외곽선은 거의 1.5px 흐림)
  안티앨리어싱 픽셀 제거
  팔레트 위반 색상 스왑
  소요: 20-30분 (일반) / 60분 (엘리트) / 2-3시간 (보스)

[5. 애니메이션 프레임 분기]
  idle/attack/hit/death 프레임 추가
  보스는 페이즈 전환 모션
  소요: 1-2시간 (일반) / 6-8시간 (보스)
```

**총 시간 추산**: 일반 1.5h, 엘리트 4h, 보스 12h → 160종 전부면 약 **640시간** ≈ 4명 풀타임 1개월. 

### 3.3 라이선스 안전성 검증 게이트

```
PR 머지 전 자동 체크:
  ✅ AI 모델 + LoRA 출처 메타데이터 첨부 (.json sidecar)
  ✅ 프롬프트에 작가명 직접 호출 0건 (정규식 차단)
  ✅ 픽셀 차이율: AI 원본 vs 최종 ≥ 60% (사람 손길 증명)
  ✅ 팔레트 양자화 통과 (정의된 16/24/32색 외 0픽셀)
  ✅ Reverse image search 자동 실행 (Google Lens API) → 유사도 < 80%

이 게이트를 통과한 에셋만 `client/public/assets/monsters/` 진입 허용.
```

심요연 데이터 기반으로 **자동 검증 스크립트** 만들 수 있을 것 같아요!

## 4. UX 관점 사용자 경험 영향

### 4.1 플레이어가 느끼는 "다양성" = 실루엣 차이

160종이 있어도 실루엣이 비슷하면 "맨날 똑같은 거 잡는다"고 느낌.
- 지역당 20종 중 **실루엣 카테고리 5종 이상** (인간형/짐승형/곤충형/기계형/원소형) 강제
- 16x16 썸네일에서 5초 내 카테고리 식별 가능해야 함 (도감 UI 기준)

### 4.2 보스전 진입 임팩트

현재 부재:
- 보스 등장 컷씬 (3-5초 화면 전환)
- HP바 등장 애니메이션
- 보스 이름 타이포 페이드인

→ Build 단계에서 계섬월에게 **`BossIntroSequence` 컴포넌트** 신설 요청 필요!

### 4.3 도감/콜렉션 UI 영향

160종 도감을 만들면 **그리드에서 비주얼 일관성**이 결정타. 등급 토큰이 명확해야 도감 UI에서:
- 일반 = 일반 그리드 셀
- 엘리트 = 골드 테두리 셀
- 보스 = 풀폭 카드 + 시네마틱 배경

가 자연스럽게 나와요. 정경패 PM의 PRD에 도감 UI 스코프 포함 권장!
