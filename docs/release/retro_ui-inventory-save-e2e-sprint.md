## 6. Keep · Problem · Try

### 🟢 Keep (이어갈 것)
- **계섬월 development 단계 회복** — 443초 투입으로 1,758 LOC 테스트 작성. 외부 모델 흡수 정책이 효과를 보였사옵니다.
- **9단계 자동 스프린트 흐름의 안정성** — 단계 성공률 89%, 평균 95초로 페이스 유지.
- **문서·코드 병행 산출** — 진채봉·가춘운이 가이드/템플릿/디자인 자산을, 계섬월이 코드를 동시 산출.
- **이소화 보안 감사의 코드 동반** — `security-audit-ui-inventory-save.md`가 실제 테스트 335 LOC와 짝을 이루었사옵니다.

### 🟡 Problem (반복되는 결함)
- **커밋·PR 게이트 부재** — 1,758 LOC 테스트가 워킹트리에 잠들어 있음. **지난 retro에서 지적된 동일 패턴 재발**.
- **테스트 실행 보고 부재** — `tests/reports/` 디렉터리만 생성, 실제 통과/실패 결과 미기록.
- **`launch_checklist` 처리 추적 단절** — 커밋이 없으니 체크박스 갱신 불가.
- **워킹트리 누적** — 73개 변경 파일이 다음 스프린트의 머지 충돌 폭탄으로 작용할 위험.

### 🔵 Try (다음 스프린트에서 시도)
- **자동 커밋 게이트 도입** — development 단계 종료 시 `git status` 확인 → 변경 시 자동 `git add` + 커밋 PR 생성 훅.
- **테스트 실행 게이트** — qa 단계에서 `npm test` 자동 실행 → `tests/reports/<sprint-id>.json` 산출 강제.
- **스프린트별 브랜치** — `auto/sprint-<topic>-<date>` 브랜치 자동 생성 → master 보호.
- **DoD 3종 합본** — ① `git diff --stat ≠ 0` ② `git log -1 --since="1 day ago" ≠ 0` ③ `tests/reports/*.json` 존재.
