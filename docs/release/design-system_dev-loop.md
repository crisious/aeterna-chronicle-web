## 0. 디자인 철학 — "터미널이 친구처럼 말 걸게"

대표님 본인이 1인 사용자입니다. 화면이 아니라 **터미널이 사용자 인터페이스** 입니다.
잠깐~ 그래서 디자인 원칙이 뒤집히네요!! 🙃✨

| 원칙 | 보통 GUI | 본 시스템 (CLI) |
|---|---|---|
| **첫 시선** | 큰 글자/색 | **첫 7글자 + 색 1점** |
| **실패 표현** | 모달/배너 | **2줄 — 1줄=원인, 2줄=경로:라인** |
| **계층** | 그림자/크기 | **들여쓰기 + 트리 글리프(`├─└─`)** |
| **모션** | 트랜지션 | **스피너 1종 + 진행 바 1종 — 끝** |
| **접근성** | 색약 대응 | **`NO_COLOR=1` 강제 mono 모드** |

> **봉인 1**: 색은 *상태(state)* 만 표현. 장식·그라데이션·이모지 무지개 금지.
> **봉인 2**: 출력 폭 **80자** 안에서 끊는다 — 80자 넘으면 `…` 절단.
> **봉인 3**: 첫 글자가 늘 *상태 글리프* — 사용자는 1열만 봐도 된다.

## 1. 컬러 토큰 — 4상태 × 1 보조

### 1.1 ANSI 팔레트 (코어 SSOT)

| 토큰 | ANSI | RGB hint | 의미 | 사용처 |
|---|---|---|---|---|
| `loop.ok` | `\x1b[32m` (green) | `#3DD68C` | 성공 — 게이트 통과, 시나리오 PASS | `✓` 글리프, 진행바 채움 |
| `loop.warn` | `\x1b[33m` (yellow) | `#F5C24A` | 경고 — 5분 임박(>4:00), 부팅 느림 | `▲` 글리프, 시간표시 |
| `loop.err` | `\x1b[31m` (red) | `#E5484D` | 실패 — 시나리오 FAIL, 빌드 에러 | `✗` 글리프, 에러 1줄 |
| `loop.info` | `\x1b[36m` (cyan) | `#4CC9F0` | 정보 — 단계 시작, 경로/라인 | `›` 글리프, 파일 경로 |
| `loop.dim` | `\x1b[2m` (faint) | `#6B7280` | 보조 — 시각, 밀리초, 트리 글리프 | `[12.4s]`, `├─└─` |

> **이유**: `DESIGN.md` §2 클라이언트 팔레트(`#3DD68C` 성공 / `#E5484D` 위험 / `#F5C24A` 경고 / `#4CC9F0` 마법)와 RGB hint를 1:1 미러. 게임이랑 터미널이 같은 영혼을 갖게 해요~ 💚

### 1.2 NO_COLOR 폴백

```
환경변수 NO_COLOR=1 → 모든 ANSI 제거, 글리프만 사용
  ✓ ▲ ✗ › ─    (5종 — UTF-8 보장, ASCII 폴백 시 [OK]/[!]/[X]/[>]/[-])
```

봉인: `NO_COLOR=1` 모드에서 정보 손실 0건. 색이 가져가던 의미는 **글리프**가 100% 가져갑니다.

## 2. 타이포그래피 — 모노스페이스 단일

### 2.1 폰트 정책

| 종류 | 폰트 | 비고 |
|---|---|---|
| 본문 | **터미널 기본 모노스페이스** (D2Coding/JetBrains Mono/Cascadia 등) | 사용자 설정 존중 — 강제 안 함 |
| 강조 | **굵게(`\x1b[1m`)** — 색이 아닌 굵기로 분리 | 게이트 라벨, 결과 한 줄에만 |
| 약화 | **희미(`\x1b[2m`)** — 시각/숫자 노이즈 누르기 | `[12.4s]`, `├─` |

### 2.2 위계 — 4단

```
█ Title (Bold + 색)         ▶ npm run loop — boot ⨯ scenario ⨯ gate
█ Section (Bold)            ▶ ① boot
█ Body (default)            ▶   ✓ vite ready in 1.42s
█ Aux (dim)                 ▶   [01:42.3]  client/src/main.ts
```

**규칙**: 같은 줄에 *Bold + 색* 두 개 동시 X. 하나만.

## 3. 레이아웃 — 트리 + 진행 바, 끝

### 3.1 `npm run loop` 출력 표준 (5분 게이트)

```
▶ npm run loop                                                    [16:42:01]
├─ ① boot (Phaser dev server)
│   ✓ vite ready                                              1.42s   ▮▮░░░░░░░░
│   › http://localhost:5173                                  
├─ ② scenario (headless × 3)
│   ✓ battle: ATB → 적 처치 → EXP                             18.7s   ▮▮▮▮░░░░░░
│   ✓ save: 슬롯1 저장 → 새로고침 → 로드                      12.3s   ▮▮▮▮▮▮░░░░
│   ✓ map: 시작방 → 던전1 → 마을복귀                          15.8s   ▮▮▮▮▮▮▮▮░░
├─ ③ gate (≤ 5:00)
│   ✓ total                                                  48.2s ◀ 4:11.8 여유
└─ done                                                       PASS · 4 changed
```

**핵심 룰**:
1. **첫 열은 늘 트리 글리프** (`├─└─│`) — 시각적 골격
2. **둘째 열은 글리프** (`✓ ▲ ✗ ›`) — 상태
3. **셋째 열은 라벨** — Bold X (이미 글리프가 강조)
4. **끝 열은 시간** — `dim` — 노이즈 안 됨
5. **진행바**는 **10칸 고정** (`▮▮▮▮▮▮▮▮▮▮`) — 가변 폭 금지
6. **80자 폭** — 라벨이 길면 가운데서 `…` 절단

### 3.2 실패 시 — 2줄 규칙 (대표님 핵심 요구!!)

```
├─ ② scenario (headless × 3)
│   ✗ battle: ATB → 적 처치 → EXP                             FAILED
│   › TypeError: Cannot read 'hp' of undefined
│   › client/src/scenes/BattleScene.ts:142:18    ← 클릭→VSCode 이동
```

> **대표님이 원하신 건 "에러 발생 시 원인 파일/라인 즉시 노출"** — 그래서 **항상 2줄**!! ✨
> 1줄: 사람이 읽는 원인 (자연어, 색약 폴백 시 글리프만)
> 2줄: 기계가 읽는 좌표 (`path:line:col` — 터미널이 자동 링크화)
> Vite의 sourcemap 정보를 그대로 가공 — 신규 도구 X (백능파 마마 봉인).

### 3.3 빌드 에러 — 같은 2줄 규칙, 다른 색

```
✗ build error                                                   2 issue(s)
› [vite] Failed to resolve import "./Boss" from "BattleScene.ts"
› client/src/scenes/BattleScene.ts:8:18

› [vite] Unexpected token (52:14)
› client/src/data/skills.ts:52:14
```

**규칙**: 한 에러당 **2줄 + 빈 줄 1**. 더 길게 안 만든다 — 스택트레이스는 `--verbose` 플래그로만.

## 4. 5분 게이트 표지 — 한 점에 모든 의미

### 4.1 게이트 레전드

```
PASS  ◀ 4:11.8 여유    ✓ 5분 게이트 통과
WARN  ◀ 0:24.6 임박    ▲ 4:35 — 4:60 사이, 곧 끊김
FAIL  ▶ 0:32.1 초과    ✗ 5:32 — 사이클 약속 깨짐
```

**시각 룰**:
- `◀` = 여유 (남은 시간) — 화살표가 **안쪽**으로
- `▶` = 초과 (넘긴 시간) — 화살표가 **바깥**으로
- 4상태 색 1:1 매칭 — `ok / warn / err / err`

### 4.2 한 줄 요약 (CI/Discord embed로 바로 복붙 가능)

```
PASS · loop 48.2s · boot 1.42s · scenario 46.8s · gate 4:11.8 여유
```

> **재사용**: 이 한 줄은 **PR 본문 / Discord 알림 / `gh pr comment`** 어디 붙여도 80자 안. 진채봉 Editor 봉인이랑 정합~ 🤝

## 5. 모션 — 1 스피너 / 1 진행바

```
스피너    ⠋ ⠙ ⠹ ⠸ ⠼ ⠴ ⠦ ⠧ ⠇ ⠏      (Braille — 80ms 회전)
진행바    ▮▮▮▮▮░░░░░       (5/10 채움 — 5단계만, 보간 X)
```

**금지**: 색 페이드, 글자 깜빡임, 무지개 그라데이션, 이모지 회전.
**이유**: 터미널 다시 그리기 비용 = 측정 노이즈. 깜빡임 = 5분 게이트 측정 정확도 깎임.

    ok: '✓', warn: '▲', err: '✗', info: '›',
    tree: { mid: '├─', end: '└─', vert: '│ ' },
