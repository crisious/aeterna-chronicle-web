## 0. 어머~ 왜 이 문서가 필요한가요? 💭

세이브·로드는요, **유저가 "내 모험이 잘 보존되고 있구나"를 느끼는 유일한 접점**이거든요!

- 자동세이브 알림이 너무 튀면? → 몰입 깨짐 😢
- 너무 조용하면? → "저장됐나?" 불안감
- 손상 복구 다이얼로그가 무서우면? → 유저 패닉, 게임 종료
- 슬롯 카드가 헷갈리면? → 잘못된 슬롯 덮어쓰기 (유저 분노 유발 1순위!)

→ **세이브 UI는 "조용히 신뢰를 쌓는" 디자인 언어가 필요해요.** 화려함보다 **확신·복구·안심**이 핵심 정서!

## 1. 디자인 원칙 (5계명) ✨

| 원칙 | 설명 | 실천 |
|---|---|---|
| **무방해 (Non-intrusive)** | 자동세이브는 게임플레이를 방해하지 않는다 | 토스트 우하단, 8px fade, 1.5s 후 자동 해제 |
| **상태 명료 (Status Clarity)** | 슬롯 4가지 상태가 한눈에 구분 | 색상 + 아이콘 + 텍스트 라벨 3중 인코딩 |
| **불안 최소화 (Reassuring Tone)** | 손상 복구 시에도 안심시킨다 | "걱정 마세요, 백업이 있어요" 카피톤 |
| **되돌릴 수 있음 (Reversible)** | 덮어쓰기 직전 항상 확인 | 5초 Undo 토스트 + 모달 confirm 2중 |
| **일관성 (Consistency)** | DESIGN.md §2 컬러팔레트 그대로 미러 | 신규 컬러 추가 금지, 기존 토큰 조합만 |

## 2. 컬러 시스템 — 슬롯 상태 토큰 🎨

> DESIGN.md §2 코어 팔레트의 **조합**으로만 구성. 신규 hex 추가 ❌

### 2.1 슬롯 상태 5종

| 상태 ID | 한글 라벨 | 핵심 색 (border/glow) | 보조 색 (bg) | 텍스트 | 아이콘 | 의미 |
|---|---|---|---|---|---|---|
| `active` | **활성** ⚔️ | `--color-accent-gold` `#FFD24A` | `#1A1A2E` | `#F5F5F5` | ⚔️ | 현재 진행 중인 메인 슬롯 |
| `backup` | 백업 🛡️ | `--color-accent-blue` `#4A9EFF` | `#16162A` | `#D8D8E8` | 🛡️ | 직전 정상 스냅샷 (자동 보호) |
| `auto` | 자동세이브 🌀 | `--color-accent-blue-soft` `#4A9EFF` 60% | `#14142A` | `#C8C8D8` | 🌀 | 트리거 기반 자동 슬롯 |
| `corrupted` | 손상 ⚠️ | `--color-warning` `#E8A33A` | `#2A1F14` | `#F5E1B8` | ⚠️ | 체크섬 실패 — 복구 후보 표시 |
| `migration` | 마이그레이션 필요 🔄 | `--color-info-indigo` `#7A6BFF` | `#1A1830` | `#DCD8FF` | 🔄 | v1→v2 호환 로드 가능 |
| `empty` | 빈 슬롯 ➕ | `--color-neutral-600` `#3A3A52` (dashed) | `#0F0F1E` | `#7A7A92` | ➕ | 사용 가능 |

### 2.2 절대 쓰지 않는 색

- ❌ **순수 빨강 (`#FF0000` 류)** — 손상도 위험으로 받아들이지 않게! `corrupted`는 따뜻한 호박색(amber)으로!
- ❌ **새로운 액센트** — DESIGN.md §2.2 팔레트 내에서만!
- ❌ **순수 초록 "성공"** — 활성 슬롯은 금색이 정체성!

### 2.3 글로우/섀도우 토큰 (활성/백업 강조)

```
--save-slot-active-glow:    0 0 12px rgba(255, 210, 74, 0.4),  inset 0 0 8px rgba(255, 210, 74, 0.15);
--save-slot-backup-glow:    0 0  8px rgba( 74,158,255, 0.3),  inset 0 0 6px rgba( 74,158,255, 0.12);
--save-slot-corrupted-glow: 0 0  6px rgba(232,163, 58, 0.35), inset 0 0 4px rgba(232,163, 58, 0.10);
```

→ glow는 **상태가 강조된 슬롯에만** 적용 (모든 슬롯에 = 노이즈)

## 4. 레이아웃 & 컴포넌트 🖼️

### 4.1 세이브/로드 슬롯 리스트 화면

```
┌─────────────────────────────────────────────────┐
│  ⚔️ 세이브 슬롯                       [닫기 ✕]  │  ← title bar 56px
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─────────────────────────────────────────┐   │
│  │ ⚔️ ACTIVE   슬롯 1 — 에리언 Lv.42       │   │  ← 활성: 금색 glow
│  │            Ch.4 / 망각의 고원 / 32:14   │   │     높이 88px
│  │            2분 전 자동저장              │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ 🛡️ BACKUP  슬롯 2 — 에리언 Lv.41        │   │  ← 백업: 청 glow
│  │            Ch.4 / 솔라리스 / 28:32      │   │
│  │            1시간 전 자동백업            │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ 🌀 AUTO    슬롯 3 — 에리언 Lv.40        │   │
│  │ ⚠️ CORRUPT 슬롯 4 — (손상 — 백업 사용)  │   │  ← 호박색 점선 테두리
│  │ ➕ EMPTY   슬롯 5 — 비어 있음           │   │  ← 점선 dashed
│  └─────────────────────────────────────────┘   │
│                                                 │
│  [수동 저장]            [백업 슬롯 복구]        │  ← 액션바 64px
└─────────────────────────────────────────────────┘
        스페이싱: 8/12/16/24 그리드 (DESIGN.md §4)
```

**핵심 결정**:
- 슬롯 카드 높이 **88px 고정** (정보밀도 vs 스캔성 균형)
- 카드 사이 간격 **12px** (붙으면 시각 피로)
- 활성/백업/자동 그룹 **시각적 클러스터링** (백색 1px 구분선)
- 손상 슬롯은 **상호작용 가능** (복구 후 진입), 빈 슬롯과 다름!

### 4.2 자동세이브 토스트 (무방해)

```
                                ┌────────────────────────┐
                                │ ✨ 여기까지 안전하게   │  ← 우하단 32px 마진
                                │    기록했어요          │     400px × 56px
                                │                  [⚔️]  │     fade-in 200ms
                                └────────────────────────┘     hold 1500ms
                                                                fade-out 300ms
```

- **위치**: viewport 우하단 (게임 HUD 안 가림)
- **트리거**: 두련사 설계의 5개 트리거 지점 (씬 전환/보스 처치/레벨업/Ch 클리어/30s 주기)
- **사운드 SFX**: `ui.save.success` (LUFS -18, 매우 짧은 chime, sound-system §UI 9액션 정합)
- **애니메이션**: 우측에서 8px 슬라이드 + opacity 0→1
- **z-index**: HUD 위, 모달 아래 (`z-toast: 9000`)

### 4.3 손상 복구 다이얼로그 (불안 최소화)

```
┌──────────────────────────────────────────┐
│  🛡️                                      │  ← 큰 아이콘 48px
│                                          │
│  마지막 정상 시점에서                    │  ← H2 22px
│  이어가시겠어요?                         │
│                                          │
│  ─────────────────────────────────────   │
│                                          │
│  슬롯 1이 살짝 흔들렸어요. 다행히        │  ← 본문 14px
│  1시간 전 백업이 안전하게 남아 있어요.   │     3줄 이내
│  잃는 진척은 약 12분 분량이에요.         │
│                                          │
│  [백업으로 이어가기 ⚔️]    [취소]        │  ← Primary 금색
└──────────────────────────────────────────┘     Secondary 회색
   modal 480px × auto, 중앙, backdrop blur 8px
```

**카피톤**:
- ❌ "Critical Error / Data Loss"
- ✅ "살짝 흔들렸어요" + "안전하게 남아 있어요" + **잃는 분량 명시** (사실 + 안심)

### 4.4 로드 검증 진행 인디케이터 (4단계)

```
   ① 파일 읽기  →  ② 스키마 검증  →  ③ 마이그레이션  →  ④ 참조 검증
   ●●●●●●●●●●     ●●●●●●○○○○         ○○○○○○○○○○         ○○○○○○○○○○
```

- 4단계가 두련사의 SaveSnapshot 파이프라인 그대로 미러
- 각 단계 ~150ms 이내 완료 시 인디케이터 **생략** (UX 노이즈 방지)
- 마이그레이션 단계는 v1→v2 케이스에서만 노출
- 실패 시 **해당 단계 색만 호박색 전환** + 복구 다이얼로그(4.3) 호출

## 6. 접근성 (a11y AAA 미러) ♿

> 기존 `design_a11y-aaa-audit-report.md` SSOT 그대로 따름. 추가 요건만 명시.

- **색약 대응**: 슬롯 상태는 색상 + 아이콘 + 텍스트 라벨 **3중 인코딩** (색맹 검사기 통과 의무)
- **명도 대비**:
  - 활성 슬롯 텍스트 `#F5F5F5` on `#1A1A2E` = **17.4:1** (AAA ✓)
  - 손상 슬롯 텍스트 `#F5E1B8` on `#2A1F14` = **9.8:1** (AAA ✓)
- **키보드 내비**: 슬롯 카드 ↑↓, Enter 진입, Delete 삭제 (confirm 2중)
- **스크린리더 라벨**: "슬롯 1, 활성, 에리언 레벨 42, 4장 망각의 고원, 2분 전 자동저장"
- **모션 감소 모드**: `prefers-reduced-motion` 시 토스트 fade only, 슬라이드 제거

```
       ↓

```

### 7.1 신설 권고 파일 인터페이스 (계섬월에게)

```typescript
// client/src/design_tokens/save_slot.ts (신설)
export const SAVE_SLOT_STATE_ORDER = [
  'active', 'backup', 'auto', 'corrupted', 'migration', 'empty'
] as const;

export type SaveSlotStateId = typeof SAVE_SLOT_STATE_ORDER[number];

export interface SaveSlotStateToken {
  readonly id: SaveSlotStateId;
  readonly labelKo: string;
  readonly labelEn: string;
  readonly icon: string;        // 이모지 or 아이콘 키
  readonly border: { colorHex: string; widthPx: number; style: 'solid' | 'dashed'; };
  readonly background: { colorHex: string; };
  readonly text: { primaryHex: string; metaHex: string; };
  readonly glow?: { boxShadow: string; };  // 활성/백업만
}
```

→ 본 문서 §2.1 표가 SSOT, 토큰 파일은 **단순 미러**. 단방향 흐름!
