## 4. 권장 P9 구성

**트랙 A(Unity) + C(보안/결제)** = **P9: 멀티 플랫폼 + 실서비스 준비**

| 이유 | 설명 |
|------|------|
| GDD 원래 로드맵 | Unity 클라이언트는 GDD Phase 2에서 이미 계획됨 |
| 실서비스 필수 | 안티치트/결제/GDPR 없이 라이브 불가 |
| 시즌 3는 라이브 이후 | 콘텐츠 추가보다 플랫폼 확장이 DAU 증가에 효과적 |
| 에셋은 병렬 가능 | 아트 작업은 별도 파이프라인으로 진행 가능 |

### 추천 구성 (20 티켓, 12주)

```
Sprint 1-2 (4주): Unity 클라이언트 — A-01~A-05 (프로젝트+통신+인증+맵+전투)
Sprint 3-4 (4주): Unity 완성 + 보안 — A-06~A-08 + C-01~C-04 (UI/빌드 + 안티치트/결제)
Sprint 5-6 (4주): 크로스플레이 + 법적 — A-09~A-10 + C-05~C-10 + P9-20 통합 RC
```

### P10 예고 (시즌 3 + 에셋)
```
트랙 B + D → P10: 시즌 3 콘텐츠 + 비주얼 폴리싱
챕터 7 + 5번째 클래스 + 무한 던전 + 에셋 파이프라인
```
