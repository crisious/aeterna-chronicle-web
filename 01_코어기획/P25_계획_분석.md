# P25 계획 분석 — 클라이언트-서버 연동

> 작성일: 2026-03-14 | Phase: P25 | 순서: 469~488

---

## 1. 목표

모든 Phaser 씬에 NetworkManager를 연결하고, 에셋 경로를 실제 파일과 매핑하며, UI 컴포넌트를 서버 데이터에 바인딩한다. 현재 BattleScene, WorldScene, DungeonScene, CharacterSelectScene이 API 연동 없이 하드코딩된 상태이다.

## 2. 현황 분석

### 2.1 API 미연동 씬
| 씬 | 줄 수 | 현재 상태 |
|----|-------|-----------|
| BattleScene | 527줄 | 전투 UI만 구현, 서버 전투 엔진 미연결 |
| WorldScene | 231줄 | 존 표시만, 서버 위치/몬스터 동기화 없음 |
| DungeonScene | 358줄 | 던전 구조 하드코딩, 서버 DungeonRun 미연결 |
| CharacterSelectScene | 300줄 | 목 데이터, 서버 Character API 미연결 |

### 2.2 에셋 현황
- `unity-client/Assets/Resources/` — Unity 포맷 에셋 원본
- `client/public/assets/` — Phaser용 에셋 경로 (일부 누락)
- 아틀라스 파일 미배치

### 2.3 네트워크 인프라
- Socket.IO 클라이언트 라이브러리 설치 완료
- REST API 엔드포인트 완비 (30+ 라우트)
- Protobuf 코덱 shared/codec 존재

## 3. 구현 계획

### 3.1 NetworkManager 통합
- 싱글톤 NetworkManager 클래스 (REST + Socket.IO)
- 인증 토큰 자동 주입, 요청 재시도 (3회)
- 연결 상태 모니터링 + 자동 재접속

### 3.2 씬별 연동
- BattleScene ↔ P24 전투 엔진 (combat:start/action/tick/end)
- WorldScene ↔ 월드 상태 브로드캐스트 (world:state)
- DungeonScene ↔ 던전 API (POST /dungeons/enter, /dungeons/clear)
- CharacterSelectScene ↔ 캐릭터 API (GET /characters, POST /characters)

### 3.3 에셋 파이프라인
- Unity 에셋 → Phaser 포맷 변환 스크립트
- 아틀라스 JSON 생성 + 배치
- preload 함수에 실제 파일 경로 매핑

### 3.4 에러 핸들링
- 네트워크 에러 → 사용자 알림 + 자동 재시도
- 세션 만료 → 로그인 화면 리다이렉트
- 로딩 화면 + 진행 바 (에셋 + 데이터 로딩)

## 4. 리스크

| 리스크 | 대응 |
|--------|------|
| 에셋 포맷 불일치 | 변환 스크립트 + 수동 검수 |
| 네트워크 지연으로 UX 저하 | 클라이언트 예측 + 서버 보정 |
| 씬 전환 시 소켓 누수 | 씬 destroy 시 리스너 해제 |
