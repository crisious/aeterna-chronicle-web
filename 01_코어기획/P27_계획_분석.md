# P27 계획 분석 — 파티/거래/소셜

> 작성일: 2026-03-14 | Phase: P27 | 순서: 509~528

---

## 1. 목표

파티, 거래, 경매장, 채팅, 친구, 길드 등 소셜 시스템의 UI를 완성하고 서버 API와 연동한다. 서버 측에는 이미 guildRoutes, auctionRoutes, chatSocketHandler 등이 구현되어 있다.

## 2. 현황

### 2.1 기존 서버 코드
| 모듈 | 상태 |
|------|------|
| `guildRoutes.ts` | 길드 CRUD, 멤버 관리, 길드 스킬 API 완료 |
| `auctionRoutes.ts` | 경매 등록/입찰/낙찰 API 완료 |
| `chatSocketHandler.ts` | 일반/파티/길드/귓속말 소켓 핸들러 완료 |
| `socialSocketHandler.ts` | 친구 요청/수락/거절, 온라인 상태 완료 |
| `matchmakingRoutes.ts` | 매칭 큐 API 완료 |
| `guildWarSocketHandler.ts` | 길드전 소켓 핸들러 완료 |

### 2.2 클라이언트 측
- 파티/거래/소셜 UI 모두 미구현
- 채팅 기본 구조만 존재

## 3. 구현 계획

### 3.1 파티 시스템 UI
- 파티 생성, 초대, 수락/거절, 탈퇴
- 파티 멤버 HP/MP 상태 표시
- 파티 전투 동기화 (전투 룸 공유)

### 3.2 거래 시스템
- 1:1 거래 요청/수락 UI
- 아이템/골드 교환 인터페이스
- 거래 확인/취소 플로우

### 3.3 경매장 UI
- 아이템 검색/필터 (종류, 등급, 가격)
- 등록/입찰/즉시구매
- 나의 경매 관리 (등록 목록, 입찰 목록)

### 3.4 채팅 UI
- 채널 탭 (일반/파티/길드/귓속말)
- 메시지 입력/전송
- 이모지, 시스템 메시지 구분

### 3.5 친구/소셜
- 친구 목록 + 온라인/오프라인 표시
- 친구 요청/수락/차단
- 유저 프로필 조회

### 3.6 길드 UI
- 길드 목록/검색/가입/생성
- 길드 멤버 관리 (직급, 추방)
- 길드 스킬/버프 관리
- 길드 전쟁 UI

## 4. 리스크

| 리스크 | 대응 |
|--------|------|
| 파티 전투 동기화 복잡도 | P24 전투 엔진의 멀티 플레이어 지원 활용 |
| 거래 악용 (복사/탈취) | 서버 트랜잭션 + 양측 확인 필수 |
| 채팅 스팸 | 레이트 리밋 + 신고 시스템 |
