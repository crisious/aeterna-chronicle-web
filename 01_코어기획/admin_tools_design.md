# 에테르나 크로니클 — 운영 도구

> **작성일:** 2026-03-10 | **버전:** v1.0 | **분류:** 코어기획 — 운영/관리

---

## 1. 개요

본 문서는 에테르나 크로니클의 게임 운영(Game Operations) 도구를 정의한다. GM 도구, 아이템 관리, 제재 시스템, 이벤트 관리, 실시간 대시보드, 로그 시스템, 권한 체계를 포함한다.

### 1.1 설계 원칙

| 원칙 | 설명 |
|------|------|
| **최소 권한** | 각 역할에 필요한 최소한의 권한만 부여 |
| **감사 추적** | 모든 운영 행위는 로그로 기록, 추적 가능 |
| **실수 방지** | 위험 작업은 2단계 확인(더블 컨펌) 필수 |
| **자동화 우선** | 반복 작업은 스크립트/스케줄링으로 자동화 |

## 2. GM 도구

### 2.1 유저 검색

| 검색 키 | 설명 | 예시 |
|---------|------|------|
| 유저 ID | 고유 식별자 | `user_abc123` |
| 캐릭터명 | 인게임 닉네임 (부분 일치 지원) | `에리언*` |
| 이메일 | 계정 이메일 | `user@example.com` |
| IP 주소 | 최근 접속 IP | `192.168.1.*` |
| 디바이스 ID | 기기 식별자 | - |

**검색 결과 표시 정보:**
- 계정 정보: ID, 이메일, 가입일, 마지막 접속, 계정 상태
- 캐릭터 목록: 이름, 레벨, 클래스, 전직 단계, 서버
- 제재 이력: 경고/밴 기록
- 결제 이력: 총 결제액, 최근 결제일

### 2.2 캐릭터 상태 조회/수정

**조회 가능 정보:**

| 카테고리 | 항목 |
|----------|------|
| 기본 정보 | 이름, 레벨, 클래스, 전직, 경험치 |
| 스탯 | HP/MP, 공격력/방어력, 치명타 등 전체 |
| 장비 | 14포지션 착용 장비 상세 (강화/소켓/세트) |
| 인벤토리 | 전체 아이템 목록 (검색/필터 지원) |
| 퀘스트 | 진행 중/완료 퀘스트 목록, 진행도 |
| 위치 | 현재 지역, 좌표, 인스턴스 ID |
| 소셜 | 길드, 친구 목록, 차단 목록 |
| 화폐 | 골드, 에테르 결정, 기타 재화 |
| 전투 기록 | 최근 50회 전투 로그 (PvE/PvP) |

**수정 가능 항목 (권한별):**

| 항목 | GM | 슈퍼어드민 | 2단계 확인 |
|------|-----|-----------|-----------|
| 레벨 변경 | ❌ | ✅ | ✅ |
| 경험치 조정 | ❌ | ✅ | ✅ |
| 골드 조정 | ✅ (±100만 이내) | ✅ (무제한) | 100만 초과 시 |
| 스탯 리셋 | ❌ | ✅ | ✅ |
| 퀘스트 상태 변경 | ✅ | ✅ | ❌ |
| 스킬 리셋 | ✅ | ✅ | ❌ |
| 닉네임 변경 | ✅ | ✅ | ❌ |

### 2.3 텔레포트

| 기능 | 설명 | 권한 |
|------|------|------|
| GM → 유저 위치 | GM 캐릭터를 대상 유저 위치로 이동 | GM+ |
| 유저 → 지정 위치 | 대상 유저를 특정 좌표로 강제 이동 | GM+ |
| 유저 → 유저 | 대상 유저를 다른 유저 위치로 이동 | 슈퍼어드민 |
| 지역 강제 이동 | 특정 지역의 전체 유저를 안전 지점으로 이동 (긴급) | 슈퍼어드민 |

## 3. 아이템 관리

### 3.1 아이템 지급

| 항목 | 설명 |
|------|------|
| 대상 | 단일 유저 / 다중 유저 (CSV 업로드) / 전체 유저 |
| 아이템 선택 | 아이템 DB 검색 (ID/이름/등급/카테고리 필터) |
| 수량 | 1~9999 (등급별 상한 설정 가능) |
| 귀속 설정 | 획득 즉시 귀속 / 거래 가능 / 기간 한정 |
| 만료일 | 없음 / 1일~365일 |
| 사유 | 필수 입력 (보상/보정/이벤트/테스트) |
| 우편 발송 | 인벤토리 직접 추가 또는 시스템 우편 선택 |

**지급 제한:**
- 신화 등급 아이템: 슈퍼어드민만 가능 + 2단계 확인
- 전설 등급 이상 일 50개 상한 (남용 방지)
- 전체 유저 지급: 슈퍼어드민 + 2단계 확인 + 사유 첨부 필수

### 3.2 아이템 회수

| 항목 | 설명 |
|------|------|
| 대상 | 단일 유저 / 아이템 인스턴스 ID 지정 |
| 회수 방식 | 인벤토리에서 직접 제거 / 착용 중이면 자동 해제 후 제거 |
| 사유 | 필수 입력 (버그 보정/부정 획득/이벤트 종료) |
| 보상 대체 | 회수 시 대체 아이템/골드 지급 옵션 |
| 대량 회수 | 아이템 ID 기준 전 서버 일괄 회수 (슈퍼어드민) |

### 3.3 아이템 로그

모든 아이템 이동은 로그로 기록된다:

| 로그 필드 | 설명 |
|-----------|------|
| timestamp | 발생 시각 (UTC) |
| action | 획득/사용/파괴/거래/강화/드롭/GM지급/GM회수 |
| itemId | 아이템 인스턴스 고유 ID |
| itemTemplateId | 아이템 종류 ID |
| fromUserId | 출발지 유저 (시스템/NPC/유저) |
| toUserId | 도착지 유저 |
| quantity | 수량 |
| metadata | 추가 정보 (강화 단계, 가격 등) |
| operatorId | GM/시스템 ID (운영 작업 시) |
| reason | 사유 |

## 4. 제재(밴) 시스템

### 4.1 제재 단계

| 단계 | 유형 | 기간 | 적용 범위 |
|------|------|------|-----------|
| **1단계** | 경고 | - | 알림만 (인게임 + 이메일) |
| **2단계** | 채팅 제한 | 1~72시간 | 채팅 기능만 비활성 |
| **3단계** | 임시 밴 | 1~30일 | 로그인 차단 |
| **4단계** | 영구 밴 | 영구 | 로그인 차단, 계정 비활성화 |

### 4.2 제재 사유 분류

| 사유 코드 | 설명 | 기본 제재 |
|-----------|------|-----------|
| `HACK_01` | 스피드핵/데미지핵 | 영구 밴 |
| `HACK_02` | 오토 프로그램 | 임시 밴 7일 (재범 영구) |
| `BUG_01` | 버그 악용 (경미) | 경고 + 이득 회수 |
| `BUG_02` | 버그 악용 (중대/의도적) | 임시 밴 30일 + 이득 회수 |
| `ABUSE_01` | 욕설/비매너 | 채팅 제한 24시간 |
| `ABUSE_02` | 혐오 발언/성희롱 | 임시 밴 7일 (재범 영구) |
| `ABUSE_03` | 현금 거래(RMT) | 임시 밴 30일 + 재화 회수 (재범 영구) |
| `FRAUD_01` | 결제 사기/차지백 | 영구 밴 |
| `OTHER` | 기타 | GM 재량 |

### 4.3 이의제기(Appeal) 플로우

```
[유저 제재 발생]
    │
    ├── 인게임/이메일로 제재 알림 발송
    │   (사유 코드, 기간, 이의제기 안내)
    │
    ├── [유저] 이의제기 접수 (웹 폼)
    │   ├── 제재 ID
    │   ├── 사유 설명 (최대 2000자)
    │   └── 증빙 자료 첨부 (선택, 최대 5MB)
    │
    ├── [CS팀] 1차 검토 (48시간 이내)
    │   ├── 기각 → 유저 알림 (최종)
    │   ├── 추가 조사 필요 → GM팀 에스컬레이션
    │   └── 해제 → 제재 즉시 해제 + 유저 알림
    │
    ├── [GM팀] 2차 검토 (72시간 이내)
    │   ├── 기각 → 유저 알림 (최종)
    │   ├── 감경 → 제재 단계 하향 조정
    │   └── 해제 → 제재 즉시 해제 + 보상 검토
    │
    └── [슈퍼어드민] 최종 검토 (영구 밴 이의 시)
        ├── 기각 (최종, 재이의 불가)
        └── 해제/감경
```

**이의제기 제한:**
- 경고: 이의제기 불가
- 임시 밴: 1회 이의제기 가능
- 영구 밴: 2회 이의제기 가능 (30일 간격)

| 기능 | 설명 |
|------|------|

| 배포 방식 | 설명 | 타이밍 |
|-----------|------|--------|
| **자동 지급** | 조건 충족 시 즉시 우편 발송 | 실시간 |
| **수동 지급** | GM이 대상 선정 후 일괄 발송 | 수동 |
| **클레임 방식** | 유저가 이벤트 UI에서 직접 수령 | 유저 주도 |
| **랭킹 보상** | 이벤트 종료 후 랭킹 기준 자동 배포 | 이벤트 종료 +1시간 |

## 7. 로그 시스템

### 7.1 로그 카테고리

| 카테고리 | 기록 내용 | 보관 기간 |
|----------|-----------|-----------|
| **행동 로그** | 로그인/아웃, 레벨업, 퀘스트 완료, 던전 입장, 전투 결과 | 30일 |
| **거래 로그** | 아이템 거래, 상점 구매/판매, 강화, 경매장 | 30일 |
| **채팅 로그** | 일반/파티/길드/귓속말/월드 채팅 | 30일 |
| **결제 로그** | 인앱 결제, 환불, 차지백 | 365일 |
| **운영 로그** | GM/어드민 모든 작업 (수정/지급/밴) | 365일 |
| **보안 로그** | 로그인 시도, IP 변경, 비밀번호 변경, 의심 행위 | 365일 |
| **시스템 로그** | 서버 에러, 성능 메트릭, 배포 이력 | 90일 |

### 7.2 로그 검색

| 검색 조건 | 설명 |
|-----------|------|
| 유저 ID | 특정 유저의 모든 로그 |
| 시간 범위 | 시작~종료 시각 |
| 카테고리 | 행동/거래/채팅/결제/운영/보안 |
| 액션 타입 | 세부 액션 필터 (예: 거래 로그 중 "강화"만) |
| 키워드 | 자유 텍스트 검색 (채팅 로그) |
| IP 주소 | 특정 IP에서의 모든 활동 |
| 아이템 ID | 특정 아이템의 이동 경로 추적 |

### 7.3 로그 보관 정책

```
실시간 로그 (hot)
  │  0~7일: Elasticsearch (즉시 검색 가능)
  │
  ├── 7~30일: PostgreSQL 아카이브 (쿼리 가능, 느림)
  │
  └── 30일+: S3/Glacier 콜드 스토리지 (결제/운영/보안만)
      └── 365일 후: 자동 삭제 (법적 보관 의무 항목 제외)
```

## 8. 권한 체계

### 8.1 역할 정의

| 역할 | 인원 목표 | 설명 |
|------|-----------|------|
| **슈퍼어드민** | 1~3명 | 전체 시스템 접근. 제재/지급 무제한. 설정 변경 |
| **GM** | 5~10명 | 유저 관리, 아이템 지급/회수, 제재, 이벤트 운영 |
| **CS** | 10~20명 | 유저 조회(읽기 전용), 신고 처리, 이의제기 1차 검토 |

### 8.2 API 엔드포인트 권한 매핑

```
# 유저 관리
GET    /api/admin/users/search        — CS, GM, 슈퍼어드민
GET    /api/admin/users/:userId       — CS, GM, 슈퍼어드민
PATCH  /api/admin/users/:userId       — GM, 슈퍼어드민

# 캐릭터 관리
GET    /api/admin/characters/:charId  — CS, GM, 슈퍼어드민
PATCH  /api/admin/characters/:charId  — GM (제한적), 슈퍼어드민 (전체)
```

## 9. GM 인게임 도구

### 9.1 GM 캐릭터 모드

| 기능 | 설명 |
|------|------|
| 투명 모드 | 다른 유저에게 보이지 않음 |
| 무적 모드 | 데미지 면역 |
| 관전 모드 | 특정 유저 시점으로 관전 |
| 속도 부스트 | 이동속도 ×5 |
| 자유 비행 | 지형 무시 이동 |
| GM 표식 | GM 닉네임 옆 특수 아이콘 (필요 시 표시) |

### 9.2 GM 명령어 (인게임 콘솔)

```
/gm goto <유저명>           — 대상 위치로 텔레포트
/gm bring <유저명>          — 대상을 GM 위치로 소환
/gm kick <유저명> <사유>     — 강제 접속 종료
/gm mute <유저명> <시간>     — 채팅 제한
/gm ban <유저명> <기간> <사유> — 제재 적용
/gm item <유저명> <아이템ID> <수량> — 아이템 지급
/gm gold <유저명> <금액>     — 골드 지급/차감
/gm announce <메시지>       — 서버 전체 공지
/gm event <이벤트ID> start|stop — 이벤트 즉시 시작/중단
/gm inspect <유저명>        — 캐릭터 상세 정보 팝업
/gm log <유저명> <카테고리>  — 최근 로그 조회
```

## 10. 운영 워크플로우 예시

### 10.1 버그 악용 신고 처리

```
1. CS: 신고 접수 → 로그 1차 확인
2. CS: 악용 정황 확인 → GM 에스컬레이션
3. GM: 상세 로그 분석 (거래/아이템/행동 로그 교차 확인)
4. GM: 이득 수치 산정
5. GM: 제재 적용 (BUG_01/02) + 이득 회수
6. GM: 동일 버그 영향 유저 일괄 검색 → 추가 조치
7. GM: 버그 리포트 → 개발팀 전달
8. 개발: 핫픽스 배포
9. GM: 전체 공지 (수정 완료 + 향후 악용 시 제재 안내)
```

### 10.2 긴급 서버 점검

```
1. 대시보드: 에러율 급등 알림 수신
2. 슈퍼어드민: 원인 파악 (로그/메트릭 확인)
3. 슈퍼어드민: 인게임 공지 "10분 후 긴급 점검"
4. 슈퍼어드민: 5분 후 추가 공지 + 신규 접속 차단
5. 슈퍼어드민: 서버 점검 모드 전환
6. 개발: 핫픽스 배포
7. 슈퍼어드민: 내부 테스트 서버 검증
8. 슈퍼어드민: 서버 재개 + 전체 공지 + 보상 우편 배포
```
