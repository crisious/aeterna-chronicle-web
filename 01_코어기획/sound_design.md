## 1. 사운드 디자인 철학

### 1.1 핵심 원칙

| 원칙 | 설명 |
|------|------|
| 기억-망각 이원성 | '기억'은 선명하고 따뜻한 사운드, '망각'은 흐릿하고 차가운 사운드로 표현 |
| 에테르-기계 융합 | 마법적 음색(현악/합창)과 기계적 음색(톱니/증기/금속)의 레이어드 조합 |
| 내러티브 사운드 | BGM/SFX가 스토리 진행과 플레이어 선택에 반응하여 감정 몰입을 극대화 |
| 위치 기반 오디오 | 지역별 환경음과 BGM이 자연스럽게 크로스페이드 |
| 접근성 | 자막 시스템, 시각적 사운드 큐 표시 옵션 제공 |

### 1.2 사운드 프로필

| 요소 | 기억 (Memory) | 망각 (Oblivion) |
|------|-------------|----------------|
| 악기 | 현악 앙상블, 첼로, 플루트, 하프 | 리버스 피아노, 디튠드 신스, 보이스 패드 |
| 질감 | 맑고 투명, 잔향 짧음 | 흐릿하고 울림, 긴 리버브, 로우패스 |
| 리듬 | 명확한 박자, 멜로디 중심 | 비정형 리듬, 드론/앰비언트 |
| 온도감 | 따뜻함 (아날로그 색채) | 차가움 (디지털 글리치) |

## 2. BGM 설계

### 2.1 지역별 테마

> AI 생성 BGM 프롬프트 상세는 `bgm_ai_music_design.md` 참조

| 지역 | 테마 키워드 | 주요 악기 | BPM | 키 |
|------|-----------|---------|-----|-----|
| **아르겐티움** | 화려한 디스토피아, 재즈 스팀펑크 | 색소폰, 피아노, 브라스, 기계 타악 | 110~130 | Dm / Am |
| **실반헤임** | 신비로운 고대 숲, 시간 왜곡 | 하프, 플루트, 목관, 셀틱 드럼, 글로켄슈필 | 70~90 | Em / G |
| **에레보스** | 음울한 유령 도시, 처연한 고요 | 솔로 첼로, 리버스 피아노, 디튠드 오르골, 바람 소리 | 50~70 | Fm / Ab |
| **솔라리스 사막** | 혹독한 낮/환상적 밤, 이국적 열기 | 우드, 타블라, 시타르, 브라스 앙상블, 사막 바람 | 90~110 | Am / E |
| **북방 영원빙원** | 웅장한 극지, 숭엄한 시간 정지 | 합창, 오르간, 호른, 현악 오케스트라, 에테르 공명음 | 60~80 | Cm / Eb |
| **브리탈리아 자유항** | 활기찬 항구, 위험한 낭만 | 아코디언, 바이올린 피치카토, 기타, 벤조, 뱃고동 | 120~140 | G / D |

### 2.2 상황별 BGM

| 상황 | 스타일 | 특수 처리 |
|------|--------|---------|
| **필드 탐험** | 지역 테마 기본 트랙 | 시간대(낮/밤)에 따라 악기 레이어 전환 |
| **전투 (일반)** | 지역 테마 + 퍼커션/브라스 레이어 추가 | 적 수에 따라 강도 3단계 |
| **전투 (보스)** | 보스 전용 테마 (보스별 고유) | Phase 전환 시 BGM 전환 |
| **던전** | 지역 테마 다크 변주 | 깊이에 따라 잔향/로우패스 강화 |
| **NPC 대화** | 피아노 + 현악 소품 | 감정 태그에 따라 자동 변주 |
| **시즌 이벤트** | 시즌 전용 테마 | 시즌 기간 한정 |
| **메인 메뉴** | 메인 테마 오케스트라 편곡 | 스토리 진행도에 따라 3단계 변화 |

### 2.3 보스별 테마

| 보스 | 테마 | 특징 |
|------|------|------|
| 기억의 골렘 (Ch.1) | "잊혀진 자들의 합창" | 왜곡된 합창 + 기계적 타격음, 페이즈 전환 시 합창 선명해짐 |
| 수호 고목 말라투스 (Ch.2) | "천년의 수호" | 셀틱 하프 + 목관 앙상블, 설득 루트 시 평화로운 변주 |
| 모래 폭풍의 왕 라와르 (Ch.3) | "황사의 기억" | 타블라 + 시타르 + 브라스 팡파르, 기억 복원 시 솔리안 테마 삽입 |
| 기억 사냥꾼 대장 케인 (Ch.4) | "제국의 송곳니" | 군악대 스타일 스네어 + 전자음, 대사 선택 시 템포 변화 |
| 화신 레테 (Ch.5 형태 1) | "망각의 서곡" | 오르간 + 디튠드 합창, 화려하고 압도적 |
| 화신 레테 (Ch.5 형태 2) | "최후의 기억" | 메인 테마 오케스트라 + 합창 + 에테르 공명음 풀 오케스트레이션 |

### 2.4 AI 생성 BGM 연동

`bgm_ai_music_design.md`의 프롬프트 체계를 기반으로:

| 단계 | 내용 |
|------|------|
| 프리프로덕션 | AI(Suno/Udio) 기반 BGM 프로토타입 생성 → 방향성 검증 |
| 프로덕션 | AI 프로토타입 기반 작곡가 편곡/폴리싱, 또는 AI 트랙 직접 사용 (라이선스 확보) |
| 인터랙티브 레이어 | 스템(퍼커션/멜로디/패드/베이스) 분리 생성 → 게임 엔진 레이어 시스템 적용 |
| 품질 기준 | 44.1kHz/16bit 이상, 루프 지점 크로스페이드 처리, 팝/클릭 잡음 없음 |

## 4. 인터랙티브 오디오

### 4.1 전투 강도별 BGM 레이어 전환

```
[레이어 시스템]

베이스 레이어 (항상 재생):
  - 지역 테마 패드/앰비언트

레이어 1 — 탐험 (전투 없음):
  + 지역 테마 멜로디
  + 환경 SFX 풀 볼륨

레이어 2 — 전투 진입 (일반 1~3마리):
  + 퍼커션 레이어 추가
  + 환경 SFX -12dB
  + 전투 SFX 활성화

레이어 3 — 전투 격화 (4마리 이상 또는 엘리트):
  + 브라스/현악 추가 레이어
  + 퍼커션 강화
  + BPM +10~20%

레이어 4 — 보스 전투:
  → 보스 전용 테마로 크로스페이드 (2초)
  + 보스 Phase 전환 시 BGM 섹션 전환

전환 규칙:
  - 레이어 상승: 0.5초 크로스페이드
  - 레이어 하강: 3초 크로스페이드 (전투 종료 후 여운)
  - 보스 전투 → 필드: 5초 크로스페이드 + 승리 스팅거 (3초)
```

### 4.2 기억 파편 접근 시 오디오 큐

```
[기억 파편 거리 기반 오디오]

거리 30m+: 무음
거리 20~30m: SFX-MEM-001 희미하게 시작 (볼륨 20%)
거리 10~20m: 볼륨 50% + 현 BGM에 에테르 공명음 레이어 추가
거리 5~10m: 볼륨 80% + BGM 로우패스 필터 적용 (파편에 집중)
거리 0~5m: 볼륨 100% + BGM 페이드 (-18dB) + 파편 고유 멜로디 재생
수집 시: SFX-MEM-002 + 스팅거 (기억 복원 모티프, 3초) + BGM 복귀
```

### 4.3 스토리 이벤트 연동

| 이벤트 | 오디오 반응 |
|--------|-----------|
| 기억 소멸 발생 (NPC) | 현 BGM 로우패스 + 리버브 증가 + SFX-MEM-005 |
| 에리언 과거 기억 복원 | 메인 테마 피아노 변주 삽입 (8마디) |
| 배신자 베르나르도 발각 | BGM 급정지 → 2초 정적 → 긴장 스팅거 |
| 엔딩 A 진입 | "기억의 수호자" 테마 오케스트라 풀 편곡 |
| 엔딩 B 진입 | 메인 테마 솔로 피아노 → 점차 합창 가세 |
| 엔딩 C 진입 | 메인 테마 역재생 + 디튠 처리, 점차 소멸 |
| 엔딩 D 진입 | 열두 신 합창 테마 (새로운 곡) → 속편 암시 스팅거 |

### 4.4 시간대/날씨 반응

| 조건 | BGM 변화 | 환경 SFX 변화 |
|------|---------|-------------|
| 낮 → 밤 | 멜로디 악기 축소, 앰비언트 패드 확대 | 주간 소음 축소, 야행성 동물/벌레 소리 추가 |
| 맑음 → 비 | 현악 레이어 추가, 템포 -5% | 빗소리 레이어, 물 웅덩이 발소리 변경 |
| 모래 폭풍 (솔라리스) | 디스토션 증가, 고주파 감쇄 | 바람 강도 상승, 가시거리 연동 |

## 6. 기술 구현

### 6.1 엔진별 오디오 시스템

| 항목 | Phase 1 (Phaser.js) | Phase 2 (Unity) | Phase 3 (UE5) |
|------|-------------------|----------------|---------------|
| 오디오 엔진 | Web Audio API + Howler.js | FMOD/Wwise Unity 통합 | MetaSounds + Wwise UE5 |
| 포맷 | OGG Vorbis (웹 호환) | OGG + WAV | OGG + WAV + ADPCM |
| 동시 채널 | 최대 16ch | 최대 64ch | 최대 128ch |
| 3D 오디오 | 2D 패닝 (좌우) | 3D 스패셜 (HRTF 옵션) | 3D 스패셜 + Ambisonics |
| 인터랙티브 | 커스텀 레이어 시스템 | FMOD/Wwise 이벤트 시스템 | MetaSounds 그래프 |

### 6.2 오디오 에셋 규격

| 항목 | 기준 |
|------|------|
| BGM | 44.1kHz / 16bit / Stereo / OGG (q8) |
| SFX | 44.1kHz / 16bit / Mono (환경: Stereo) / OGG (q6) |
| 보이스 | 48kHz / 24bit / Mono / WAV (원본) → OGG (게임 내) |
| BGM 루프 | 루프 시작/끝 지점 마커 필수, 크로스페이드 50ms |
| SFX 길이 | 일반: 0.2~2초 / 환경 루프: 5~30초 |
| 파일명 규칙 | `{카테고리}_{ID}_{설명}_{버전}.ogg` (예: `btl_001_sword_hit_light_v2.ogg`) |

### 6.3 오디오 메모리 예산

| Phase | BGM 예산 | SFX 예산 | 보이스 예산 | 총합 |
|-------|---------|---------|-----------|------|
| Phase 1 (웹) | 30MB | 15MB | 0MB | 45MB |
| Phase 2 (Unity) | 80MB | 40MB | 60MB | 180MB |
| Phase 3 (UE5) | 200MB | 100MB | 400MB | 700MB |
