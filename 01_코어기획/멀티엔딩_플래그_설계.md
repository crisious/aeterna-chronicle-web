## 개요

에테르나 크로니클의 4가지 멀티 엔딩은 플레이어가 게임 전반에 걸쳐 내린 선택들이 **플래그(Flag)** 형태로 누적되어 최종 분기를 결정한다. 이 문서는 다음을 정의한다.

1. 전체 플래그 목록 및 초기값
2. 챕터별 선택지와 플래그 변화량 매핑
3. 엔딩 분기 판정 알고리즘
4. 데이터베이스 저장 구조
5. UI 피드백 설계

---

## 1. 핵심 플래그 체계

### 1.1 플래그 정의

| 플래그 키 | 타입 | 초기값 | 최대값 | 설명 |
|-----------|------|--------|--------|------|
| `story.fragment_count` | int | 0 | 4 | 수집한 신성 기억 파편 수 (메인 스토리 핵심) |
| `story.party_alive` | int (비트마스크) | 0 | 63 | 동료 생존 여부. 비트 0~5 각각 동료 대응 |
| `story.betrayal_score` | int | 0 | 100 | 베르나르도 설득 / 레테 교단 공감 누적치 |
| `story.artifact_count` | int | 0 | 12 | 수집한 신성 유물 수 (히든 엔딩 조건) |
| `story.seal_visited` | int (비트마스크) | 0 | 63 | 방문한 봉인 유적 수. 비트 0~5 각각 유적 대응 |
| `story.ending_a_score` | int | 0 | 100 | 엔딩 A(최선) 진행도 누적 포인트 |
| `story.emperor_saved` | bool | false | — | 황제 레나르도 5세 구원 여부 |
| `story.lethe_understood` | bool | false | — | 레테의 진심(기억되고 싶었어)을 이해했는가 |

---

### 1.2 동료 비트마스크 정의 (`story.party_alive`)

| 비트 위치 | 동료 | 값 (해당 비트 ON) |
|-----------|------|-----------------|
| 비트 0 | 세라핀 (엘파리스 정찰병) | 1 |
| 비트 1 | 벤자민 크로스 (형사) | 2 |
| 비트 2 | 이그나 (이프리타) | 4 |
| 비트 3 | 우르그롬 (우르살리스 장로) | 8 |
| 비트 4 | 레이나 (교단 이탈자) | 16 |
| 비트 5 | 마에스트로 크리오 (정보상) | 32 |

- **전원 생존** = `story.party_alive == 63`
- **3명 이상 생존** = `popcount(story.party_alive) >= 3`

---

### 1.3 봉인 유적 비트마스크 (`story.seal_visited`)

| 비트 위치 | 유적 | 위치 |
|-----------|------|------|
| 비트 0 | 에레보스 봉인 유적 | 챕터 1 — 기억의 골렘 처치 후 |
| 비트 1 | 실반헤임 봉인 유적 | 챕터 2 — 말라투스 뿌리 아래 |
| 비트 2 | 솔리안 봉인 유적 | 챕터 3 — 라와르 봉인실 |
| 비트 3 | 북방 기억석 사원 | 챕터 4 — 우르그롬 동행 필요 |
| 비트 4 | 아르겐티움 지하 연구소 | 챕터 4 — 황제 기생 현장 |
| 비트 5 | 망각의 고원 제단 | 챕터 5 최종 — D 엔딩 전용 해금 |

## 2. 챕터별 선택지 — 플래그 매핑

### 챕터 1: 잊혀진 도시의 목격자

---

#### 선택지 1-A: 수호자단 합류 여부 (씬 1-2)

```
[배경] 세라핀이 수호자단 합류를 권유한다.

▶ A. "알겠어요. 함께 가겠습니다."  → 메인 루트 합류
   B. "혼자서도 할 수 있어요."      → 독자 행동 (서브 퀘스트 우선 탐색)
   C. "시간이 필요해요."             → 지연 (추가 대화 후 A 선택으로 수렴)
```

| 선택 | 플래그 변화 | 효과 |
|------|-----------|------|
| A (합류) | `ending_a_score +5` | 세라핀 신뢰도 초기값 40 |
| B (독자 행동) | 없음 | 세라핀 신뢰도 초기값 20, 독자 서브 퀘스트 해금 |
| C (지연) | 없음 | 추가 대화 → A로 수렴, 세라핀 신뢰도 초기값 30 |

> **설계 의도**: 합류 선택이 엔딩 A에 유리하지만 B도 나중에 따라잡을 수 있다. 독자 행동 루트는 마에스트로 크리오와의 조기 친밀도 보너스를 제공해 대안적 장점을 제시한다.

---

#### 선택지 1-B: 기억의 골렘 처치 후 에레보스 봉인 유적 처리

```
[배경] 기억의 골렘을 쓰러뜨린 후 유적 봉인실 발견.
       안에 기억이 봉인되어 있다.

▶ A. 봉인을 해제하여 기억을 해방시킨다.
   B. 봉인을 유지하고 수집만 한다.
```

| 선택 | 플래그 변화 | 효과 |
|------|-----------|------|
| A (해방) | `seal_visited |= 1`, `ending_a_score +10`, `fragment_count +1` | 에레보스 기억 유령들 안식 → 엔딩 A 진행도 적립 |
| B (유지 수집) | `seal_visited |= 1`, `fragment_count +1` | 추가 효과 없음 |

---

#### 선택지 1-C: 베르나르도와의 첫 만남 (수호자단 본부)

```
[배경] 베르나르도가 에리언에게 접근해 "과거에 집착하지 말라"고 조언한다.

▶ A. "과거를 기억해야 미래를 만들 수 있습니다."
   B. "...생각해볼게요."
   C. "맞는 말이에요. 앞으로만 보겠습니다."
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (반박) | `betrayal_score +0` (베르나르도 철학 거부) |
| B (중립) | `betrayal_score +5` |
| C (동의) | `betrayal_score +15` |

> **설계 의도**: 베르나르도의 철학에 공감할수록 엔딩 C(망각의 선택)로 유도된다. 플레이어는 이 대화가 중요한지 아직 알지 못한다.

---

### 챕터 2: 기억이 자라는 숲

---

#### 선택지 2-A: 말라투스 고목과의 기억 연결

```
[배경] 실반헤임 말라투스 고목이 에리언의 기억 공명을 감지한다.
       수천 년의 기억을 공유하려 한다. 위험하지만 파편에 다가설 수 있다.

▶ A. 말라투스의 기억 흐름에 완전히 열린다.  (위험, 파편 획득 + 추가 정보)
   B. 조심스럽게 일부만 공명한다.            (안전, 파편 획득)
   C. 거부하고 강제로 파편을 채취한다.        (파편 획득 but 세라핀 신뢰도 -20)
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (완전 개방) | `fragment_count +1`, `seal_visited |= 2`, `ending_a_score +15`, 솔리안 문명 추가 기억 단서 획득 |
| B (부분 공명) | `fragment_count +1`, `seal_visited |= 2` |
| C (강제 채취) | `fragment_count +1`, `seal_visited |= 2`, 세라핀 신뢰도 -20, 실반헤임 세력 친밀도 -30 |

---

#### 선택지 2-B: 케인(기억 사냥꾼 간부)과의 협상

```
[배경] 케인이 세라핀을 인질로 잡고 파편 반납을 요구한다.

▶ A. 파편을 넘기지 않고 구출 작전을 펼친다.
   B. 파편 1개를 넘기는 척 교란작전을 편다.  (스킬 필요: 기억 복사)
   C. 케인의 요구에 굴복하고 파편을 반납한다. (fragment_count -1, 세라핀 구출)
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (구출 작전) | 세라핀 신뢰도 +25, 벤자민 인정도 +10 (이 사건 소문이 퍼짐) |
| B (교란) | 세라핀 신뢰도 +15, 케인과의 대결 격화 |
| C (굴복) | `fragment_count -1`, 세라핀 신뢰도 -10, `betrayal_score +10` |

> **설계 의도**: C 선택은 베르나르도가 "보여? 포기하는 것이 때론 지혜야"라고 말하는 이벤트를 트리거한다.

---

### 챕터 3: 불꽃과 모래 속의 진실

---

#### 선택지 3-A: 라와르 처리 방식 (씬 3-4, 이미 시나리오에 명시)

```
[배경] 솔리안 마지막 왕 라와르. 3,000년간 봉인에 갇혀 분노로 굳어진 존재.

▶ A. [기억 루트] 전투 중 기억 공명 스킬 3회 사용 → 전투 스킵, 라와르 안식
   B. [전투 루트] HP를 0으로 만들어 제압 후 기억 부여
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (기억 루트) | `fragment_count +1`, `seal_visited |= 4`, `ending_a_score +15`, `lethe_understood` 단서 적립 |
| B (전투 루트) | `fragment_count +1`, `seal_visited |= 4`, `ending_a_score +5` |

> **시나리오 원문**: "엔딩 연동: [기억 루트] 선택 시 엔딩 A 진행도 +15" — 기존 설계 반영

---

#### 선택지 3-B: 이그나의 아버지 족장 이그리스 치료 방식

```
[배경] 족장의 기억이 기억 사냥꾼에게 의도적으로 탈취된 상태.
       에리언이 기억 공명으로 일부 복원 가능하지만 위험 부담이 있다.

▶ A. 모든 기억을 복원하려 시도한다. (에리언 HP 30% 소모, 완전 복원)
   B. 안전한 범위에서만 복원한다.   (부분 복원, 이그나 신뢰도 +20)
   C. 불가능하다고 판단, 유적에서 방법을 찾겠다고 제안. (이그나 동행 유도)
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (완전 복원) | 이그나 신뢰도 +35, 이프리타 세력 친밀도 +40, `ending_a_score +10` |
| B (부분 복원) | 이그나 신뢰도 +20, 이프리타 세력 친밀도 +20 |
| C (유적 탐색) | 이그나 신뢰도 +10, 챕터 3 메인 퀘스트 자연스럽게 연결 |

---

#### 선택지 3-C: 베르나르도의 제안 수락 여부 (챕터 3 말미)

```
[배경] 베르나르도가 은밀히 접촉. "파편 수집을 잠시 멈추고
       제국과 타협해 더 많은 사람을 살릴 수 있지 않겠냐"고 제안한다.

▶ A. "아니요. 파편 수집을 계속해야 합니다."
   B. "...당신 말도 일리는 있어요."
   C. "제국과의 타협을 진지하게 고려해볼게요."
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (거부) | `betrayal_score +0` |
| B (중립) | `betrayal_score +20` |
| C (수락 고려) | `betrayal_score +35` |

> **설계 의도**: 챕터 3에서 `betrayal_score`가 50을 넘으면 챕터 4에서 베르나르도의 배신 의지가 강화되는 이벤트 추가 트리거.

### 챕터 4: 제국의 심장과 배신자

---

#### 선택지 4-A: 벤자민 크로스 신뢰 결정

```
[배경] 벤자민이 레이나를 의심한다.
       에리언은 어느 쪽을 선택하는가.

▶ A. 벤자민을 믿는다. (레이나를 임시 감시)
   B. 레이나를 믿는다. (벤자민이 실망하여 잠시 이탈)
   C. 둘 다 직접 조사하겠다고 한다.
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (벤자민 신뢰) | 벤자민 신뢰도 +20, 레이나 신뢰도 -10, `party_alive` 안정 유지 |
| B (레이나 신뢰) | 레이나 신뢰도 +25, 벤자민 신뢰도 -15, 벤자민 이탈 이벤트 (임시 비트 OFF) |
| C (직접 조사) | 양쪽 신뢰도 +5, 추가 탐문 서브 퀘스트 해금 |

> **주의**: B 선택 시 벤자민이 '임시 이탈'하며 `party_alive`의 비트 1이 일시적으로 꺼진다. 챕터 4 중반에 화해 이벤트를 완료하면 복원된다. 화해를 놓치면 `party_alive` 영구 손실.

---

#### 선택지 4-B: 황제 레나르도 5세 대면

```
[배경] 황제와 대면. 레테에 기생당하고 있지만 의식이 일부 남아있다.
       황제가 "모든 것을 잊고 싶었다"고 고백한다.

▶ A. "황제 폐하, 아직 싸울 수 있습니다. 기억을 되찾아 드리겠습니다."
   B. "이해합니다. 하지만 그 방법은 틀렸습니다."
   C. "...그 고통을 짊어지면서까지 싸워야 하는 이유가 있나요?"
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (구원 의지) | `emperor_saved` 플래그 가능성 ON, `ending_a_score +10` |
| B (이해+반박) | `ending_a_score +5`, `lethe_understood` 단서 적립 |
| C (공감) | `betrayal_score +20`, `lethe_understood` 단서 적립 |

---

#### 선택지 4-C: 베르나르도 배신 발각 후 처리

```
[배경] 베르나르도가 딸 밀리아를 살리기 위해 레테 교단에 정보를 넘겼음이 드러난다.

▶ A. "배신은 용서할 수 없지만, 이유는 이해합니다. 함께 다른 방법을 찾겠습니다."
   B. "동료를 배신했어요. 당신을 믿을 수 없습니다." (베르나르도 추방)
   C. "...밀리아를 살릴 방법이 레테 외에 없다면, 당신 선택이 옳을 수도 있어요."
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (이해+동행) | `ending_a_score +15`, 베르나르도 마지막 동행 가능, `betrayal_score -10` (리셋 효과) |
| B (추방) | 베르나르도 NPC 비활성화, 이후 등장 없음 |
| C (공감 동의) | `betrayal_score +40`, 엔딩 C 확정 임박, 레이나 신뢰도 -30 |

> **설계 의도**: 누적 `betrayal_score >= 70`이고 C를 선택하면 챕터 5에서 엔딩 C 자동 확정 대화가 트리거된다.

---

#### 선택지 4-D: 4번째 신성 기억 파편 수집 (북방 기억석 사원)

```
[배경] 우르그롬의 안내로 기억석 사원 도달.
       사원 내부는 기억 역전 현상으로 에리언의 과거 기억이 역류한다.

▶ A. 기억 역류를 받아들이고 전생(카일)의 기억을 완전히 복원한다.
   B. 기억 역류를 차단하고 파편만 수집한다.
```

| 선택 | 플래그 변화 |
|------|-----------|
| A (기억 복원) | `fragment_count +1`, `seal_visited |= 8`, `ending_a_score +15`, `lethe_understood` 완전 획득 |
| B (차단 수집) | `fragment_count +1`, `seal_visited |= 8` |

> **특이사항**: A를 선택하면 에리언이 카일로서 레테와 봉인 의식을 진행했던 기억을 완전히 복원한다. 이것이 `lethe_understood = true`를 완성시키는 핵심 순간이다.

## 3. 엔딩 분기 판정 알고리즘

### 3.1 판정 순서 (우선순위 높은 순)

```
[판정 시점: 선택지 5-B 직후]

1. 엔딩 D 조건 체크 (최우선)
   if (선택지 5-B == D) AND (artifact_count == 12) AND (popcount(seal_visited) >= 6):
       → 엔딩 D (신들의 귀환)

2. 엔딩 C 조건 체크
   if (선택지 5-B == C) OR (betrayal_score >= 70):
       → 엔딩 C 최종 확인 선택지 제시
       if 최종 확인 == "예":
           → 엔딩 C (망각의 선택)
       else:
           → 엔딩 B로 강제 이동

3. 엔딩 A 조건 체크
   if (선택지 5-B == A) AND (fragment_count >= 4) AND (party_alive == 63) AND (ending_a_score >= 60):
       → 엔딩 A (기억의 수호자)

4. 나머지 → 엔딩 B (기본값)
   조건: 위 1~3 해당 없음, fragment_count >= 3
```

---

### 3.2 엔딩별 최종 조건 요약

| 엔딩 | 조건 | 주요 결정 요소 |
|------|------|--------------|
| **A: 기억의 수호자** | `fragment_count == 4` AND `party_alive == 63` AND `ending_a_score >= 60` AND `선택지 5-B == A` | 전원 구원 + 라와르/말라투스 기억 루트 + 레테 이해 |
| **B: 마지막 증인의 선택** | `fragment_count >= 3` AND 위 조건 미충족 | 기본 정통 엔딩 |
| **C: 망각의 선택** | `betrayal_score >= 70` OR `선택지 5-B == C` 확정 | 베르나르도 철학 동조 누적 |
| **D: 신들의 귀환** | `artifact_count == 12` AND `popcount(seal_visited) >= 6` AND `선택지 5-B == D` | 히든 루트 전체 완료 |

## 5. 엔딩 콘텐츠 상세

### 5.1 엔딩 A: 기억의 수호자 (최선 엔딩)

**조건 충족 시**: `fragment_count == 4`, `party_alive == 63`, `ending_a_score >= 60`, `lethe_understood == true`

**스토리**:
에리언이 레테의 진심(기억되고 싶었어)을 이해했음을 선언한다. "당신을 기억하겠다"는 말에 레테의 망각의 의지가 흔들린다. 에리언이 기억 공명으로 레테의 200년 된 고통을 공유하며, 레테가 스스로 안식을 선택한다. 기억 소멸 증후군이 멈추고 세계가 치유되기 시작한다. 에리언은 기억을 유지한 채, 동료 전원이 살아남은 상태로 에필로그를 맞이한다.

**에필로그**:
- 에레보스에 새 식물이 자라기 시작하는 장면
- 세라핀이 실반헤임에서 처음으로 웃는 장면
- 벤자민이 아내의 이름을 기억하는 장면 (아내가 살아있는 경우)
- 이그나의 아버지 이그리스가 딸의 이름을 부르는 장면
- 마에스트로 크리오가 에레보스를 재건하는 장면

---

### 5.2 엔딩 B: 마지막 증인의 선택 (정통 엔딩)

**조건**: 엔딩 A 미충족, `fragment_count >= 3`

**스토리**:
레테를 설득할 수 없음을 깨닫고, 에리언이 자신의 모든 기억을 희생해 레테를 봉인한다. 에리언은 기억을 잃지만 세계는 구원된다. 동료들은 에리언을 기억하지만 에리언은 동료들을 기억하지 못한다. "기억해줘서 고마워"라는 에리언의 마지막 말.

**에필로그 분기**:
- 생존 동료 수에 따라 에필로그 분량 달라짐
- 모든 동료 생존 시: 동료들이 에리언을 이름 없이 "기억의 수호자"로 기억하는 기념비 건립
- 일부 동료만 생존 시: 더 쓸쓸한 에필로그, "당신의 희생을 기억한다" 홀로 독백

---

### 5.3 엔딩 C: 망각의 선택 (비극 엔딩)

**조건**: `betrayal_score >= 70` AND 최종 선택지에서 확정

**스토리**:
에리언이 레테의 철학에 공감하고 손을 잡는다. "고통을 가진 기억보다 평화로운 망각이 낫다." 두 사람이 함께 선택적 망각 시스템을 구축한다. 사람들은 자신이 원하는 기억만 남기고 나머지를 지울 수 있게 된다. 그러나 그 자유로 인해 세계는 역사를 잃어가고, 서서히 빈 껍데기가 된다.

**에필로그**:
- 50년 후. 에테르나에 전쟁이 없다. 분쟁도 없다. 그러나 음악도, 예술도, 오래된 우정도 사라졌다.
- 레이나(생존 시)가 홀로 기억을 지우지 않고 버텨내며 "기억이 고통이더라도"를 되뇌는 마지막 장면.

---

### 5.4 엔딩 D: 신들의 귀환 (히든 엔딩)

**조건**: `artifact_count == 12` AND `popcount(seal_visited) >= 6` AND `선택지 5-B == D`

**히든 콘텐츠 조건**:

| 신성 유물 | 획득 방법 |
|----------|---------|
| 신성 기억 파편 4개 | 챕터 1~4 메인 스토리 |
| 미네르바의 봉인석 | 실반헤임 히든 던전 (세라핀 신뢰도 80+ 필요) |
| 이프리타 불꽃 성배 | 이그나 신뢰도 80+ 후 솔라리스 히든 지역 해금 |
| 우르살리스 기억구 | 우르그롬 동행 + 북방 빙원 히든 지역 |
| 라와르의 황금 왕관 | 챕터 3 라와르 기억 루트 완료 시 드롭 |
| 솔리안 에테르 결정 왕홀 | 솔리안 유적 전체 탐사 완료 서브 퀘스트 보상 |
| 에레보스의 마지막 등불 | 에레보스 히든 구역 탐사 완료 |
| 카일의 봉인 서적 | 세계관 외전 퀘스트 완료 (챕터 2~3 사이) |
| 레테의 창세 이전 기억편린 | 챕터 5 직전 망각의 고원 단독 방문 |

**스토리**:
에리언이 열두 신의 이름을 모두 복원하고 봉인을 해제한다. 레테를 포함한 열두 신 전부가 자유를 되찾는다. 레테가 처음으로 "기억된다는 것이 이런 느낌이었구나"라고 말한다. 에리언은 신과 인간의 중재자가 된다.

**에필로그**: "에테르나 크로니클 2"를 암시하는 장면. 새로운 위협의 그림자.

## 6. 데이터베이스 저장 구조

```sql
-- 스토리 플래그 테이블
CREATE TABLE story_flags (
  character_id     UUID NOT NULL REFERENCES characters(id),
  fragment_count   SMALLINT NOT NULL DEFAULT 0 CHECK (fragment_count BETWEEN 0 AND 4),
  party_alive      SMALLINT NOT NULL DEFAULT 0 CHECK (party_alive BETWEEN 0 AND 63),
  betrayal_score   SMALLINT NOT NULL DEFAULT 0 CHECK (betrayal_score BETWEEN 0 AND 100),
  artifact_count   SMALLINT NOT NULL DEFAULT 0 CHECK (artifact_count BETWEEN 0 AND 12),
  seal_visited     SMALLINT NOT NULL DEFAULT 0 CHECK (seal_visited BETWEEN 0 AND 63),
  ending_a_score   SMALLINT NOT NULL DEFAULT 0 CHECK (ending_a_score BETWEEN 0 AND 100),
  emperor_saved    BOOLEAN NOT NULL DEFAULT FALSE,
  lethe_understood BOOLEAN NOT NULL DEFAULT FALSE,
  current_chapter  SMALLINT NOT NULL DEFAULT 1,
  ending_achieved  CHAR(1) CHECK (ending_achieved IN ('A','B','C','D')),
  updated_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (character_id)
);

-- NPC 신뢰도 테이블
CREATE TABLE npc_trust (
  character_id UUID NOT NULL REFERENCES characters(id),
  npc_id       VARCHAR(20) NOT NULL,
  trust_value  SMALLINT NOT NULL DEFAULT 0 CHECK (trust_value BETWEEN 0 AND 100),
  is_alive     BOOLEAN NOT NULL DEFAULT TRUE,
  PRIMARY KEY (character_id, npc_id)
);

-- 선택지 이력 테이블 (분석 및 디버깅용)
CREATE TABLE choice_history (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  character_id  UUID NOT NULL REFERENCES characters(id),
  choice_id     VARCHAR(10) NOT NULL,  -- 예: '1-A', '3-A', '5-B'
  selected      CHAR(1) NOT NULL,      -- 'A', 'B', 'C', 'D'
  chapter       SMALLINT NOT NULL,
  chosen_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_story_flags_chapter ON story_flags(current_chapter);
CREATE INDEX idx_choice_history_char ON choice_history(character_id, chapter);
```

### 7.1 선택지 표시 규칙

- 선택지는 대화 박스 아래 **최대 4개** 표시
- 각 선택지는 키보드 단축키 **1~4** 또는 마우스 클릭
- 엔딩 D 전용 선택지는 **[히든]** 태그 없이 표시 — 플레이어가 조건 충족 여부를 스스로 알아야 함
- 선택 후 **2초** 간 선택 내용 하이라이트 후 대화 진행

### 7.2 플래그 시각화 (선택지 후 비밀 피드백)

- **엔딩 A 진행도**: 플레이어에게 직접 수치 미공개. 대신 세라핀이 "이 결정이 옳은 것 같아"처럼 대사로 간접 피드백
- **betrayal_score**: 베르나르도가 점점 더 자주 "당신도 느끼지 않나요"류 대사를 건네는 빈도로 간접 피드백
- **party_alive 손실 시**: 해당 동료의 테마 음악이 조용히 페이드아웃되는 연출

### 7.3 뉴 게임 플러스 (NG+)

NG+ 시작 시:
- `ending_achieved`가 DB에 기록되어 어떤 엔딩을 달성했는지 저장
- NG+ 플레이어에게 이전 회차 엔딩에 따른 초반 대사 변화 적용
  - 엔딩 A 달성 후 NG+: 에리언이 세라핀을 보며 "...왠지 낯설지 않은 얼굴이에요" 대사
  - 엔딩 B 달성 후 NG+: "이번엔... 다르게 해보고 싶어" 독백 추가
- `betrayal_score`와 `ending_a_score`는 0으로 리셋
- `party_alive`는 63(전원)으로 리셋
