## 1. 언어 우선순위 및 로드맵

### 1.1 언어 계층

| 계층 | 언어 | 코드 | Phase | 번역 방식 |
|------|------|------|-------|---------|
| 1차 (기본) | 한국어 | `ko` | Phase 1~ | 원문 (내부 작성) |
| 2차 | 영어 | `en` | Phase 1~ | 내부 + 외주 |
| 2차 | 일본어 | `ja` | Phase 2~ | 외주 + 내부 감수 |

### 1.2 Phase별 로드맵

| Phase | 지원 언어 | 번역 범위 |

## 2. 번역 파이프라인

### 2.1 파이프라인 흐름

```
[1단계: 원문 작성 (ko)]
    기획팀 → ko 원문 확정 → L10N 키 등록
         ↓
[2단계: 컨텍스트 패키지 준비]
    키 + 원문 + 스크린샷 + 용어집 + 스타일 가이드
         ↓
[3단계: 번역]
    ┌─ 영어(en): 내부 번역 (1차) → 외주 네이티브 감수 (2차)
    ├─ 일본어(ja): 외주 번역 → 내부 게임 문맥 감수
    └─ 중국어: 외주 번역 → AI 보조 검증 → 네이티브 감수
         ↓
[4단계: 통합 및 검증]
    L10N 키 정합성 러너 실행 → 누락/오류 검출
         ↓
[5단계: 인게임 QA]
    각 언어별 플레이스루 → UI 오버플로우/잘림 확인
         ↓
[6단계: 릴리즈]
```

### 2.2 번역 도구

| 도구 | 용도 |
|------|------|
| Crowdin / Lokalise | L10N 관리 플랫폼 (키 관리, 번역 메모리, 용어집) |
| GitHub Actions | L10N 키 정합성 자동 검증 (PR 체크) |
| DeepL / GPT-4 | AI 초벌 번역 → 인간 감수 (비용 절감) |
| 인게임 L10N 테스터 | 언어 전환 시 실시간 UI 검증 도구 |

### 2.3 번역 일정 기준

| 볼륨 | 예상 소요 (언어당) | 비고 |
|------|-----------------|------|
| Phase 1 (~15,000 어절) | 2주 | 내부 영어 번역 |
| Phase 2 추가분 (~100,000 어절) | 6~8주 | 외주 병행 |
| Phase 3 추가분 (~90,000 어절) | 6~8주 | 4개 언어 병행 |

## 3. L10N 키 관리 규칙

### 3.1 키 명명 규칙

```
형식: {domain}.{category}.{subcategory}.{identifier}

예시:
  ui.menu.main.new_game         → "새 게임"
  ui.hud.hp_bar.label           → "HP"
  quest.main.ch1.q01.title      → "잊혀진 도시의 목격자"
  quest.main.ch1.q01.desc       → "에레보스에서 첫 번째 신성 기억 파편을 수집하라"
  quest.sub.memory.q01.title    → "어머니의 얼굴"
  item.weapon.sword.ether_blade.name  → "에테르 블레이드"
  item.weapon.sword.ether_blade.desc  → "에테르 에너지가 깃든 기사의 검"
  npc.seraphin.greeting.first   → "당신이 에리언인가요?"
  npc.seraphin.battle.skill_01  → "나무의 힘을 빌려!"
  system.error.connection_lost  → "서버 연결이 끊어졌습니다"
  achievement.memory_restorer.title → "역사의 복원자"
  class.memory_weaver.name          → "기억술사"
  class.memory_weaver.skill.fireball.name → "파이어볼"
```

### 3.2 도메인 목록

| 도메인 | 설명 |
|--------|------|
| `ui` | 인터페이스 텍스트 (메뉴, HUD, 설정) |
| `quest` | 퀘스트 제목, 설명, 대화 |
| `item` | 아이템 이름, 설명, 플레이버 텍스트 |
| `npc` | NPC 대화, 전투 대사, 상점 대사 |
| `system` | 시스템 메시지, 에러, 알림 |
| `achievement` | 업적, 칭호 |
| `class` | 클래스, 스킬, 스탯 |
| `lore` | 세계관 용어, 도감, 연대표 |
| `tutorial` | 튜토리얼 텍스트 |
| `season` | 시즌 콘텐츠 (시즌 패스, 이벤트) |

### 3.3 키 관리 규칙

| 규칙 | 설명 |
|------|------|
| 단일 소스 | 한국어(ko) JSON이 마스터. 다른 언어 파일은 ko 키와 1:1 매핑 |
| 키 삭제 금지 | 사용하지 않는 키는 `@deprecated` 태그 후 3버전 유지 → 이후 정리 |
| 변수 표기 | `{variable_name}` 형식, 모든 언어에서 동일 변수명 사용 |
| 복수형 | `{count}` 변수 사용 시 `_one` / `_other` 접미사 지원 (ICU MessageFormat) |
| 컨텍스트 주석 | 각 키에 `@context` 주석으로 사용 위치/상황 기재 |
| 최대 길이 | UI 키는 `@maxLength` 주석으로 표시 가능 문자수 명시 |

### 3.4 파일 구조

```
src/
  locales/
    ko/
      ui.json
      quest.json
      item.json
      npc.json
      system.json
      achievement.json
      class.json
      lore.json
      tutorial.json
      season.json
    en/
      (동일 구조)
    ja/
      (동일 구조)
    zh-Hans/
      (동일 구조)
    zh-Hant/
      (동일 구조)
    _glossary.json        ← 용어집 (전 언어 공통 참조)
    _style_guide.json     ← 스타일 가이드 메타데이터
```

### 3.5 L10N 키 정합성 러너 (P2-08 참조)

```
검증 항목:
  1. 키 누락: ko에 있는 키가 다른 언어에 없는 경우 → ERROR
  2. 미사용 키: 코드에서 참조되지 않는 키 → WARNING
  3. 변수 불일치: ko의 {var}가 번역에서 누락/변경 → ERROR
  4. 최대 길이 초과: @maxLength 기준 초과 번역 → WARNING
  5. 빈 값: 키는 존재하나 번역이 비어있음 → ERROR
  6. 인코딩: UTF-8 BOM 검사 → ERROR (BOM 존재 시)
  7. 중복 키: 동일 파일 내 중복 키 → ERROR

실행: GitHub Actions PR 체크 + 주 1회 전수 검사
출력: JSON 리포트 + Slack 알림 (ERROR 발생 시)
```

## 4. 문화적 로컬라이징

### 4.1 고유명사 로컬라이징 정책

| 카테고리 | 정책 | 예시 |
|---------|------|------|
| 세계관 고유명사 | 음차 + 의미 보조 | 에테르나(Aeterna), 레테(Lethe) → 전 언어 동일 라틴 표기 유지 |
| 캐릭터 이름 | 원어 유지 (발음 표기만 조정) | 에리언 → Eryan (en) / エリアン (ja) / 艾利安 (zh) |
| 종족명 | 의미 번역 + 원어 병기 | 엘파리스 → Elpharis (전 언어 동일) |
| 지역명 | 원어 유지 | 아르겐티움 → Argentium (전 언어 동일 라틴 표기) |
| 일반 NPC명 | 문화권 적합 이름 | 한국 NPC "김형사" → 영어 "Detective Kim" (성은 유지) |
| 아이템명 | 의미 번역 | "에테르 블레이드" → "Ether Blade" / "エーテルブレイド" / "以太之刃" |
| 음식/문화 아이템 | 문화권 적합 조정 | 게임 내 음식이 특정 문화 한정 시 → 해당 문화 등가물 사용 |

### 4.2 문화적 주의사항

| 대상 시장 | 주의 항목 | 대응 |
|---------|---------|------|
| 일본 (ja) | 종교적 상징 민감도 (신 명칭/교단) | "열두 신"은 유지, 교단 묘사 시 실제 종교와 거리 확인 |
| 일본 (ja) | 경어 체계 | NPC 대화의 존대/반말 일본어 맥락 재조정 |
| 중국 (zh) | 해골/유령 등 시각 표현 규제 | 기억 유령의 시각 표현 중국 버전 별도 에셋 검토 |
| 중국 (zh) | 역사 왜곡 민감도 | 가상 세계관이므로 직접 문제 없으나, 제국/식민 서사 중립 표현 확인 |
| 영어 (en) | 젠더 중립 표현 | 플레이어 호칭은 젠더 중립 (they/them 옵션) |
| 전체 | 색상 의미 차이 | 빨강=위험(보편적), 그러나 축하(중국)와 충돌 시 컨텍스트 명확화 |

## 5. 텍스트 확장률 대응

### 5.1 언어별 텍스트 확장률

한국어를 기준(100%)으로:

| 언어 | 평균 확장률 | 최대 확장률 | 주요 원인 |
|------|-----------|-----------|---------|
| 영어 (en) | 120~130% | 150% | 알파벳 단어 길이, 관사/전치사 |
| 일본어 (ja) | 90~110% | 130% | 한자 압축 vs 히라가나 확장 |
| 중국어 간체 (zh-Hans) | 80~100% | 120% | 한자 압축 효과 |
| 중국어 번체 (zh-Hant) | 85~105% | 125% | 간체와 유사, 약간 길어짐 |

### 5.2 UI 여백 설계 규칙

| UI 요소 | 여백 규칙 | 적용 방법 |
|---------|---------|---------|
| 버튼 텍스트 | 최대 확장률 150% 기준 여백 확보 | `min-width` 설정, 텍스트 오버플로우 시 `...` 처리 |
| 대화 창 | 최대 4줄 기준, 초과 시 스크롤 | 고정 높이 + 스크롤바 |
| 아이템 이름 | 최대 20자(ko) / 30자(en) | 초과 시 2줄 표시, 툴팁 전체 표시 |
| 알림 토스트 | 최대 2줄 | 초과 시 `...` + 클릭으로 전체 표시 |
| 퀘스트 제목 | 최대 15자(ko) / 25자(en) | 목록에서는 말줄임, 상세 뷰에서 전체 표시 |
| 스킬명 | 최대 8자(ko) / 15자(en) | 퀵슬롯 내 아이콘 + 줄임 표시 |

### 5.3 폰트 대응

| 언어 | 기본 폰트 | 폴백 | 비고 |
|------|---------|------|------|
| ko | Noto Sans KR | 맑은 고딕 | WebFont (Phase 1) |
| en | Rajdhani (수치) / Cinzel (아이템) | Arial | — |
| ja | Noto Sans JP | メイリオ | CJK 통합 가능 |
| zh-Hans | Noto Sans SC | 微软雅黑 | — |
| zh-Hant | Noto Sans TC | 微軟正黑體 | — |

**Phase별 폰트 로딩:**
- Phase 1 (웹): Google WebFont CDN, 필요 언어만 로드
- Phase 2 (Unity): TextMeshPro SDF 폰트, 언어별 에셋 번들 분리 (Addressables)
- Phase 3 (UE5): Slate 텍스트 렌더러, 언어별 폰트 에셋 자동 전환

## 6. 품질 보증

### 6.1 번역 품질 기준

| 등급 | 기준 | 적용 범위 |
|------|------|---------|
| A (최고) | 원어민 수준, 문학적 품질 | 메인 스토리 대사, 엔딩 씬, 핵심 NPC 대화 |
| B (높음) | 정확하고 자연스러운 | 서브 퀘스트, 일반 NPC 대화, 아이템 설명 |
| C (기능적) | 의미 전달 정확, 약간의 부자연스러움 허용 | UI 텍스트, 시스템 메시지, 툴팁 |

### 6.2 검증 프로세스

| 단계 | 검증자 | 검증 항목 |
|------|--------|---------|
| 1차 | 자동 러너 | 키 정합성, 변수 일치, 길이 초과, 인코딩 |
| 2차 | 내부 감수자 | 용어 일관성, 게임 문맥 적합성, 톤/스타일 |
| 3차 | 외부 네이티브 | 자연스러움, 문법, 문화 적합성 |
| 4차 | QA 플레이테스트 | 인게임 UI 적합성, 잘림/오버플로우, 맥락 확인 |

### 6.3 번역 업데이트 사이클

| 이벤트 | 대응 |
|--------|------|
| 새 콘텐츠 추가 | ko 원문 확정 → 48시간 내 en 번역 → 1주 내 ja/zh |
