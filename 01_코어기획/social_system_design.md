# 에테르나 크로니클 — 소셜 기능

> **작성일:** 2026-03-10 | **버전:** v1.0 | **분류:** 코어기획 — 소셜

---

## 1. 개요

본 문서는 에테르나 크로니클의 소셜 기능을 정의한다. 친구, 차단, 신고, 채팅, 우편, 파티 시스템의 규칙과 기술 구현을 포함한다.

### 1.1 설계 원칙

| 원칙 | 설명 |
|------|------|
| **안전한 커뮤니티** | 악용/비매너 차단 도구를 유저에게 충분히 제공 |
| **간편한 소통** | 파티/친구 찾기에 최소한의 단계 |
| **프라이버시 존중** | 유저가 자신의 노출 범위를 직접 제어 |
| **비동기 지원** | 오프라인 유저와도 우편/메시지 교환 가능 |

## 2. 친구 시스템

### 2.1 기본 기능

| 기능 | 설명 |
|------|------|
| 친구 추가 | 캐릭터명 검색 → 친구 요청 발송 |
| 요청 수락/거절 | 알림 팝업 + 소셜 메뉴에서 관리 |
| 친구 삭제 | 즉시 양방향 삭제 (상대에게 알림 없음) |
| 친구 상한 | 100명 |
| 즐겨찾기 | 상위 10명 즐겨찾기 (목록 최상단 고정) |

### 2.2 온라인 상태

| 상태 | 아이콘 | 설명 |
|------|--------|------|
| 🟢 온라인 | 초록 원 | 현재 접속 중 |
| 🟡 자리비움 | 노랑 원 | 접속 중이나 10분 이상 무입력 |
| 🔴 방해금지 | 빨강 원 | 수동 설정. 알림 차단 |
| ⚪ 오프라인 | 회색 원 | 미접속 |
| 👻 숨김 | 회색 원 (오프라인과 동일) | 접속 중이나 오프라인으로 표시 |

**프라이버시 설정:**

| 설정 | 옵션 | 기본값 |
|------|------|--------|
| 온라인 상태 공개 | 전체/친구만/숨김 | 친구만 |
| 현재 위치 공개 | 전체/친구만/비공개 | 친구만 |
| 최근 접속 표시 | ON/OFF | ON |
| 친구 요청 수신 | 전체/친구의친구/비공개 | 전체 |

### 2.3 최근 접속 정보

- 오프라인 친구: "최근 접속: N분/시간/일 전" 표시
- 7일 이상 미접속: "7일 이상 전" 고정 (정확한 날짜 미공개)
- 30일 이상 미접속: "장기 미접속" 표시

## 3. 차단 시스템

### 3.1 차단 기능

| 항목 | 설명 |
|------|------|
| 차단 방법 | 캐릭터 우클릭 → 차단 / 소셜 메뉴 → 차단 목록에서 추가 |
| 차단 해제 | 소셜 메뉴 → 차단 목록에서 해제 |
| 차단 상한 | 200명 |
| 알림 | 차단 시 상대에게 알림 없음 |

### 3.2 상호작용 제한 범위

차단된 유저와의 상호작용이 다음과 같이 제한된다:

| 기능 | 차단 시 동작 |
|------|-------------|
| **채팅 (모든 채널)** | 차단 대상의 메시지 숨김 |
| **귓속말** | 발송/수신 불가 |
| **파티 초대** | 불가 (쌍방) |
| **파티 자동매칭** | 같은 파티 매칭 제외 |
| **거래** | 거래 요청 불가 (쌍방) |
| **경매장** | 제한 없음 (익명 거래이므로) |
| **우편** | 발송 불가 (쌍방) |
| **PvP 매칭** | 가능 (매칭 제외 시 악용 우려) |
| **길드** | 같은 길드 시 채팅만 숨김, 기능은 제한 없음 |
| **친구 요청** | 불가 |
| **위치 공유** | 차단 대상에게 위치 비공개 |

## 4. 신고 시스템

### 4.1 신고 카테고리

| 코드 | 카테고리 | 설명 | 자동 처리 |
|------|----------|------|-----------|
| `RPT_01` | 욕설/비매너 | 욕설, 인신공격, 비아냥 | 비속어 필터 자동 감지 → 경고 |
| `RPT_02` | 혐오/차별 | 인종/성별/종교 등 혐오 발언 | 필터 + GM 우선 검토 |
| `RPT_03` | 핵/치트 | 스피드핵, 데미지핵, 월핵 등 | 서버 사이드 탐지 + GM 검토 |
| `RPT_04` | 오토/매크로 | 자동 사냥 프로그램 | 행동 패턴 탐지 + GM 검토 |
| `RPT_05` | 버그 악용 | 의도적 버그 활용 | GM 검토 |
| `RPT_06` | 현금 거래 | 게임 외 거래 (RMT) | 채팅 키워드 감지 + GM 검토 |
| `RPT_07` | 사칭 | GM/운영진 사칭 | GM 즉시 검토 |
| `RPT_08` | 기타 | 위 카테고리에 해당하지 않음 | GM 검토 |

### 4.2 신고 처리 플로우

```
[유저] 신고 접수
  │
  ├── 자동 필터 단계
  │   ├── 비속어 매칭 → 즉시 채팅 경고 (자동)
  │   ├── 핵 탐지 데이터 매칭 → 우선 큐 배정
  │   └── 중복 신고 체크 (동일 대상 5건+ → 우선 큐)
  │
  ├── GM 검토 큐
  │   ├── 우선순위: 혐오/치트/사칭 > 버그/RMT > 욕설/기타
  │   ├── 검토 SLA: 우선 24시간 / 일반 72시간
  │   └── 증거 확인: 채팅 로그, 행동 로그, 전투 데이터
  │
  ├── 판정
  │   ├── 무혐의 → 신고 종결
  │   ├── 유죄 → 제재 적용 (admin_tools_design.md 참조)
  │   └── 추가 조사 필요 → 에스컬레이션
  │
  └── 결과 알림
      ├── 신고자: "신고가 처리되었습니다" (상세 미공개)
      └── 피신고자: 제재 시에만 알림 (사유 포함)
```

### 4.3 악의적 신고 방지

| 방지 수단 | 설명 |
|-----------|------|
| 일일 신고 상한 | 5건/일 |
| 허위 신고 페널티 | 무혐의 3회 연속 시 경고, 5회 시 신고 기능 24시간 제한 |
| 동일 대상 반복 | 같은 대상 24시간 내 1회만 가능 |
| 증거 필수 | 핵/치트 신고 시 스크린샷/영상 첨부 권장 (미첨부 시 낮은 우선순위) |

## 5. 채팅 시스템

### 5.1 채팅 채널

| 채널 | 범위 | 쿨타임 | 레벨 제한 | 설명 |
|------|------|--------|-----------|------|
| **일반** | 주변 50m | 없음 | 없음 | 근처 유저와 대화 |
| **지역** | 현재 지역 전체 | 3초 | 10+ | 같은 지역 유저 전체 |
| **월드** | 서버 전체 | 10초 | 20+ | 서버 전체 공개 채팅 |
| **파티** | 파티원 | 없음 | 없음 | 파티 전용 |
| **길드** | 길드원 | 없음 | 없음 | 길드 전용 |
| **귓속말** | 1:1 | 1초 | 없음 | 개인 메시지 |
| **거래** | 서버 전체 | 30초 | 20+ | 거래 전용 (아이템 링크 지원) |

### 5.2 비속어 필터

| 기능 | 설명 |
|------|------|
| 사전 기반 | 한국어/영어 비속어 사전 (5,000+ 단어) |
| 변형 감지 | 초성 분리, 특수문자 삽입, 유사 문자 대체 감지 |
| 처리 방식 | `***`으로 마스킹 (발신자 본인에게는 원문 표시) |
| 단계적 제재 | 5회/시간 → 경고, 10회/시간 → 채팅 제한 10분, 20회/일 → 채팅 제한 1시간 |
| 커스텀 필터 | 유저 개인 필터 단어 추가 가능 (최대 50개) |
| 화이트리스트 | 아이템명/스킬명 등 게임 용어 오탐 방지 |

### 5.3 채팅 기능

| 기능 | 설명 |
|------|------|
| 아이템 링크 | 채팅에 아이템 정보 링크 삽입 (클릭 시 툴팁) |
| 위치 링크 | 현재 위치 공유 (클릭 시 미니맵 표시) |
| 이모지 | 기본 이모지 50종 + 게임 커스텀 이모지 30종 |
| 스티커 | 캐릭터 스티커 (상점 구매 / 이벤트 보상) |
| 채팅 매크로 | 자주 쓰는 문구 10개 등록 (Ctrl+1~0) |
| 채팅 탭 | 채널별 탭 분리 / 통합 뷰 선택 가능 |
| 채팅 로그 | 최근 500줄 로컬 보관 (닫기 전까지) |
| 타임스탬프 | ON/OFF 선택 (기본: OFF) |

## 6. 우편 시스템

### 6.1 기본 규칙

| 항목 | 값 |
|------|-----|
| 우편함 용량 | 100통 (가장 오래된 것부터 자동 삭제 경고) |
| 우편 만료 | 30일 (미수령 시 자동 삭제) |
| 시스템 우편 만료 | 90일 (보상/이벤트/GM 발송) |
| 아이템 첨부 | 최대 5개 / 통 |
| 골드 첨부 | 최대 1,000,000 골드 / 통 |
| 발송 비용 | 일반 우편 100골드, 아이템 첨부 시 500골드 |
| 수신 거부 | 시스템 우편 제외, 유저 우편 수신 거부 가능 |

### 6.2 우편 종류

| 유형 | 발신자 | 특징 |
|------|--------|------|
| **유저 우편** | 다른 유저 | 텍스트 + 아이템/골드 첨부 |
| **시스템 우편** | 시스템 | 이벤트 보상, 경매장 결과, 길드 공지 등 |
| **GM 우편** | 운영팀 | 보정 아이템, 사과 보상 등 |
| **거래 우편** | 경매장 시스템 | 낙찰 아이템/골드 배달 |

### 6.3 우편 악용 방지

| 방지 수단 | 설명 |
|-----------|------|
| 일일 발송 상한 | 30통/일 (시스템/GM 제외) |
| 레벨 제한 | 우편 발송 레벨 10+ |
| 골드 상한 | 단일 우편 100만 골드 (RMT 방지) |
| 귀속 아이템 | 첨부 불가 |
| 차단 유저 | 발송 불가 |

## 7. 파티 시스템

### 7.1 기본 구성

| 항목 | 값 |
|------|-----|
| 기본 파티 인원 | 4인 |
| 레이드 파티 인원 | 8인 (4+4 연합) |
| 파티장 | 생성자 또는 위임 |
| 파티 해산 | 파티장 해산 또는 전원 탈퇴 |

### 7.2 파티 모집

#### 모집 게시판

| 필드 | 설명 |
|------|------|
| 콘텐츠 | 던전/레이드/퀘스트/보스/자유 |
| 목표 | 구체적 목표 (예: "에레보스 던전 클리어") |
| 필요 역할 | 탱커 N명 / 딜러 N명 / 서포터 N명 |
| 최소 레벨 | 입장 조건 레벨 |
| 최소 전투력 | 파티장 설정 (선택) |
| 메모 | 자유 메모 (100자) |
| 자동 초대 | ON 시 조건 충족 신청자 자동 수락 |

**게시판 규칙:**
- 동시 등록: 1인 1게시 (기존 게시 자동 취소)
- 게시 유효기간: 30분 (갱신 가능)
- 레벨/전투력 미달 시 신청 불가
- 파티장이 오프라인 전환 시 게시 자동 취소

#### 자동매칭

| 항목 | 설명 |
|------|------|
| 매칭 기준 | 콘텐츠 + 레벨 범위 + 역할 |
| 역할 매칭 | 탱커 1 + 딜러 2 + 서포터/힐러 1 (4인 기본) |
| 대기 시간 | 평균 30초~2분 (역할별 편차) |
| 수락 시간 | 매칭 후 30초 내 수락 (미수락 시 재매칭) |
| 페널티 | 수락 거부 3회/시간 → 10분 매칭 금지 |

### 7.3 파티 기능

| 기능 | 설명 |
|------|------|
| 파티 채팅 | 파티 전용 채팅 채널 |
| 위치 공유 | 파티원 미니맵 표시 |
| HP 공유 | 파티원 HP/MP바 HUD 표시 |
| 전리품 분배 | 순서제/자유입찰/파티장분배 (파티장 설정) |
| 추방 투표 | 파티원 과반수 동의 시 추방 (파티장 포함 투표) |
| 파티장 위임 | 파티장 → 지정 파티원 |
| 자동 해산 | 전원 오프라인 30분 경과 시 |

### 7.4 전리품 분배 방식

| 방식 | 설명 | 적합 상황 |
|------|------|-----------|
| **순서제** | 드롭 순서대로 파티원에게 순환 지급 | 일반 던전 |
| **자유 입찰** | 아이템별 Need/Greed/Pass 투표. Need 우선 → 랜덤 | 레이드 |
| **파티장 분배** | 파티장이 직접 지정 | 고정 파티/길드 |

**입찰 규칙:**
- Need: 해당 클래스가 착용 가능한 아이템만 선택 가능
- Greed: 제한 없음
- 미선택 15초 경과: 자동 Pass
- 영웅 등급 이상: 입찰 투표 필수 (자동 분배 안 됨)

### 8.2 칭호 시스템

| 획득 경로 | 예시 |
|-----------|------|
| 메인 퀘스트 | "기억의 수호자", "망각을 걸은 자" |
| 업적 | "천 번의 전투", "수집광" |
| PvP 시즌 | "시즌 1 챔피언" |

## 10. 기술 구현 노트

### 10.1 채팅 아키텍처

```
[클라이언트] ←→ [WebSocket Gateway (Redis Pub/Sub)]
                    │
                    ├── 일반/지역/월드: Redis Channel (지역별 토픽)
                    ├── 파티/길드: Redis Channel (그룹별 토픽)
                    ├── 귓속말: Direct Message (P2P via 서버)
                    └── 비속어 필터: 메시지 전처리 미들웨어
```

- **WebSocket:** Socket.IO 또는 native WebSocket
- **메시지 큐:** Redis Pub/Sub (수평 확장)
- **메시지 저장:** 최근 1,000건 Redis 캐시 + PostgreSQL 아카이브
- **비속어 필터:** 서버 사이드 전처리 (Aho-Corasick 알고리즘)

### 10.2 우편 처리

- 우편 발송: PostgreSQL INSERT + 수신자 실시간 알림 (WebSocket)
- 첨부 아이템: 트랜잭션 내 발신자 인벤토리 차감 → 우편 아이템 테이블 INSERT
- 수령: 트랜잭션 내 우편 아이템 DELETE → 수신자 인벤토리 INSERT
- 만료 처리: 크론잡 (매일 04:00 UTC) → 만료 우편 일괄 삭제, 첨부 아이템 원 발신자에게 반환 (반환 불가 시 소멸)

### 10.3 매칭 큐

- Redis Sorted Set (MMR/전투력 기준 정렬)
- 매칭 워커: 1초 주기로 큐 스캔 → 역할 조건 충족 그룹 추출 → 매칭 제안
- 수락 타임아웃: 30초 (Redis TTL)
