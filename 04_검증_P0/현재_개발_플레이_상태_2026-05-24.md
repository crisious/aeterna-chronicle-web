## 2. 필드 NPC 대화 상태

| 항목 | 결과 |
|---|---|

| 자동 QA 선택지 수 | 108개 |

| 닫기 선택지 | 14개 |
| role 분포 | quest 23 / shop 15 / dialogue 6 / craft 1 / boss 1 |
| QA 실패 | 0 |
| 브라우저 콘솔 에러 | 0 |
| 결과 | PASS |

유령 상인 고로디 관련 회귀는 다음 3가지를 테스트로 고정했다.

- 고로디를 선택하면 대화 화자가 `유령 상인 고로디`로 표시되고 `누아리엘`이 표시되지 않는다.
- `희귀 물건을 얻는다.` 선택 시 획득 확인 대사를 반환한다.
- `거래한다` 선택 시 거래품/망각의 은화 안내 대사를 반환한다.

해당 선택지는 모두 최종적으로 `닫기` 선택지를 가진 응답 대화로 수렴한다. 알 수 없는 NPC 입력도 기본 대화로 폴백하므로 null/빈 문자열 입력에서 UI가 멈추지 않는다.

## 3. 현재 플레이 가능 범위

자동 플레이 로그 기준으로 재현된 흐름은 다음과 같다.

1. 홈 화면 부팅
2. 게임 시작 후 로그인/회원가입 패널 표시
3. QA 월드맵 진입
4. `E` 키 기반 시대 전환
5. 월드맵 노드 선택 및 지역 패널 표시
6. 필드 진입
7. 필드 몬스터 클릭
8. ATB 전투 HUD 진입

`.ac/play-logs/20260515-023142/play-log-final.md` 기준 API health는 200, 신규 브라우저 콘솔 에러는 0이다. 서버 측 근거로 API 서버 `:3000`, Redis 연결, `/api/world/zones/abyss_gate` 200 응답, Socket 연결이 기록되어 있다.

현재 판단: **월드맵 → 필드 → NPC 대화 → 몬스터 클릭 → ATB 진입의 핵심 플레이 루프는 자동 QA 근거상 작동 상태**다.

## 5. 남은 위험 및 다음 QA 대상

| 위험 | 상태 | 다음 조치 |
|---|---|---|
| 전체 `npm run verify` | 이번 문서 갱신 중 새로 실행하지 않음 | 릴리즈 전 lint/typecheck/test/build 전체 실행 |
| 장시간 플레이 안정성 | 최근 로그는 짧은 자동 플레이 중심 | 10~30분 연속 필드/전투/대화 반복 QA |
| save/load 왕복 | 이번 확인 대상 아님 | 저장 후 재접속, 필드 위치/퀘스트/NPC 상태 복원 확인 |
| 엔딩 풀 플로우 | 이번 확인 대상 아님 | 엔딩 A/B/C/D/FAIL 회귀 시나리오 재실행 |
| `getScenarioRegistryStats()` 최신성 | legacy 25 도메인 중심 helper | SYNC-101~105 runtime narrative 도메인 통계 반영 필요 |

이번 상태는 "플레이 불가"가 아니라, 자동 QA로 확인한 범위와 미확인 범위를 분리한 것이다. 다음 full QA에서는 `qa:auto:full` 또는 `npm run verify`를 기준으로 장시간 플레이와 저장/엔딩 경로를 함께 확인한다.
