## 4. SSOT wiring 챕터 (dead code 해소 시작)

204개 narrative SSOT 도메인(`shared/types/scenarioRegistry.ts`)이 그동안 전부 dead code였다. 10-에이전트 recon 워크플로로 8개 scene의 하드코딩 88지점을 분석하고 우선순위를 산정한 뒤 첫 sprint를 착수했다.

### 실제 SSOT ↔ 게임 연결 현황 (3곳)

| 연결 지점 | 도메인 | 비고 |
|---|---|---|
| `client/src/gameplay/npcDialogue.ts` | FIELD_NPC_DIALOGUE_TEMPLATES | sync-101 (기존) |
| `client/src/combat/escapeNarration.ts` | **ESCAPE_NARRATIVES** | **SSOT-WIRE-01 (신규)** |
| `server/src/quest/questSeeds.ts` | (서버 퀘스트) | 기존 |

> **중요 구조 사실**: 메모리/계획에 언급되던 `scenarioRegistryClient.ts`는 **미존재**. client→SSOT 표준은 직접 `../../../shared/types/scenarioRegistry` 상대경로 import (BattleScene이 이미 chronoEraAtb/dualTech/chronoField를 이 방식으로 사용 중).

### SSOT-WIRE-01 상세
- 신규 `escapeNarration.ts`: `composeEscapeLog(outcome)` / `escapeOutcomeFromResult(bool)` 순수 함수
- SSOT `getEscapeNarrative` label을 단일 출처로, emoji는 표현 계층(`ESCAPE_LOG_EMOJI`)에서 분리
- `BattleScene._attemptFlee()` 하드코딩 `'🏃 도주 성공!'`/`'❌ 도주 실패!'` 제거 → 화면 텍스트 100% 무가시 보존
- TDD RED→GREEN→REFACTOR, `tests/unit/escapeNarration.test.ts` 6 케이스
