## 1) 테스트 전역 규칙

- 판정 시점: **선택지 5-B 직후**
- 판정 우선순위: **D → C → A → B**
- 기본 변수:
  - `fragment_count` (0~4)
  - `party_alive` (bitmask, 전원 생존=63)
  - `betrayal_score` (0~100)
  - `artifact_count` (0~12)
  - `seal_visited` (bitmask, 6유적 방문=popcount 6)
  - `ending_a_score` (0~100)
  - `choice_5b` (A/B/C/D)

## 3) 실행 절차 (케이스 공통)

1. 테스트 세이브 로드 (케이스별 사전 플래그 세팅)
2. 챕터5 최종 선택지 `5-B` 도달
3. `choice_5b` 선택 및 확인 선택(해당 시) 수행
4. 결과 엔딩 ID/컷신 ID/저장된 `ending_achieved` 확인
5. 기대 결과와 비교 후 Pass/Fail 기록

---

## 4) 케이스별 검증 포인트

### TC-A-01 / TC-A-02 / TC-A-03
- A 조건 중 어떤 항목이 실패했는지 로그에 명시
- A 실패 시 반드시 B로 폴백되는지 확인

### TC-C-01 / TC-C-02 / TC-C-03
- C는 **직접 선택** 또는 **배신점수 70+ 강제 트리거** 모두 커버
- C 확인 NO 시 B 강제 이동 확인

### TC-D-01 / TC-D-02
- D는 최우선 판정인지 확인 (다른 조건보다 먼저 평가)
- D 실패 시 다음 우선순위 로직이 정상 작동하는지 확인

### TC-PR-01 / TC-PR-02
- 패배 루트가 정규 A/B/C/D와 분리되어 저장되는지 확인
- 재시도 시 스토리 플래그 오염(중복 가산/비정상 리셋) 여부 점검
