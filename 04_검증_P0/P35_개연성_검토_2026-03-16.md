## 이슈 2: 전투 키 입력

### 수정 내역
- Phaser config에 `input: { keyboard: true, mouse: true, touch: true }` 명시
- BattleScene 키보드 null-safe 처리 (`this.input.keyboard` 체크 강화)
- 마우스/터치 전투 액션 버튼 3개 추가: [공격] [타겟] [일시정지]

### 검증
- 🟢 `this.input.keyboard` null 체크 → 키보드 없는 환경에서도 crash 방지
- 🟢 마우스 버튼으로 전투 조작 가능 → 모바일/터치 환경 대응
- 🟡 주의: `_useSkill`은 private 메서드이나 BattleUI에서 `(scene as any)._useSkill(i)` 캐스팅으로 호출 중 — 기존 패턴 유지, 리팩토링은 별도 이슈

## 이슈 4: 초기 아이템 지급

### 수정 내역
- `characterRoutes.ts`에서 캐릭터 생성 후 `inventoryManager.addItem()` 호출
- 클래스별 초기 무기/방어구 매핑 추가

### 아이템 코드 정합성
| 클래스 | 무기 코드 | 방어구 코드 | itemSeeds 존재 |
|--------|-----------|-------------|---------------|
| ether_knight | WPN_SWORD_C | ARM_PLATE_C | ✅ / ✅ |
| memory_weaver | WPN_STAFF_C | ARM_ROBE_C | ✅ / ✅ |
| shadow_weaver | WPN_DAGGER_C | ARM_LEATHER_C | ✅ / ✅ |
| memory_breaker | WPN_SWORD_C | ARM_PLATE_C | ✅ / ✅ |
| time_guardian | WPN_STAFF_C | ARM_ROBE_C | ✅ / ✅ |
| void_wanderer | WPN_STAFF_C | ARM_LEATHER_C | ✅ / ✅ |

| 소비 아이템 | 코드 | 수량 | itemSeeds 존재 |
|-------------|------|------|---------------|
| HP 포션 (소) | CON_HP_S | 5 | ✅ |

- 🟢 모든 아이템 코드가 itemSeeds.ts에 정의됨
- 🟢 inventoryManager.addItem()은 userId + itemCode 기반 → 캐릭터 ID가 아닌 유저 ID 사용
- 🟡 주의: inventoryManager는 userId 기반이므로 동일 유저의 다중 캐릭터가 인벤토리를 공유함 — 기존 설계 의도 확인 필요 (별도 이슈)
