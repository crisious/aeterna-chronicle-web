## B. P17 접근성 검증

### 7. 색약 모드 — 3모드 팔레트 vs 스타일 가이드 9지역 팔레트

**결과: 🟢 정합**

- P17-11 색약 모드: Protanopia/Deuteranopia/Tritanopia 3종 ✅
- `accessibility_design.md` §2.1 등급 색상 매핑: 6등급 × 4모드(기본+3색약) 전수 정의 ✅
- "색상만으로 정보를 전달하지 않는다" 원칙 + 형태(아이콘/테두리/패턴) 보조 식별자 ✅
- 9지역 스타일 가이드 팔레트와 충돌 없음 — 색약 모드는 포스트프로세싱 셰이더 + UI 팔레트 오버라이드 방식이므로 지역 팔레트에 비파괴적

### 8. 키 리바인딩 — 26액션 매핑

**결과: 🟢 정합**

- P17-12: "이동(4), 전투(8), UI(6), 카메라(4), 시스템(4) = 26개" ✅
- `accessibility_design.md` §4.1 키보드 전용 플레이: 이동(WASD), 스킬(1~8), 소비(F1~F4), Active Pause, 타겟, NPC, 인벤, 캐릭, 지도, 메뉴(방향키/Tab), 확인, 취소 → 실제 나열 ~15개
- 차이는 accessibility_design이 "대표 키"만 기재했기 때문. P17-12의 26개 분류가 상세 버전으로 상위 호환 ✅

**결과: 🟢 정합**

### 10. UI 스케일링 — 고대비 vs 기존 UI 스킨 3테마

**결과: 🟡 Minor → ✅ 수정 완료**

- P17-14: "75%/100%/125%/150%/200% 5단계"

- **불일치**: 50% vs 75% 하한 차이
- **수정**: `accessibility_design.md` 하한을 75%로 통일 (50%는 실용적으로 너무 작음, P17-14 기준 채택)
- 고대비 7:1 ✅, 포커스 인디케이터 3px ✅, 기존 3테마(기본/다크/라이트)와 호환 ✅
