- `server/src/telemetry/dialogueTelemetryPersistence.ts` (신규)
  - Redis 영속 저장:
    - `telemetry:dialogue_choice:{eventId}` 해시 저장 + TTL 60일

    - `telemetry:dialogue_choice:counter:{chapter}:{scene}:{npc}` choice별 카운팅

    - `telemetry_dialogue_choice_events` 테이블에 raw insert 시도
    - `idempotency_key` 충돌 시 deduped 업데이트
    - 테이블 미구축/오류 시 서버 동작 유지(경고 로그)

- `server/src/telemetry/dialogueTelemetryServer.ts`
  - 핸들러를 async로 전환
  - 기존 dedupe + ack 흐름 유지
  - 영속 저장 함수 호출 추가

- `server/src/socket/socketHandler.ts`
  - `telemetry:dialogue_choice` 리스너를 async await로 변경
