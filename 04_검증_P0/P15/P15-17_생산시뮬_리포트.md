## 2) 생산 워크플로우 시뮬레이션

### 2.1 단계별 파이프라인

```
[프롬프트 준비] → [이미지 생성] → [후처리] → [QA 검증] → [시트 조립] → [카탈로그 등록]
     ↓               ↓              ↓            ↓              ↓
  JSON 파일      AI API 호출    remove_bg     자동 검증     spritesheet
  수동 작성      batch_gen      color_corr    qa_runner     assembler
  ~완료          병렬 실행      파이프라인     반려→재생성    asset_catalog
```

## 4) 최적화 제안

### 4.1 스프라이트 생성 최적화

| 방법 | 효과 | 구현 난이도 |
|------|------|-----------|
| **ControlNet 포즈 재사용** | 동일 포즈 → 다른 캐릭터. 생성 시간 30% 절감 | ✅ 이미 구현 (P15-01) |
| **배치 병렬화 4→8** | 시간 50% 추가 절감 (GPU 메모리 의존) | 중간 |
| **Tile 모드 생성** | 타일 전용 seamless 생성. 후처리 생략 | 중간 |
| **LCM/Turbo 모델** | 35→8 steps. 생성 속도 4배 (품질 약간 하락) | 낮음 |

### 4.2 QA 반려율 감소 전략

| 방법 | 예상 반려율 감소 |
|------|--------------|
| 프롬프트 파인튜닝 (사전 10장 테스트) | 20% → 12% |
| LoRA 모델 추가 학습 | 12% → 8% |
| 자동 색보정 임계값 조정 | 8% → 5% |

### 4.3 권장 실행 순서

```
Phase 1 (Day 1): 일러스트 + 초상화 (252장) → QA → 확정
Phase 2 (Day 2-3): 스프라이트 배치 (3,744장) → 자동 QA
Phase 3 (Day 4): 타일셋 + 배경 (243장) → 심리스 검증
Phase 4 (Day 5): 레이드보스 img2img (128호출) → 수동 QA
Phase 5 (Day 5): 시트 조립 + 카탈로그 + 최종 확인
```
