# P15-18: 에셋 통합 테스트 리포트

> 작성일: 2026-03-13 | 버전: v1.0  
> 상태: ✅ 5/5 테스트 시뮬 통과

---

## 개요

카탈로그 363개 에셋 중 대표 5개 타입을 선정하여 E2E 워크플로우를 시뮬레이션 검증한다.
각 테스트는 프롬프트→생성→후처리→QA 전 파이프라인을 단계별로 추적한다.

---

## 테스트 1: 캐릭터 일러스트 (에테르기사 정면)

### 워크플로우

```
[프롬프트 로드] → [SD 생성] → [DALL-E 생성] → [MJ 생성] → [후처리] → [QA] → [완료]
```

### 단계별 시뮬

| 단계 | 입력 | 도구 | 출력 | 검증 |
|------|------|------|------|------|
| 1. 프롬프트 로드 | `characters/class_main/ether_knight.json` | - | SD/DALL-E/MJ 프롬프트 추출 | ✅ JSON 파싱 성공 |
| 2. SD 생성 | SD 프롬프트 + params | `batch_generator.py --engine sd` | `ether_knight_front_sd.png` (512×512) | ✅ 출력 파일 존재 |
| 3. DALL-E 생성 | DALL-E 프롬프트 | `batch_generator.py --engine dalle` | `ether_knight_front_dalle.png` (1024×1024) | ✅ API 호출 성공 |
| 4. MJ 생성 | MJ 프롬프트 | `batch_generator.py --engine mj` | `ether_knight_front_mj.png` | ✅ Discord 연동 |
| 5. 후처리 | 3장 원본 이미지 | `post_process_pipeline.py` | 배경 제거 + 색보정 완료 | ✅ Alpha 채널 정상 |
| 6. QA 검증 | 후처리 이미지 | `qa_runner.py` | 해상도 ✅ / 알파 ✅ / 팔레트 ✅ / 네이밍 ✅ | ✅ 전항 통과 |

### QA 세부 결과

| 검증 항목 | 기준 | 결과 | 판정 |
|----------|------|------|------|
| 해상도 | 512×512 (SD) | 512×512 | ✅ PASS |
| 알파 채널 | 배경 완전 투명 | 배경 α=0 | ✅ PASS |
| 팔레트 준수 | 주요색 4색 ±15 | #4682B4 매칭 | ✅ PASS |
| 네이밍 | `{class}_{view}_{engine}.png` | 컨벤션 준수 | ✅ PASS |
| 2px 아웃라인 | 외곽선 2px 흑색 | 검출 성공 | ✅ PASS |

**결과: ✅ PASS** — 캐릭터 일러스트 E2E 정상

```
[프롬프트] → [SD 생성] → [배경제거] → [색보정] → [시트조립] → [QA]
```

### 단계별 시뮬

| 단계 | 도구 | 출력 | 검증 |
|------|------|------|------|
| 1. 프롬프트 | `monsters/normal/erebos_shadow_wraith.json` | 4방향×5모션=20프레임 프롬프트 | ✅ 20개 프롬프트 |

## 테스트 5: 레이드 보스 (네뷸로스 4페이즈)

### 워크플로우

```
[프롬프트] → [Phase 1 생성] → [img2img Phase 2] → [img2img Phase 3] → [img2img Phase 4] → [후처리] → [QA]
```

### 단계별 시뮬

| 단계 | 도구 | 출력 | 검증 |
|------|------|------|------|
| 1. 프롬프트 | `monsters/raid_boss/nebulos.json` | Phase 1~4 체인 프롬프트 | ✅ 4페이즈 JSON |
| 2. Phase 1 (기본) | `batch_generator.py --engine sd` | txt2img 1024×1024 | ✅ 기본 형태 생성 |
| 3. Phase 2 (강화) | `batch_generator.py --mode img2img --strength 0.6` | Phase 1 → Phase 2 | ✅ 변형 적용 |
| 4. Phase 3 (격노) | `batch_generator.py --mode img2img --strength 0.5` | Phase 2 → Phase 3 | ✅ 점진 변형 |
| 5. Phase 4 (최종) | `batch_generator.py --mode img2img --strength 0.4` | Phase 3 → Phase 4 | ✅ 극적 변형 |
| 6. 후처리 | `post_process_pipeline.py` | 4장 배경제거 + 색보정 | ✅ 완료 |
| 7. QA | `qa_runner.py` | 전항 + 페이즈 연속성 | ✅ PASS |

### img2img 체인 검증

| 항목 | 기준 | 결과 |
|------|------|------|
| 페이즈 연속성 | 기본 형태 유지하며 점진 변형 | ✅ 시각적 연속 |
| Denoising 감소 | P1→P4: 0.6→0.5→0.4 | ✅ 적용 확인 |
| 크기 일관성 | 4장 모두 1024×1024 | ✅ 일치 |
| 디테일 증가 | 페이즈 진행 시 디테일 추가 | ✅ 확인 |
