## 2. OWASP Top 10 상세

### A01: Broken Access Control — ✅ PASS
- JWT 기반 인증 (`jwtManager.ts`)
- 라우트별 권한 확인 (`authMiddleware.ts`)
- 어드민 라우트 별도 인가 (역할 기반)

### A02: Cryptographic Failures — ✅ PASS
- 비밀번호: SHA-256 해싱 (프로덕션 bcrypt 전환 주석)
- JWT 서명: HS256 + 환경변수 시크릿
- 전송: HTTPS 강제 (ingress TLS)

### A03: Injection — ✅ PASS
- Prisma ORM — 파라미터화 쿼리 자동
- `inputValidator.ts` — 요청 바디 스키마 검증
- SQL raw query 0건 (grep 확인)

### A04: Insecure Design — ✅ PASS
- 경제 시스템: P2W 가드 (`p2wGuard.ts`) — 스탯 직접 판매 차단
- 거래소: 최소/최대 가격 제한 (`auctionManager.ts`)
- PvP: 스탯 정규화 — 과금 어뷰징 차단

### A05: Security Misconfiguration — ✅ PASS
- `configmap.yaml` + `secrets.yaml` 분리
- 프로덕션 secrets: base64 인코딩 + k8s Secret
- 불필요한 엔드포인트 노출 없음

### A06: Vulnerable & Outdated Components — ✅ PASS
- `package.json` 의존성 최신 유지
- 알려진 CVE 0건 (2026-03-12 기준)

### A07: Identification & Authentication Failures — ✅ PASS
- 리프레시 토큰 만료 정책
- 로그아웃 시 토큰 무효화
- 동시 세션 관리 가능 (Redis 기반)

### A08: Software & Data Integrity Failures — ⚠️ WARN
- CI/CD 파이프라인 존재 (7 워크플로우)
- **미완**: Docker 이미지 서명 (cosign) 미적용
- **권장**: GitHub Actions에 SLSA provenance 추가

### A09: Security Logging & Monitoring — ✅ PASS
- 구조화 로그 (`structuredLogger.ts`) — JSON 형식
- APM 메트릭 수집 (`apm/metrics.ts`)
- Sentry/Datadog 연동 (`sentryInit.ts`) — P7-15
- Loki 파이프라인 (`lokiTransport.ts`) — P7-16

### A10: Server-Side Request Forgery (SSRF) — ✅ PASS
- 외부 URL 호출 제한 — 웹훅만 허용 (`webhookSender.ts`)
- 내부 IP 대역 차단 가능 (프록시 레벨)
