## 1. 목표

서버 측 전투 엔진을 완성한다. 현재 `server/src/combat/`에는 상태이상(statusEffectManager), 콤보(comboManager), HP 계산(hpResolver)만 존재한다. 이를 기반으로 전투 루프, 데미지 공식, 몬스터 AI, 보상, 레벨업, 스킬, 보스 메카닉, 전투 로그, 실시간 소켓 핸들러를 구현한다.

## 3. 구현 계획

### 3.1 전투 루프 엔진 (`combatEngine.ts`)
- 턴/실시간 하이브리드: 1초 틱 기반 ATB(Active Time Battle) 방식
- 각 틱마다: 속도 게이지 충전 → 가득 차면 행동 실행
- 전투 상태 머신: `IDLE → PREPARING → IN_PROGRESS → RESOLVING → COMPLETED`
- 다수 대 다수 전투 지원 (파티 vs 몬스터 그룹)

### 3.2 데미지 계산 (`damageCalculator.ts`)
- 물리 데미지: `(ATK × skillMultiplier - DEF × 0.5) × elementMultiplier × critMultiplier × randomVariance`
- 마법 데미지: `(MATK × skillMultiplier - MDEF × 0.4) × elementMultiplier × critMultiplier × randomVariance`
- 속성 상성: 6속성 (화/수/풍/지/명/암) 순환 상성 1.5x / 0.67x
- 크리티컬: `critRate` 확률, `1.5 + critDamage%` 배율
- 방어 관통, 데미지 감소 상한(90%)

### 3.3 몬스터 AI (`monsterAI.ts`)
- 3단계 AI: Basic(랜덤 공격) / Tactical(약한 대상 우선) / Boss(페이즈별 패턴)
- 어그로 테이블: 데미지 기여 + 힐량 + 디버프 가중치
- 스킬 사용 조건: HP%, 쿨다운, 대상 수, 확률
- 197종 몬스터 행동 패턴 지원

### 3.4 보상 시스템 (`rewardEngine.ts`)
- 경험치: 몬스터 레벨 × 기본 EXP × 레벨 차이 보정
- 골드: 몬스터 기본 골드 × (1 + 파티 보너스)
- 아이템 드롭: 드롭 테이블 기반, 등급별 확률 (커먼~레전더리)
- 파티 분배: 균등 분배 / 순차 / 자유 경쟁

### 3.5 레벨업 시스템 (`levelUpSystem.ts`)
- 경험치 테이블: `requiredExp = baseExp × level^1.5`
- 레벨업 시 스탯 자동 증가 (클래스별 성장률)
- 자유 스탯 포인트: 레벨당 3포인트
- 최대 레벨 100, 전직 요구 레벨 (30, 60, 80)

### 3.6 스킬 시스템 (`skillSystem.ts`)
- 스킬 쿨다운 관리 (틱 단위)
- 마나 소모 + 잔여 마나 검증
- 마나 자연 회복: 틱당 `maxMp × 0.01`
- 6클래스 × 스킬 트리 연동

### 3.7 보스 페이즈 (`bossPhaseManager.ts`)
- HP% 기반 페이즈 전환 (100→75→50→25→10)
- 페이즈별 스킬셋 변경, 소환, 광역 패턴
- 분노(Enrage) 타이머: 시간 초과 시 공격력 2배

### 3.8 전투 로그 (`combatLogger.ts`)
- 모든 전투 액션 기록 (데미지, 힐, 버프, 스킬 사용)
- 리플레이 데이터 직렬화 (JSON 형식)
- 전투 통계 집계 (DPS, 최대 데미지, 총 데미지)

### 3.9 실시간 소켓 핸들러 (`combatSocketHandler.ts`)
- `combat:start` — 전투 인스턴스 생성
- `combat:action` — 스킬 사용 / 아이템 사용 / 방어
- `combat:tick` — 서버→클라이언트 전투 상태 브로드캐스트
- `combat:end` — 전투 종료 + 보상 전달

### 3.10 REST API 확장 (`combatRoutes.ts`)
- `POST /combat/start` — 전투 시작
- `POST /combat/action` — 행동 명령
- `GET /combat/:combatId/state` — 전투 상태 조회
- `POST /combat/:combatId/end` — 전투 종료
- `GET /combat/:combatId/log` — 전투 로그 조회

## 4. 기술 제약
