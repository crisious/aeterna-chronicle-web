## 2. 목표

전체 프로젝트를 **처음부터 끝까지 빌드 → 실행 → 테스트 → 배포 준비**까지 완료한다.

### 2.1 핵심 과제

1. **전체 빌드 파이프라인**
   - `npm install` → `tsc` → 서버/클라이언트 동시 빌드
   - 빌드 에러 0건 달성

2. **DB 완전 마이그레이션**
   - PostgreSQL 프로덕션 스키마 적용
   - 전체 시드 데이터 (챕터 1~8, 시즌 1~4)
   - Prisma generate + migrate deploy

3. **환경 설정 표준화**
   - `.env.production` 템플릿
   - 시크릿 관리 (JWT_SECRET, DB_URL 등)
   - 환경별 분기 (dev/staging/prod)

4. **통합 테스트**
   - 서버 API 전 엔드포인트 테스트
   - 클라이언트 전체 라우트 접근 테스트
   - 서버-클라이언트 E2E 플로우 테스트
   - 에셋 로딩 전수 검증

5. **CI/CD 파이프라인**
   - GitHub Actions 워크플로우 (lint → test → build → deploy)
   - Docker 컨테이너화 (서버 + 클라이언트)
   - docker-compose.yml (DB + 서버 + 클라이언트)

6. **프로덕션 배포 준비**
   - 배포 가이드 문서
   - 모니터링/로깅 설정
   - 성능 기준선 측정

## 3. 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| tsc 전체 빌드 에러 | 배포 불가 | 데모 범위 외 코드의 타입 에러 사전 스캔 |
| 전체 시드 데이터 양 방대 | 시딩 시간 과다/실패 | 단계별 시딩 (시즌별) + 트랜잭션 |
| Docker 이미지 크기 과다 | 배포 지연 | 멀티스테이지 빌드, 에셋 CDN 분리 |
| CI/CD 파이프라인 구성 복잡 | 구성 시간 과다 | 최소 구성(lint+test+build) → 점진 확장 |
| 프로덕션 환경 차이 | 로컬 성공/프로덕션 실패 | docker-compose로 로컬 프로덕션 환경 모사 |
