## 2) 핵심 기술 전환: SD WebUI → ComfyUI API

### 2.1 현재 문제

`batch_generator.py`는 **SD WebUI API** (`/sdapi/v1/txt2img`)를 대상으로 구현되어 있으나, 실제 가동 중인 엔진은 **ComfyUI 0.16.4**. API 프로토콜이 완전히 다르다.

| 항목 | SD WebUI | ComfyUI |
|------|----------|---------|
| 엔드포인트 | `/sdapi/v1/txt2img` | `/prompt` (WebSocket 기반) |
| 요청 형식 | 파라미터 JSON | 워크플로우 JSON 그래프 |
| 응답 | Base64 이미지 직접 반환 | 비동기 — history 조회 후 다운로드 |
| ControlNet | `alwayson_scripts` 필드 | 워크플로우 노드로 직접 연결 |
| 진행 확인 | 없음 (동기 응답) | WebSocket `/ws` 실시간 이벤트 |

### 2.2 전환 설계

```
ComfyUI API 워크플로우:

1) 워크플로우 JSON 템플릿 준비 (txt2img, txt2img+ControlNet)
2) 프롬프트/시드/파라미터를 워크플로우 노드에 주입
3) POST /prompt → prompt_id 획득
4) WebSocket /ws 로 진행률 모니터링
5) GET /history/{prompt_id} → 출력 파일명 획득
6) GET /view?filename={name} → 이미지 다운로드
```

### 2.3 ComfyUI 워크플로우 템플릿 구조

**txt2img 기본:**
```json
{
  "3": { "class_type": "KSampler", "inputs": {
    "seed": -1, "steps": 35, "cfg": 8,
    "sampler_name": "dpmpp_2m", "scheduler": "karras",
    "denoise": 1.0, "model": ["4", 0], "positive": ["6", 0],
    "negative": ["7", 0], "latent_image": ["5", 0]
  }},
  "4": { "class_type": "CheckpointLoaderSimple", "inputs": {
    "ckpt_name": "v1-5-pruned-emaonly.safetensors"
  }},
  "5": { "class_type": "EmptyLatentImage", "inputs": {
    "width": 512, "height": 512, "batch_size": 1
  }},
  "6": { "class_type": "CLIPTextEncode", "inputs": {
    "text": "{{POSITIVE_PROMPT}}", "clip": ["4", 1]
  }},
  "7": { "class_type": "CLIPTextEncode", "inputs": {
    "text": "{{NEGATIVE_PROMPT}}", "clip": ["4", 1]
  }},
  "8": { "class_type": "VAEDecode", "inputs": {
    "samples": ["3", 0], "vae": ["4", 2]
  }},
  "9": { "class_type": "SaveImage", "inputs": {
    "filename_prefix": "{{ASSET_ID}}", "images": ["8", 0]
  }}
}
```

**txt2img + ControlNet OpenPose:**
```
위 기본 + ControlNetLoader + ControlNetApply 노드 추가
→ 포즈 JSON을 OpenPosePreprocessor 입력으로 연결
```

- 디스크: 368 × 349KB ≈ 125MB

**Phase 2: 프레임 변형 생성** (P19+ 또는 P18 후반)
- ControlNet 포즈 360개 활용
- 모션별 프레임 생성 (idle 4장, walk 8장 등)

## 7) 기술 구현 상세

### 7.1 batch_generator.py ComfyUI 엔진 설계

```python
class ComfyUIEngine:
    """ComfyUI API 연동 엔진"""
    
    def __init__(self, config):
        self.api_url = config.get("api_url", "http://127.0.0.1:8188")
        self.client_id = str(uuid.uuid4())
        self.workflow_dir = Path(config.get("workflow_dir", "workflows/"))
    
    def generate(self, job, output_dir):
        # 1) 워크플로우 템플릿 로드 + 파라미터 주입
        workflow = self._load_workflow(job)
        workflow = self._inject_params(workflow, job)
        
        # 2) POST /prompt
        resp = requests.post(f"{self.api_url}/prompt", json={
            "prompt": workflow,
            "client_id": self.client_id
        })
        prompt_id = resp.json()["prompt_id"]
        
        # 3) WebSocket 진행률 모니터링 (선택적)
        self._wait_completion(prompt_id)
        
        # 4) GET /history → 출력 파일명
        history = requests.get(f"{self.api_url}/history/{prompt_id}").json()
        outputs = history[prompt_id]["outputs"]
        
        # 5) GET /view → 이미지 다운로드
        for node_id, node_output in outputs.items():
            for image in node_output.get("images", []):
                img_data = requests.get(f"{self.api_url}/view", params={
                    "filename": image["filename"],
                    "subfolder": image.get("subfolder", ""),
                    "type": image.get("type", "output")
                }).content
                save_path = output_dir / f"{job.id}.png"
                save_path.write_bytes(img_data)
                job.result_path = str(save_path)
        
        return True
```

### 7.2 프롬프트 주입 로직

기존 프롬프트 JSON 구조를 ComfyUI 워크플로우에 매핑:

```python
def _inject_params(self, workflow, job):
    """프롬프트 JSON → ComfyUI 워크플로우 노드 주입"""
    # 긍정 프롬프트 → CLIPTextEncode 노드
    workflow["6"]["inputs"]["text"] = job.prompt
    # 부정 프롬프트 → CLIPTextEncode (negative)
    workflow["7"]["inputs"]["text"] = job.negative_prompt
    # 시드 → KSampler
    if job.params.get("seed", -1) != -1:
        workflow["3"]["inputs"]["seed"] = job.params["seed"]
    else:
        workflow["3"]["inputs"]["seed"] = random.randint(0, 2**32)
    # 해상도 → EmptyLatentImage
    workflow["5"]["inputs"]["width"] = job.params.get("width", 512)
    workflow["5"]["inputs"]["height"] = job.params.get("height", 512)
    # 파일명 프리픽스
    workflow["9"]["inputs"]["filename_prefix"] = job.id
    return workflow
```

### 12.1 catalog_to_prompts.py 설계

현재 `asset_catalog.py`에 1,248개 에셋 정의가 있지만, 실제 프롬프트 JSON은 368개뿐. 나머지 680개를 카탈로그 메타데이터로부터 자동 생성:

```
입력: asset_catalog.py의 에셋 정의 (카테고리, 이름, 색상, 설명)
  ↓
변환 규칙:
  - 카테고리별 프롬프트 템플릿 (아이콘/VFX/코스메틱 각각)
  - 스타일 가이드 + 지역 팔레트 자동 적용
  - 네거티브 프롬프트 공용 세트
  ↓
출력: assets/prompts/{category}/{asset_id}.json
```
