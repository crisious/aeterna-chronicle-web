## P10 개요 — 웹 아키텍처 리팩터링

`docs/web-architecture-analysis.md` 분석 리포트에서 식별된 구조적 문제를 우선순위별로 해소한다.

**핵심 원칙:**
- 기능 변경 없이 구조만 개선 (리팩터링 전후 기능 동등성 보장)
- shared 계약을 SSOT(Single Source of Truth)로 확립
- 서버/클라이언트/어드민 3앱의 경계와 계약을 명확히 분리
- 테스트 커버리지와 문서를 리팩터링에 맞춰 보강
