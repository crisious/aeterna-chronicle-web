## 3. P14 트랙 분석: F+G 병합

### 3.1 트랙 F — 데이터 분석 + ML

**목표**: 운영 자동화 + 밸런스 지능화 + 사용자 행동 기반 의사결정

**근거**: P6에서 analyticsEngine(KPI/BI)을 구축했으나 ML 기반 예측/추천/밸런스 자동화는 미구현. 248티켓 규모의 시스템에서 수동 밸런스 조정은 비효율적.

| 영역 | 현재 상태 | P14 목표 |
|------|----------|---------|
| 데이터 수집 | Prisma 직접 쿼리, KPI 스냅샷 | ClickHouse 웨어하우스 + 이벤트 SDK |
| 이탈 예측 | 미구현 | 로그인/레벨/소셜 기반 이탈 점수 모델 |
| 추천 시스템 | 미구현 | 아이템/던전/퀘스트 개인화 (협업 필터링) |
| 밸런스 조정 | 수동 시드 수정 | 승률/클리어율 기반 자동 튜닝 제안 |
| A/B 테스트 | 미구현 | 기능/UI/보상 분기 실험 프레임워크 |
| 이상 탐지 | P7 anomalyDetector (규칙 기반) | ML 기반 매크로/봇/어뷰징 탐지 v2 |
| 운영 대시보드 | 어드민 기본 UI | ClickHouse + Grafana 실시간 대시보드 |

**핵심 산출물 (7티켓)**:
- 데이터 웨어하우스 설계 (ClickHouse 스키마 + ETL)

## 4. 병합 근거

### 4.1 F+G 시너지

| 시너지 포인트 | 설명 |
|-------------|------|
| 데이터 → 밸런스 | 신규 클래스/레이드/PvP 밸런스를 ML 자동 튜닝으로 조정 |
| 이벤트 수집 → 콘텐츠 | 시즌 4 콘텐츠 소비 패턴을 실시간 추적 → 보상 조정 |
| 추천 → 리텐션 | 신규 던전/퀘스트를 유저 레벨/선호도 기반 추천 |
| 이탈 예측 → 운영 | 시즌 4 진입 후 이탈 위험 유저에게 타겟 보상 |
| A/B 테스트 → 시즌패스 | 보상 구성 A/B 실험으로 전환율 최적화 |

### 4.2 20티켓 배분

- **트랙 F (데이터/ML)**: 10티켓 (인프라 3 + ML 3 + 운영/탐지 3 + RC 공유 1)
- **트랙 G (콘텐츠)**: 10티켓 (기반 4 + 시스템 4 + 콘텐츠 확장 2)
- **공통 RC**: P14-20 (통합 검증)

### 4.3 스프린트 배치 전략

| 스프린트 | 주제 | F 티켓 | G 티켓 |
|---------|------|--------|--------|
| 1-2 (4주) | 데이터 인프라 + 시즌4 기반 | 3 (웨어하우스/SDK/이탈예측) | 4 (챕터8/클래스/지역/시즌패스) |
| 3-4 (4주) | ML 시스템 + 콘텐츠 | 3 (추천/밸런스/A/B) | 4 (레이드/펫진화/PvP3/레벨캡) |
| 5-6 (4주) | 통합 + RC | 3 (대시보드/이상탐지v2/RC) | 4 (이벤트/몬스터/던전/RC) |
