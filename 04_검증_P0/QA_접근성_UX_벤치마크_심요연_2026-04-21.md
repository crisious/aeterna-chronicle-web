### 2.3 본궁의 소견

세 가지 실질적 결함이 드러났습니다.

1. **비활성 텍스트(`#606060`)가 AA조차 미달** — 메뉴 비활성 옵션은 시각적으로 '읽을 수 없음'이 기준이 아니라 '읽을 수 있으되 흐릿함'이 WCAG 기준입니다. `#707080` 또는 `#808090`으로 상향 권장(목표 대비 3.5~4.5).
2. **보조 텍스트 `#A0A0A0`는 AA 통과, AAA 미달** — DESIGN.md v1.0이 "WCAG AAA"를 선언했으나 AAA(7:1)를 넘기려면 `#B0B0B0` 이상으로 조정 필요.
3. **경고 텍스트(`#FF4444`) AAA 미달** — 가장 치명적인 '오류' 메시지가 AAA 미달이라는 건 모순. `#FF6666` 또는 배경을 약간 어둡게 `#12121F` 조합으로 해결 가능.
4. **패널/프레임의 배경 간 대비가 낮음** — 정보 계층이 시각적으로 약하게 느껴질 수 있음. WCAG 1.4.11은 UI 컴포넌트 '경계'(테두리)가 3:1이면 통과이므로, 테두리가 현재 금/청 계열이라면 OK지만 **단색 영역 구분만으로는 정보 계층이 무너짐**.

### 2.4 즉시 조치 권고

```diff
- TEXT Disabled:    #606060  // 대비 2.71:1 (AA 실패)
+ TEXT Disabled:    #7A7A8A  // 대비 3.60:1 (AA 비텍스트 통과, 시각적 '희미함' 유지)

- TEXT Secondary:   #A0A0A0  // AAA 미달
+ TEXT Secondary:   #B8B8B8  // 8.1:1 (AAA 통과)

- ACCENT Warning:   #FF4444  // AAA 미달
+ ACCENT Warning:   #FF6B6B  // 6.2:1 (AA 강화), 혹은 배경 #12121F와 조합
```

## 5. 치명적 갭 3종 (릴리즈 블로커급)

### 🔴 **GAP-1: 선언/실측 불일치 (DESIGN.md "AAA" vs 실측 AA)**
- **증거:** `#606060` 비활성 2.71:1로 AA 실패, 보조/경고 AAA 미달
- **영향:** WCAG 준수 마케팅 문구 사용 시 법적 리스크(특히 EU EAA 2025)
- **조치:** §2.4 팔레트 패치 + `AccessibilityAudit.ts`에 AAA 모드 추가

### 🟡 **GAP-2: 자막/색맹 사용 지표 미수집**
- **증거:** `accessibility_settings`는 localStorage에만 저장, 서버 집계 없음
- **영향:** "누가 어떤 접근성 기능을 쓰는가"를 알 수 없어 개선 우선순위 수립 불가
- **조치:** `SettingsManager.saveSettings()` 호출 시 서버 텔레메트리 1회 전송 (anonymous)

### 🟡 **GAP-3: Phaser Canvas 내부 포커스 트랩 테스트 부재**
- **증거:** `focusChain` 코드는 있으나 실제 E2E 테스트 없음 (P17-20도 "수동 검증"으로 표기)
- **영향:** 모달이 중첩될 때 Tab 트랩이 바깥으로 유출될 위험
- **조치:** Playwright + @axe-core/playwright로 주요 씬 5개(로그인/메뉴/전투/인벤/대화) 자동 스캔 추가
