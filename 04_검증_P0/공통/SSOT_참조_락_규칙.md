## 2) 수정 우선순위 규칙 (Reference Lock)

1. 수치 충돌 발생 시 아래 순서로 판정:
   - `combat_balance_table.md` → `monster_data_table.md`/`item_data_table.md`
2. 엔딩 조건 충돌 발생 시:
   - `멀티엔딩_플래그_설계.md` 우선
3. 시나리오/대사 문서와 시스템 문서 충돌 시:
   - 시스템 수치/플래그는 SSOT 우선, 시나리오 문서는 이를 따르도록 수정
4. 임시 문서(리포트/메모)는 SSOT를 덮어쓰지 못함

---

## 3) 문서 헤더 표준

모든 파생 문서 상단에 아래 라벨을 권장한다.

```markdown
> SSOT 참조: `GDD_final.md`, `멀티엔딩_플래그_설계.md`, `combat_balance_table.md`
> 이 문서는 파생 문서이며, 기준값 충돌 시 SSOT를 따른다.
```

## 4) 변경 프로세스 (Change Gate)

1. SSOT 수정 요청 발생
2. 영향 범위 체크 (`monster/item/scenario/ui`)
3. SSOT 선수정
4. 파생 문서 동기화
5. `정합성_검증_체크리스트.md` 점검 항목 갱신
6. 변경 로그 기록 (날짜/수정자/근거)

## 5) 금지 규칙
