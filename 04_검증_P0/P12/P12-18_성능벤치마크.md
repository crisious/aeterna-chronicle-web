## 6. 병목 분석 및 권장사항

### 해결된 병목
1. ✅ N+1 쿼리 14곳 → 0곳 (Prisma include/select 최적화)
2. ✅ 리더보드 매 요청 DB 풀스캔 → Redis sorted set 캐시
3. ✅ 미사용 Phaser 모듈 번들 포함 → tree-shaking 제거
4. ✅ 에셋 전체 초기 로드 → lazy load 전환

### 잔여 관찰 사항
1. ⚠️ 10K CCU 시 전투 액션 P99 480ms — 수평 스케일링(pod 증설) 또는 전투 로직 캐시 추가 권장
2. ⚠️ character 캐시 히트율 78% — TTL 30s → 60s 상향 또는 write-through 캐시 전환 검토
3. ℹ️ WebSocket 연결당 메모리 ~5KB — 10K 동시접속 시 ~50MB 추가 (현재 여유분 내)
